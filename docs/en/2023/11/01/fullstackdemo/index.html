Frame

We're using a back-to-back separation frame, FastAPI+Vue. Strengths 1.Clear lines of responsibilityThe front end is responsible for presentation and interaction, and the back end is responsible for data processing and business logic, each developing independently and without interaction. 2.Increased development efficiency: It can be developed in parallel at the back and back end, significantly reducing the development cycle of the project. 3.Improving maintenanceability: Due to the segregation of the back and back ends, if changes to UI or the back end logic are required, only the corresponding part will be required and will not affect the system as a whole. I'm sorry.Technical inflexibility: The front and back ends can select the appropriate technology stacks for development to meet the individual needs of the project. 5.Easy to expand.: Communication back and forth through API allows for easy system expansion, such as the addition of a mobile end, a small program, etc. 6.Adaptation to micro-service structures• In the micro-service architecture, back-to-back segregation can better manage and maintain services.

4. Write an interface to the back end in the vue file This part is mainly written in section js. Example code: The data section is all the variables required for the current page, and the methods section is all the functional methods. If you need to call the variables in the function, you need to specify this. 
<script>
import axios from "axios";
export default {
  name: 'HelloWorld',
  data () {
    return {
      msg: 'Welcome to Your Vue.js App',
      tableData: [],
    }
  },

  methods: {
    fetch_paper() {
      const limit=8
      const param = `num=${limit}`;
      // 在这里调接口
      axios
        .get(`http://127.0.0.1:8000/paper?${param}`)
        .then((res) => {
          this.tableData = res.data.data;
          this.time = res.data.time;
          console.log(this.tableData, this.time);
        })
        .catch(() => {
          this.$notify({title: "Error", message: "Please try again!", type: "error",});
        });

    },
  }
}
</script>

Backend

1. Installation of Python packages Use fastapi as a quick backend deployment, use pymysql if access to the database is required, and aiomysql is recommended to be used to operate the database in aiomysql. 
pip install fastapi
pip install uvicorn
pip install pymysql
2. Example codes: 
import uvicorn
from fastapi import FastAPI, Query, Form, APIRouter, File, UploadFile
from fastapi.middleware.cors import CORSMiddleware
import time

app = FastAPI(
    title="demo",
    docs_url='/api/v1/docs',
    redoc_url='/api/v1/redoc',
    openapi_url='/api/v1/openapi.json'
)

router = APIRouter()

@router.get('/paper')
async def fetch_paper(
    num: int = Query(..., description='returned paper num', example='10')
):
    start = time.time()
    print(num)
    return {'time': time.time() - start, 'data': num}

@router.post('/add_paper')
async def add_paper(
        name: str = Form(..., description='paper name', example='Attention is all you need'),
        info: str = Form(..., description='paper info', example='NIPS 2017')
):
    start = time.time()
    print(name, info)
    return {'time': time.time() - start}

@router.put('/update_paper')
async def update_paper(
        p_id: str = Form(..., description='paper id', example='1234'),
):
    start = time.time()
    print(p_id)
    return {'time': time.time() - start}

@router.delete('/delete_paper')
async def delete_paper(
        p_id: str = Query(..., description='paper is', example='1234')
):
    start = time.time()
    print(p_id)
    return {'time': time.time() - start}

app.include_router(router)

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

if __name__ == '__main__':
    uvicorn.run(app=app, host="127.0.0.1", port=8000, workers=1)

Database

1. Download and install MySQL database and database management tool and recommend the use of navicat Import data in MySQL Here you can use the example data paper_db.sql (with a small amount of paper information) to import this library directly with navicat (under the database you create) 3. Backend linked database and operated in the database (example here to extract data from the database) 
import pymysql

# 连接配置信息
config = {
    'host': 'localhost',
    'port': 3306,  # MySQL默认端口
    'user': 'root',  # mysql默认用户名
    'password': '12345678',
    'db': 'paper_db',  # 数据库
    'charset': 'utf8mb4',
    'cursorclass': pymysql.cursors.DictCursor,
}

def db_get_paper(num=10):
		# am_paper为paper_db中的一个表
    try:
        con = pymysql.connect(**config)
        sql = "SELECT * FROM am_paper limit %d" % num
        with con.cursor() as cursor:
            cursor.execute(sql)
            result = cursor.fetchall()
            con.close()
        return result
    except Exception as e:
        print(e)
        return []
