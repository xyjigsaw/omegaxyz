Large models are also an optimizer.

TechnologyMachine LearningDeep LearningNatural Language Processing人工智能神经网络Natural Language ProcessingGPTLLM
Thesis: Large Language Models as Optimizers
arXiv: https://arxiv.org/pdf/2309.03409.pdf
Affiliation: Google DeepMind

🚀Highlight

  • DeepMind introduced a new methodology called Optimization by Warning: OPRO. It no longer relies on traditional optimization algorithms, such as RMSPROP, Adam, etc., but instead shifts its focus to issues or tips that optimize models. The objective is to find instructions that maximize mission accuracy.
  • Specifically, the optimisation problem is described in natural languages, and LLM is then instructed to generate new solutions based on the description of the problem and the typologies previously found. LLM optimization can quickly adapt to different tasks by changing the question description in the prompt, and customise the optimization process by adding commands to specify the solution properties required.

Background

  • Optimization algorithms usually require tailoring to individual tasks to meet specific challenges posed by decision-making spaces and performance environments.
  • Optimization everywhere. While algorithms based on the guidance are powerful for a variety of issues, the lack of gradients poses challenges for many applications in the real world.

Methodology

Definitions (components)

  • The core of meta-prompt: meta-prompt is the basic input of LLM. It consolidates previous solutions, associated performance indicators and a concise description of current tasks. This structured input is designed to guide LLM towards more efficient decomposition.
  • Optimizer LLM: When LLM receives meta-prompt, its goal is to generate improved prompt for better results. DeepMind tested this method in various models (including text-bison, Palm 2-L, gpt-3.5-turbo and gpt-4) to ensure robustness and multifunctionality.
  • Pumper LLM: This component assesses the quality of the newly generated decomposition. It calculates the average score based on the behaviour of the hint on a given data set (e.g. GSM8K and Big-Bench Hard). DeepMind explores different configurations of LLM as an optimizer and evaluator. Notes: We will use LLM for target function evaluation as the scorer LLM, and LLM for optimization as the optimist LLM. They can be different.

Frame

OPRO Framework Summary: 1 for input; 2 optimizer LLM to generate target function; 3 scorer LLM to assess the quality of the solvency; 4 to add the new solver and its fractions to the next optimised meta-prompt. The meta-prompt contains scores of solutions obtained throughout the optimization, as well as some examples of the natural linguistic description of the task and the (in the hint optimization) task.

Case

Blue text contains solution fraction pairs; purple text describes optimization tasks and output formats; orange text is Meta-prompt.Meta-prompt contains two core message clips. The first component is the prior generation of tips and the corresponding training precision. The second is a description of the issue of optimization, which includes several examples chosen randomly from the training focus to illustrate our concerns. It also provides an explanation for LLM to understand the relationship between the different parts and the required output format.

Different from other existing jobs

  • Unlike the recent work of using LLM automatic alerts, each step of the exercise produces a new hint, with the aim of increasing the test accuracy based on the trajectory of the previously generated tip, rather than editing an input tip based on feedback from the natural language or requiring that the new tip follow the same semantic meaning.
  • OPRO uses the complete optimized trajectory to enable LLM to gradually generate new tips to improve the accuracy of tasks throughout the optimization process, where the initial hints are less accurate.

Evaluation

Continuous optimization: linear regression

Mission: In the online regression problem, the goal is to find linear coefficients that best match the response from the input variable in terms of probability.
  • Step1: A random sample of five (w, b) yes.
  • Step 2: Use a meta-prompt hint to optimizer LLM, which includes the best 20 pairs (w, b) of history against the target values and their sorting. Meta-prompt requires a new (w, b) pair, which further reduces the target value. The figure above shows a meta-prompt. Processes are issued eight times, generating up to eight new (w, b) pairs per step to improve optimal stability.
  • Step3: Assess the proposed target value for the target and add it to history.
  • Step4: The process is iterative until the conditions are met.
  • Each model explores (w,b) fewer pairs than an exhaustive search, indicating that the models optimize the black box: comparing figures and suggesting a direction of decline.
  • text-bison and gpt-4 models are better than gpt-3.5-turbo at condensing speed: they reach optimal state with fewer steps. The gpt-4 model performed best.
  • When the ground truth is further from the starting area, the problem of all models becomes more difficult to resolve: all models require more exploration and more steps.

The digestion experiment.

1. how each part of the meta-prompt matters
  • Optimizer LLM output is more vulnerable to the closer meta-prompt end command.
  • The inclusion of precision fractions in the command helps the optimist LLM to better understand the quality differences between previous commands (better than direct sequencing).
  • The example is useful in meta-prompt, as it provides intuitive information about the mission and helps the optimizer better describe the new instructions with phrases. However, more examples do not necessarily improve performance, as only a few examples are usually sufficient (like ICL). In addition, including more examples would lead to a longer meta-prompt, which could distract the optimizer LLM from other important components.
2. the number of generated instructions in each step
  • The article calculates the difference in the random gradient decline process. The generation of multiple commands at each step increases the optimal stability of LLM.
  • On the whole, the best performance has been achieved with the sampling of eight instructions per step.
3. the initial instructions for prompt optimization
  • The performance of different initial instructions varies significantly, especially at the beginning of the optimization.

Key Insights

  • Innovative Optimized Prompts: The study revealed unexpected but effective tips. For example, PaLM 2-L-IT model tips include "take a deep break..." Such a phrase implies a human-like solution to the problem.
  • Diversity in Model-Specific Prompts: Optimizing prompt for different models is different. Each model has its own unique architecture and training modules. This diversity highlights the need to customize individual models in order to obtain the best results. At the same time, a semanticly similar command may have completely different precisions. For example, the PaLM 2-L scorer on the GSM8K test set, “Let's think step by step.”, reaches precision 71.8, “Let's solve the problem.” is 60.5, while “Let's work programme the problem step by step.” is only 49.4.
  • Transferability of Optimized Prompts: OPRO methods do not simply produce valid tips for individual assignments, which are migratory in different mandates.
  • Natural Language Operations: A notable advantage of LLM is their proficiency in natural language processing. This function allows researchers and practitioners to describe complex optimization tasks in simple language, eliminating the need for complex technical terms. It streamlines the process of optimization and makes it easier to access and use.
