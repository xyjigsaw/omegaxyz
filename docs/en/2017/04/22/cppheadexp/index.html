Precompiled header file descriptionThe so-called header file pre-compilation is to pre-compile some MFC standard header files (such as Windows. This speeds up compilation and saves time. The precompiled header file is generated by compiling stdafx.cpp and is named after the project name. Since the suffix of the precompiled header file is "pch", the compilation result file is projectname.pch. The compiler uses precompiled header files through a header file stdafx.h. The header file name stdafx.h can be specified in the project's compilation settings. The compiler believes that all code before the directive #include "stdafx.h" is precompiled. It skips the #include "stdafx.h" directive and uses projectname.pch to compile all code after this directive. (During compilation, the statements before #include "stdafx. h" will not be compiled) Therefore, the first statement of all CPP implementation files is: #include "stdafx.h". In addition, each implementation file CPP contains the following statements:
#ifdef _DEBUG #undef THIS_FILE static char THIS_FILE[] = __FILE__; #endif
This means, if building a debug build, to indicate the name of the current file. __FILE__ is a macro, which is assigned the name of the file currently being compiled during the compiler's compilation process. When the end of the file cannot be found, the solution is as follows: vc.net uses precompiled headers (/Yu) by default. If you don't understand, fatal error C1010 always occurs during compilation after adding a new .h file: An unexpected end-of-file error was encountered when searching for precompiled header instructions. The solution is to add #include "stdafx.h" where the header file is included, or go to the project properties, find the "C/C++" folder, and click the "Precompiled Header" property page. Modify the "Create/Use Precompiled Header" property to "Do Not Use Precompiled Header".3. The origin of preprocessingIn the historical development of C++, many language features (especially the obscurity of the language) come from the C language, and preprocessing is one of them. C++ inherits the C language preprocessor from the C language (C language preprocessor, referred to as Cpp by Dr. Bjarne, I don’t know if it is the abbreviation of C Program Preprocessor).4. Common preprocessing functionsThe main function of the preprocessor is to equivalently replace a resource through the built-in functions of preprocessing. The most common preprocessors are: file inclusion, conditional compilation, layout control and macro replacement. File inclusion: #include is the most common preprocessing, mainly used as a reference to combine the source program text with the file. Conditional compilation: #if, #ifndef, #ifdef, #endif, #undef, etc. are also relatively common preprocessing. They mainly perform selective selection during compilation and comment out some specified codes to achieve version control and prevent repeated inclusion of files. Layout control: #pragma, this is also an important aspect of our application preprocessing. Its main function is to provide unconventional control flow information to the compiler. Macro replacement: #define, this is the most common usage. It can define various functions such as symbolic constants, function functions, renaming, and string splicing.5. Preprocessing instructionsThe format of the preprocessing directive is as follows: #directive tokens The # symbol should be the first non-empty character in this line, and generally we put it at the beginning. If the command cannot fit in one line, it can be controlled through "\", for example: #define Error if(error) exit(1) Equivalent to #defineError\ if(error) exit(1) Let’s take a look at common preprocessing instructions: #define macro definition #undef undefined macro #include text includes #ifdef compile if macro is defined #ifndef Compile if macro not defined #endif ends the control of the compiled block #if the expression is non-zero, compile the code #else Compile as remaining options for other preprocessors #elif This is a combination option of #else and #if //There are examples later #line Change the current line number and file name #error Output an error message #pragma provides non-conventional control flow information to the compiler Below we will explain these preprocessing one by one. Considering the importance and cumbersomeness of macros, we will put it at the end.6. File contains instructionsThis method of preprocessing is the most common.
