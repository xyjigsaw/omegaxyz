Overview of unbalanced data sets for machine learning

TechnologyMachine Learning
Definition: Imbalanced data set: In problems such as classification, the number of positive and negative samples, or the number of samples in each category is inconsistent.When studying imbalanced classes, it is generally believed that imbalance means that the minority class only accounts for 10~20%. In fact, some data sets are far more imbalanced than this. For example: 1. Approximately 2% of credit card accounts are defrauded every year. (Most fraud detection fields are severely imbalanced.) 2. Status medical screening usually detects a very small number of people who do have the status in a large population where the status does not exist (for example, only 0.4% of people with HIV in the United States are HIV positive). 3. Disk drive failures occur approximately 1% per year. 4. The conversion rate of online advertising is estimated to be 10-3to 10-6between. 5. Factory production failure rate is usually about 0.1%. The intuitive impact is that the prediction results of the model trained with these unbalanced data are biased towards the category with more data in the training data. In the example of face detection, most of the detection results of the detector are biased towards not detecting the face image. Another unbalanced data set is credit card fraud transactions. If the data is extracted on average, most of the data will be non-fraudulent transactions, and only a very small part of the data will be fraudulent transactions. Impact: When training and testing on an unbalanced data set, the accuracy obtained is falsely high. For example, in unbalanced data, when the ratio of positive and negative samples is 9:1, when its accuracy is 90%, we have reason to suspect that it will judge all categories as the category with more data.   1. Don’t use accuracy (or error rate) to evaluate your classifier. There are two serious problems. First, the accuracy is simply thresholded at 0.5 to handle binary classification, but this is often wrong when the data is unbalanced. Second, classification accuracy is based on a simple count of errors, and you should know better. You should know which classes are obfuscated and where. If you don't understand these points, readThe Basics of Classifier Evaluation, Part 2Might be useful. You can use a ROC curve, or a PR curve to visualize the performance of your classifier. 2. Instead of using score and predict to obtain hard classifications (labels) from the classifier, use proba and predict_proba to obtain probability estimates. 3. After you obtain the probability estimate, do not blindly use 0.5 as the threshold to divide the categories. Observe the performance curve to determine the threshold to be sampled. Many errors in early papers occurred because researchers simply cut off at 0.5. 4. No matter what you train, always test on the natural distribution. Seesklearn.cross_validation.StratifiedKFold5. You may not need probability estimates for classification, but when you do, use calibration. see sklearn.calibration.CalibratedClassifierCV The following is from the bloghttp://blog.csdn.net/chenriwei2/article/details/49227205solution Solutions: 8 1. Collect more data: Benefits: It can reveal the essential differences of data categories and increase the number of small samples for subsequent data resampling. 2. Try to change the performance evaluation criteria: When data is unbalanced, accuracy has lost its original meaning. Metrics that can be referenced are: 1> Confusion matrix CM 2> Precision 3> Recall 4> F1 score (weighing precision and recall); 5. Kappa 6, ROC curve 3. Resample data: 1. Copy a part of the data with a small number of samples and divide it into more points, reaching a balance (oversampling); 2. Delete some data with too many samples to achieve balance (undersampling); In practice, both oversampling and undersampling are used. In the test, if the total number of samples is relatively large, you can use undersampled data for testing. If the total number of samples is relatively small, you can use oversampled data for testing. In addition, you should test randomly sampled data and non-randomly sampled data, and at the same time, test data with different proportions of positive and negative samples. 4. Generate synthetic data: The simplest is to randomly sample attributes with a relatively small number of samples. Another well-known method is: SMOTE: It is an oversampling method. It creates new sample instances from categories with relatively few samples. Generally, it randomly perturbs a feature from several similar samples. 5. Use different algorithms: Don't try to solve all problems with one method, try some other different methods, such as decision trees, which generally perform better on imbalanced data sets. 6. Try a penalty model: This means adding a new penalty term to the cost function, so that the cost of misjudgment of small sample categories is greater, forcing the model to pay attention to small sample data. For example: SVM with penalty term 7. Use different perspectives: For unbalanced data sets, there are specialized neighborhoods and algorithms to do this. You can refer to their practices and terminology. For example: anomaly detection. 8. Try new improvements: For example: 1. Decompose categories with more samples into more small categories. For example: originally we wanted to distinguish the number 0 from other numbers, which is a two-class classification problem. We can divide other numbers into 9 categories and turn it into a 0-9 classification problem;   Original English text:https://www.svds.com/learning-imbalanced-classes/Original translation:http://blog.csdn.net/zhangf666/article/details/53364465
