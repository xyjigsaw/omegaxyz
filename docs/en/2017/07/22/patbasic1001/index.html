PAT Level B Exercise 1001 (3n+1) conjecture about killing someone without paying for it

Programming LanguagesC & C++TechnologyC++CPAT
Callatz conjecture: For any natural number n, if it is an even number, then cut it in half; if it is an odd number, then cut (3n+1) in half. If you keep cutting like this repeatedly, you must finally get n=1 at a certain step. Karaz announced this conjecture at the World Congress of Mathematicians in 1950. Legend has it that teachers and students at Yale University mobilized to prove this seemingly silly and naive proposition. As a result, the students were not interested in their studies and only focused on proving (3n+1). Some people said that this was a conspiracy and that Karaz was deliberately delaying the progress of teaching and scientific research in the American mathematics community... Our topic today is not to prove the Kalaz conjecture, but for any given positive integer n not exceeding 1000, simply count how many steps (how many chops) are needed to get n=1?Input format:Each test input contains 1 test case, which gives the value of the natural number n.Output format:Output the number of steps required to calculate from n to 1.Input example: 3 Output sample:5 Code:
#include<iostream>
#include<cstdio>
using namespace std;
int main(){
    int n,sum=0;
    while(scanf("%d",&n)!=EOF){
        sum=0;
        while(n!=1){
            if(n%2==0)n/=2;
            else n=(3*n+1)/2;
            sum++;
        }
        cout<<sum<<endl;
    }
}
