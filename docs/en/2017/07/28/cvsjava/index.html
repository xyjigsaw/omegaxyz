What is the difference between C language and JAVA?

高级语言C&C++技术域JAVAC++JAVAC语言

1.The respective advantages of Java and C language

C language is a process-oriented language with high execution efficiency; Java is an object-oriented language with lower execution efficiency than C language. The most important thing about C language is that it has more pointers than Java, which also shows the robustness of Java. Java's multi-threading mechanism allows programs to run in parallel. Java programs are mostly used on the network. C language is not as safe as Java. C language does not have Java's garbage collection mechanism, and the applied space must be released manually. Java has good versatility and can be directly transplanted across platforms, as long as the Java Virtual Machine (JVM) is installed.

2.Which one has better versatility, Java or C language?

In terms of speed, programs written in C language are better than Java, because Java must run in a virtual machine environment. However, because of the virtual machine, Java has gained platform independence, and C language programs may need to be re-modified and compiled to achieve platform transplantation. On the other hand, the C language is more "low-level" than the Java language, so hardware drivers, for example, can be written in C, but not in Java. Another obvious difference between Java and C language is that C language focuses on algorithms, but Java is different. You can just import the package when you want to use it. Algorithms in C language are very important, but algorithms in Java are not as complex as C language. Printing method: C language is printf("……."); Java is System.out.print("……."), you can also use println, ln means line break. The basic data types in Java programs (boolean, char, byte, short, int, long, float, double) are references to objects; C language also has many basic types, as well as struct, union, enum, arrays and pointers. There are no enumerations or union types in Java. Java uses the Unicode character set, and C language usually uses the ASCII character set, A-65, a-97. Variables in C language do not need to be initialized, but variables in Java must be initialized. In C language, the char type occupies one byte, and in Java, the char type occupies 2 bytes.

3The respective characteristics of Java and C language

The object-oriented features of Java mainly include encapsulation, inheritance, and polymorphism. The inheritance relationship of a class is single and non-multiple. A subclass has only one parent class, and the parent class of a subclass has only one parent class. C language supports "operator overloading", which is a very important polymorphic feature and a powerful tool for data abstraction and generic programming. It allows four arithmetic operations to be performed directly on objects. Just like basic data types, Java does not support this polymorphic mechanism, also to reduce complexity. Java can support method overloading and rewriting. Overloading means that a class has multiple behaviors with the same attributes; rewriting means that the behavior of the parent class has different implementations on different subclasses in the inheritance relationship. There are some more modifiers in Java than in C language, such as access modifiers Public (public), Private (private), Protected (protected), and default. Java has the super keyword, which refers to the parent class object. It is usually used to call the constructor or general method of the parent class. C language does not have the super keyword. Both have this, which refers to the current object. In Java, except for static, final, and private, which are static bindings, all methods are treated as dynamic bindings. In Java, classes can be organized and packaged with Packages, but C language does not.
