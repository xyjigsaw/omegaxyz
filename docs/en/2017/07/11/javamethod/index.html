Overview of JAVA methods

TechnologyJavaJava
JAVA methods are similar to C language functions. 1. Program modularity and reusability is one of the central goals of software engineering. Java provides many useful constructs that help accomplish this goal structure. A method is such a structure. 2. The method header specifies the method's modifiers, return value type, method name, and parameters. All methods in this chapter use static modifiers static. 3. Methods can return a value. Return value type return valueType is the data type of the value to be returned by the method. If the method is not Return value, the return value type is the keyword void. 4. The parameter list refers to the type, order and number of parameters in the method. The method name and parameter list together form the method signature (method signature). Parameters are optional, that is, a method may not contain parameters. 5. The return statement can also be used in a void method to terminate the method and return to the caller of the method. In methods, sometimes used It is useful to change the normal process control. 6. The actual parameters passed to the method should be of the same number, type, and order as the formal parameters in the method signature. 7. When the program calls a method, program control is transferred to the called method. The called method executes until the method's return statement or when the closing parenthesis at the end of the method is reached, returns program control to the caller. 8. In Java, methods with return values ​​can also be called as statements. In this case, the calling function simply ignores the return value. 9. Methods can be overloaded. This means that two methods can have the same method name as long as their method parameter lists are different. 10. Variables declared in a method are called local variables. The scope of a local variable starts from the place where it is declared to the area containing the variable. until the end of the block. Local variables must be declared and initialized before use. 11. Method abstraction separates the application and implementation of methods. Users can use methods without knowing how they are implemented. method The implementation details are encapsulated in the method and hidden from users who call the method. This is called information hiding or encapsulation. 12. Method abstraction modularizes programs into neat, hierarchical forms. Writing a program as a collection of concise methods will be better than other methods Easier to write, debug, maintain and modify. This writing style also improves the reusability of methods. 13. When implementing a large program, top-down or bottom-up coding methods can be used. Don't write the entire program in one go. This approach seems to waste more coding time (because you have to compile and run the program repeatedly), but in fact, it will be more economical time and make debugging easier.
