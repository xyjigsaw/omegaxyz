JAVA inheritance and polymorphism

Programming LanguagesTechnologyJavaJava
1. New classes can be defined from existing classes, which is called class inheritance. The new class is called a subclass, subclass or inherited class. The existing class is called a superclass, parent class or base class.
2. The constructor method is used to construct instances of the class. Unlike properties and methods, subclasses do not inherit the constructor methods of the parent class. They can only be called from the constructor of a subclass using the keyword super. 3. The constructor can call the overloaded constructor or the constructor of its parent class. This call must be the first statement of the constructor. If any of them are not explicitly called, the compiler will treat super O as the first statement of the constructor, which calls the no-argument constructor of the parent class. 4. In order to override a method, the method in the subclass must be defined with the same signature as the method in its parent class. 5. Instance methods can only be overridden when they are accessible. This way, the private method cannot be overridden because it is not accessible outside the class itself. If the method defined in the subclass is private in the parent class, then these two methods are completely unrelated. 6. Static methods can be inherited just like instance methods. However, static methods cannot be overridden. If a static method defined in the parent class is redefined in the subclass, then the method defined in the parent class is hidden. 7. Every class in Java inherits from the java.lang.Object class. If a class is defined without specifying an inheritance relationship, its parent class is Object. 8. If the parameter type of a method is a parent class (such as Object), you can pass an object of any subclass (such as Circle class or String class) to the parameter of the method. This is called polymorphism. 9. Because an instance of a subclass is always an instance of its parent class, you can always convert an instance of a subclass into a variable of the parent class. When converting a parent class instance to its subclass variable, you must use a conversion token (subclass name) to perform an explicit conversion to indicate your intention to the compiler. 10. A class defines a type. Types defined by a subclass are called subtypes, and types defined by a parent class are called parent types. 11. When an instance method is called from a reference variable, the actual type of the variable determines at runtime which implementation of the method is used. This is called dynamic binding. 12. You can use the expression obj instanceof class (object name instanceof class name) to test whether an object is an instance of a class. 13. You can use the Array List class to create an object to store a list of objects. 14. You can use the protected modifier to prevent methods and data from being accessed by non-subclasses in different packages. 15. You can use the final modifier to indicate that a class is final and cannot be inherited; it can also indicate that a method is final and cannot be overridden.
