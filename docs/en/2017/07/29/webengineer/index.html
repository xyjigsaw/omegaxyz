What is a front-end development engineer?

技术域ideasWeb开发程序人生webideas

Overview

Front-end engineer, also called Web front-end development engineer. It is an industry that has been segmented with the development of the web. Web front-end development technology mainly includes three elements: HTML, CSS and JavaScript! HTML is not even a language, just a simple markup language! CSS is just an untyped style modification language. Of course it can barely be considered a weakly typed language. The basics of Javascript are relatively easy, and it’s fairly quick to get started.The entry threshold for front-end development is very low. Compared with the learning curve of server-side languages, which is slow at first and then fast, the learning curve of front-end development is fast at first and then slow down. Because of this, there are many self-taught peers in the field of front-end development, but most of them are stuck at the stage of being able to use it, because the learning curve is getting steeper and steeper, and every step forward is difficult. People often say: A driver who doesn't want to be a tailor is not a good cook. If you simply learn front-end programming languages ​​without understanding back-end programming languages ​​(PHP, ASP.NET, JSP, Python), you cannot be considered an excellent front-end engineer. The road to becoming an excellent front-end engineer is full of sweat and hard work. With the popularization of the Web 2.0 concept and the promotion of the W3C organization, the influence of website reconstruction is growing at an alarming rate. XHTML+CSS layout, DHTML and Ajax are sweeping in like a whirlwind. IT companies of all sizes, including Sina, Sohu, NetEase, Tencent, Taobao, etc., have reconstructed their websites. The purpose of website reconstruction is not to make web pages more compliant with Web standards. The essence of reconstruction is to build a front-end flexible MVC framework, that is, HTML serves as the information model (Model), CSS controls the style (View), and JavaScript is responsible for scheduling data and implementing certain display logic (Controller). At the same time, the code needs to have good reusability and maintainability. This is the basis for high efficiency, high quality development and collaborative development. DHTML can make users' operations more dazzling and eye-catching; Ajax can realize data exchange without refreshing, making users' operations smoother. For ordinary users, there is no obvious difference between whether a website is standardized and powerful, and whether the server uses a powerful combination of J2EE+Oracle or a simple combination of ASP+Access. However, the front-end user experience gives users an intuitive impression. As people's requirements for user experience are getting higher and higher, front-end development is becoming more and more technically difficult. The profession of Web front-end development engineer has finally become independent from the situation where design and production are inseparable.

three elements

Web front-end development technology includes three elements: HTML, CSS and JavaScript, but with the popularity and popularity of RIA, Flash/Flex, Silverlight, XML and server-side languages ​​​​should also be mastered by front-end development engineers. Web front-end development engineers must communicate not only with upstream interaction designers, visual designers, and product managers, but also with downstream server-side engineers. They need to master many skills. This puts forward requirements for Web front-end development engineers in terms of breadth of knowledge. If you want to be good at front-end development, you may need to master ten lines first. However, all-rounders are rare. Therefore, for less important knowledge, we only need to "pass" it. But to what extent is "tong" enough? For many junior front-end development engineers, this question is very confusing. The Web is developing very fast, and it can even be said that these technologies are changing almost every day! If you do not have the ability to learn quickly, you will not be able to keep up with the pace of Web development. Front-end engineers must constantly improve themselves and constantly learn new technologies and new models; relying solely on today's knowledge cannot adapt to the future. The Web of tomorrow will be very different from today, and it is the job of front-end engineers to figure out how to reflect this earth-shaking change through their own Web applications. The entry threshold for front-end development is actually very low. Compared with the learning curve of server-side languages, which is slow at first and then fast, the learning curve of front-end development is fast at first and then slow down. Therefore, for those who work in IT, front-end development is a good entry point. Because of this, there are many self-taught peers in the field of front-end development, but most of them are stuck at the stage of being able to use it, because the learning curve is getting steeper and steeper, and every step forward is difficult. On the other hand, as mentioned before, front-end development is a very new profession, and research on some specifications and best practices is in the exploratory stage. There are always new inspirations and technologies emerging from time to time, such as CSS sprites, floating positioning, negative margin layout, grid layout, etc.; various JavaScript frameworks emerge in endlessly, injecting great vitality into the entire front-end development field; the browser war is becoming increasingly fierce, and cross-browser compatibility solutions are still diverse. In order to meet the needs of "high maintainability", it is necessary to master front-end knowledge more deeply and systematically, so that it is possible to create a good front-end architecture and ensure the quality of the code.

Master the technology

 
  1. Mastering various image retouching software, such as PS\Fireworks\AI, does not require professional designer level, but you must be able to use it and modify it. Today is an era of multi-purpose skills, and you will not be able to get a job just by knowing programming;
  2. Proficient in JavaScript, jQuery, ExtJS and other page effect frameworks, this is the core skill of front-end engineers and is a technology specializing in page effects. In fact, if you are proficient in JavaScript, back-end programming languages ​​such as .NET, PHP, and Ruby are easy to master because the principles are the same;
  3. Be proficient in one or two template languages ​​and be able to modify them, such as Dreamweaver or WordPress templates;
  4. Master thinkphp and zend framework, and be able to modify such open source source code;
  5. Master HTML5 and sencha, which are page front-end technologies for mobile devices such as mobile phones;
  6. Master one or two back-end programming languages, such as PHP, .NET, and JAVA, and be able to read the back-end source code. In fact, its principles are similar to JS;
  7. Master node.js and be familiar with various automated deployment tools;
  8. The front-end attaches great importance to user experience. It does not depend on the page provided by the artist. That is not a qualified front-end.
