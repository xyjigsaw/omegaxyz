Android Studio program cannot be loaded into the virtual machine solution

TechnologyAPP开发Android
After installing Android studio, create a project (hello world) with the specific description: "Waiting for target device to come online". That is "wait for the target device to come online".   Problem solving: ①: Command line (win+r–cmd) locates in the android_sdk/platform-tools/ directory and executes “adb kill-server” ②: Go to SDK tools > SDK Tools Check Android Emulator and click Apply③If the above two methods don’t work, try this method: it is definitely because the virtual machine does not have enough permissions and has not turned on the developer USB debugging mode (the picture is based on Android7.0) First enter the setting interfaceThen select About Phone -> Version Number, click the version number 7 times in a row (a trick in Android) and the system will remind you to enter developer mode.Then the developer options appear in the settings interface and enter the USB debugging mode. Just run the program
