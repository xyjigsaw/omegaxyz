Assembly directives

Hypocrisy instruction: a pseudo-direction that tells the compiler how to perform the compiler's command, which neither controls the operation of the machine nor is compiled into a machine code, can only be identified for the compiler and guides the compiler in translating the source program. The greatest difference between a pseudo-directive and an instruction is that a pseudo-director only works when compiled, whereas an instruction only works when the program is running. 1 EQU (Equate) General format: Mark: EQU Operations The command function is to assign the number of operations to the label, with the same value on both sides. Use the EQU false command to assign a label value, which is fixed throughout the source file. AREA: EQU 1000H; assigns the symbol AREA to 1000H 2. ORG (Origin) General format is: ORG xxxH (absolute address or symbol) XxxH determines the address of the first command (or data) after this statement. The source program or data in this paragraph is kept continuously at a subsequent address until the next ORG instruction. ORG 8000H; the target code has since been stored in the address space starting with 0 x 8,000h in the memory. ADD R1, #1 MOV R2, #2 3 DB (Define Byte) General format is: label: DB byte constant or character or expression Marked fields may be optional, and byte constants or characters refer to a byte data. The function of this false command is to store a byte constant or a string in the memory of a continuous address space. ORG 8000H DATA1: DB 43H, 09H, 08H DATA2: DB 07H The pseudo-director DB specified 43H,09H,08H sequenced to be stored in the storage unit starting at 8,000H, and 07H in DATA2 immediately adjacent to the address space of DATA1, i.e. 07H in 803H. Note: The DW (Definine Word) Directive definition is similar to that of DB, the difference being that DW defines a word and DB defines a byte. 4. END The general format is: An address or label can be ignored. This false instruction is used to indicate the end of the compilation language segment. Therefore, only one END in a source procedure is generally at the end of the procedure. If END is placed in the middle of the program, the statement after END will no longer be compiled. 5. AREA General format is: designation name{, attr}, attr}... The name of the section specified by the Section. Sections are designated stand-alone code or data modules that are indivisible and are processed by connectors. Attr is the attribute of one or more comma-separated sections. There are various attributes, such as: ALIGN=expression. By default, you have four-byte boundary alignment. Expression's range is 0-31. The aligned boundary is 2 expression square. CODE contains machine commands. Default value is READONLY DATA contains data but does not contain commands. Default value is READWRITE READONLY indicates that this section is read-only. Code area default. READWRITE indicates that this section is readable. Default value for the data range. ARA Express, DATA, READWRITE ;data The above code defines the readable/written data section called Example. 6. PUBLIC General format: PUBLC identifier 1, identifier 2 When more complex applications are developed, the generation of their implementation documents is typically linked to multiple target files. There is generally data interaction between multiple target files, and links between modules are addressed in the compilation language through false instructions PUBLIC and EXTRN. PUBLIC is used to describe the public identifiers in the current module that can be cited in other modules. PUBLIC _STARTUP Declaration Identifier_STARTUP can be cited in other modules. 7, EXTN or EXTERN General format: EXTRN identifier 1: Type 1, ID 2: Type 2 I don't know. Indicates that identifiers 1 and 2 are external identifiers that have been defined in other modules as types 1, 2, etc. In one module, several EXTRNs can be used to describe the external identifiers cited in this module. The substance provides the compiler with a name that is not defined in the current compilation document. Note: The identifier declared by EXTRIN must be declared as a public identifier in the module it defines and the type of identifier declared must be consistent with the type defined by the identifier. Similar: EXPORT or GLOBAL Declares a symbol with which a linker can interpret the symbol references in different objects and library files. that allows the code in other files to access the symbols in the current file. PROC The generic format is: < Process Name> PROC [Types] This directive defines the process directive. Process is a sub-program, and one process can be called by another (CALL Directive). < Process Name > PROC [Type] I'm sorry. I'm sorry. ENDP Note: PROC and ENDP must appear in pairs. 9. SPACE General format is: Mark SPACE expression This instruction is used to allocate a continuous storage area and initialize it to zero. expression is the number of bytes to be assigned. SPACE can be replaced by "%1". DataSpace SPACE 1000; allocates and initializes the 1,000-byte storage unit DataSpace. 10. Entry This instruction is used at the entry point of the declaration procedure. A program has at least one entry point. ARA TEST, CODE, READONLY ENTRY;the entry point for the program. 11, Preserve8 The generic format is: The PRESERVE8 command specifies that the current file needs to be stacked eight-bytes. Notify connectors by setting PRES8 compile properties. PRESERVE8; keeping code stacks eight bytes aligned. Preserve8{FALSE}; does not keep code stacks eight bytes aligned. 12, CODE 16, CODE 32 General format: CODE16 (or CODE32) CODE16 notifies the compiler that the subsequent command series is a 16-bit Thumb command. CODE32 notifies the compiler, followed by 32-bit ARM command. These two-day commands can be used for switching in the code of the applied mix of ARM and Thumb commands. NOTE: Instructions can only inform the compiler of the subsequent type of instruction and do not provide a state transition to the processor. 12. IMPORT This directive is similar to EXTERN. The label used to notify the compiler is defined in other source files. The label is added to the symbol table of the current source file, whether or not it is quoted in the current source file. The generic format is: IMPORT logo {[WEAK]} The label is case-sensitive in the program and [WEAK] indicates that the compiler will not display the wrong information when it is not found in all source files. ARA INIT, CODE, READONLY IMPORT Main; notifies the compiler that the current file refers to the symbol Main, but its definition is in other source files. END
