Pyinstaller usage tutorial

Programming LanguagesTechnologyPythonPythonwxpythonGUI
PyInstaller can be used to package python applications. The packaged program can be run on a machine that has a Python interpreter installed. PyInstaller supports Python 2.7 and Python 3.3+. It can be used on Windows, Mac OS

Install

linux or mac

you can start fromPyPiDownload and install it, of course you can also use pip or easy_install to install it. >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> pip install pyinstaller or easy_install pyinstaller >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> Version update >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> pip install --upgrade pyinstaller or easy_install --upgrade pyinstaller >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>

windows

Running PyInstaller on Windows also requires PyWin32 or pypiwin32, of which pypiwin32 will be installed automatically when you install PyInstaller.

Using PyInstaller

PyInstaller analyzes your python program and finds all dependencies. Then put the dependency files and python interpreter in a folder or an executable file.

Package into a folder

When packaging using PyInstaller, a folder is generated by default that contains all dependencies and executable files. The advantage of packaging into a folder is that when debugging, you can clearly see whether the dependencies are included. Another benefit is that when updating, you only need to update the executable file. Of course, the shortcomings are also obvious, inconvenient and difficult to manage. >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> pyinstaller script.py >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> So how does it work? The bootloader of PyInstaller is a binary executable program. When the user starts your program, the PyInstaller bootloader starts running, first creating a temporary Python environment, and then importing the program's dependencies through the Python interpreter. Of course, they are all in the same folder.

package into a file

We can use the onefile parameter to package all files into one executable file. >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> pyinstaller --onefile script.py >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>   Packing into a file is easier to manage than a folder. The disadvantage is that it runs relatively slowly. This file contains a compressed copy of the dependency files (.so files). When the program runs, PyInstaller's bootstrap program creates a new temporary folder. Then unzip the program's third-party dependency files into a temporary folder. This is why an executable file takes longer to execute than a folder. The rest is the same as above.

spec file

When you execute the following command >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> pyinstaller options..script.py >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> PyInstaller first creates a sepc (specification) file: script.spec. The storage address of this file can be defined using the parameter –specpath=, and it is placed in the current folder by default. What is the function of spec file? It will tell PyInstaller how to process your py file. It will encode your py file name and most of the parameters entered. PyInstaller generates the app by executing the contents of the spec file, a bit like a makefile. We do not need to manage the spec file in normal use, but the spec file needs to be modified in the following situations:
  • Resource files need to be packaged
  • Need to include some run-time libraries that PyInstaller does not know
  • Add run-time options to executable files
  • Multi-program packaging
You can generate the spec file by running the following command >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> pyi-makespec options script.py [other scripts ...] >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> After modifying the spec file, you can generate the app file through the following command >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> pyinstaller options script.spec >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> When generating app files through spec files, only the following parameters are useful:
  • –upx-dir=
  • –distpath=
  • –noconfirm=
  • –ascii

spec file parsing

Below is an example of a spec file. >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> block_cipher = None a = Analysis(['minimal.py'], pathex=['/Developer/PItests/minimal'], binaries=None, datas=None, hiddenimports=[], hookspath=None, runtime_hooks=None, excludes=None, ciper=block_cipher) pyz = PYZ(a.pure, a.zipped_data, cipher=block_cipher) exe = EXE(pyz, ...) coll = COLLECT(...) >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> The spec file mainly contains 4 classes: Analysis, PYZ, EXE and COLLECT.
  • Analysis takes a py file as input, it will analyze the dependent modules of the py file and generate corresponding information
  • PYZ is a .pyz compressed package that contains all the dependencies needed to run the program.
  • EXE is generated based on the above two items
  • COLLECT generates the output folder of other parts. COLLECT can also be omitted.

Modify spec file

As we said above, sometimes the spec file automatically generated by PyInstaller cannot meet our needs. The most common situation is that our program relies on some of our local data files. At this time, we need to edit the spec file ourselves to add data files. The data in Analysis in the above spec file analysis are the data files to be added to the project. We can edit the data. >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> a = Analysis( ... datas = [('you/source/file/path','file_name_in_project'), ('source/file2', 'file_name2')] ... ) >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> Datas can be thought of as a List, and each element is a tuple. The first element of the tuple is your local file index, and the second element is the file name after copying it to the project. In addition to the above writing method, you can also put it forward. >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> added_files = [...] a = Analysis( ... datas = added_files, ... ) >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> The method for adding other binary files is similar.

Summary

Finally, to put it simply, we need to generate an executable file through PyInstaller mainly in the following two steps. >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> pyinstaller [option] mypython.py >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> The option is empty to generate a folder, select onefile to generate a file. If the project has some dependent data files, the binary file generated above cannot be run. At this time, you can modify the spec file and then use pyinstaller to run the spec file. >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> pyinstaller [option] mypython.spec >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> Of course, generate the spec file first as above.
