Why ++i is better and faster than i++

Programming LanguagesC & C++TechnologyC++C
The increment (++) and decrement (--) operators provide a concise written form for incrementing and decrementing objects by one. In addition, these two operators can also be used on iterators, because many iterators do not natively support arithmetic operations, so at this time increment and decrement are necessary in addition to the introduction.     The increment and decrement operators have two forms: prefix form and postfix form, corresponding to ++i and i++ respectively. The prefix form of the operator adds 1 (or subtracts 1) to the object, and then uses the evaluated object as the result, while the postfix form also adds 1 (or subtracts 1) to the object, but the evaluation result is a copy of the value of the object that has not been operated on.   int i=0，j；    j=++i;                    //   i=1，j=1    j=i++;                   //   i=1, j=0 Both operations must operate on an lvalue operand. The prefix form returns the operand itself as an lvalue, and the postfix form returns a copy of the operand as an rvalue.     When writing code, try to avoid postfix forms unless necessary, because:The prefix form avoids unnecessary work. It adds 1 (or subtracts 1) to the value and directly returns the changed operand, while the postfix form needs to store the original value in order to return the unmodified original value. If we don't need the unmodified original value, then the postfix form is a waste.      For positive number and pointer types, the compiler may have performed certain optimizations on this extra work, but for relatively complex iterators, this extra work is very costly. Therefore, if you develop a good habit of preforming, you not only don’t have to worry about performance issues, but the code you write also conforms to the original intention of programming.
