Kruskal greedy algorithm of minimum spanning tree

Programming LanguagesC & C++Data StructuresTechnology
Basic idea:(1) Construct a subgraph containing only n vertices and an empty edge set. If each vertex in the graph is regarded as the root node of a tree, then it is a forest containing n trees. (2) Select an edge with the smallest weight from the edge set E of the network. If the two vertices of this edge belong to different trees, add it to the subgraph. In other words, the two trees where the two vertices are located are combined into one tree; on the contrary, if the two vertices of the edge have fallen on the same tree, it is not advisable. Instead, you should take the edge with the smallest weight and try again (3) and so on, until there is only one tree in the forest, that is, the subgraph contains n-1 edges.Vernacular:(1) Remove all edges in the picture. (2) Add edges to the graph in order from small to large weights to ensure that no loops are formed during the addition process. (3) Repeat the previous step until all vertices are connected. At this time, a minimum spanning tree is generated. This is a greedy strategy.difficulty:Determine whether the addition of an edge <u, v> will form a cycle in the selected edge set. Solution: Use union search to find the root nodes of the tree where the two vertices u and v are located. If the root nodes are the same, it means that u and v are in the same tree, and then u and v will form a ring when connected; if the root nodes are different, then u and v are not in the same tree, and they will not form a ring when connected, but the two trees will be merged.Time complexity of Kruskal's algorithmis O(eloge) (e is the number of edges in the network), so compared to Prim's algorithm, it is suitable for finding the minimum spanning tree of a network with sparse edges. Kruskal's algorithm finds the minimum spanning tree of the network from another approach. Assuming a connected network N = (V, {E}), let the initial state of the minimum spanning tree be a non-connected graph T = (V, {∮}) with only n vertices and no edges. Each vertex in the graph forms a connected component. Select the edge with the smallest cost in E. If the vertex attached to the edge falls on a different connected component in T, add this edge to T. Otherwise, discard this edge and select the next edge with the smallest cost. And so on, until all vertices in T are on the same connected component. Code:
