"__name__" attribute for Python scripts

Programming LanguagesTechnologyPythonPython
Every Python script has a "__name__" attribute when it is run. If the script is imported as a module, the value of its "__name__" attribute is automatically set to the module name; if the script is run independently, the value of its "__name__" attribute is automatically set to "__main__". For example, assume that the file nametest.py contains only the following line of code: print(__name__) When running the program directly in IDLE, or when running the program file in a command line prompt environment, the results are as follows: __main__ When importing this file as a module, the following execution results are obtained: >>> import nametest Nametest   Use the "__name__" attribute to control how a Python program runs. For example, if you write a module that contains a large number of functions that can be exploited by other programs, but do not want the module to be run directly, you can add the following code to the program file: if __name__ == '__main__': print('Please use me as a module.') In this way, when the program is executed directly, you will get the prompt "Please use me as a module.", and after using the import statement to import it as a module, you can use its classes, methods, constants or other members.Attached:Write your own packages and modulesEach directory of a package must contain an __init__.py file, which can be an empty file and is used only to indicate that the directory is a package. The main purpose of the __init__.py file is to set the __all__ variable and contain the code required for package initialization. The objects defined in the __all__ variable can all be imported correctly when using from...import *.
