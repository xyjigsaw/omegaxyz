Python shallow copy and deep copy

Programming LanguagesTechnologyPythonPython
In Python, there are differences between object assignment and copying (deep/shallow copy). If you donвҖҷt pay attention when using them, unexpected results may occur. Note: The difference between shallow copy and deep copy is only for composite objects. The so-called composite objects are objects that contain other objects, such as lists and class instances. For numbers, strings, and other "atomic" types, there is no copy, and all generated are references to the original objects. copy will not completely copy the sub-objects of a complex object. What are the sub-objects of a complex object? For example, nested sequences in sequences, nested sequences in dictionaries, etc. are all sub-objects of complex objects. For sub-objects, Python will store it as a public image, and all copies of it will be treated as a reference, so when one of the references changes the image and another reference uses the image, the image has been changed.So look at origin[2] here, which is the list [3, 4]. According to the definition of shallow copy, cop1[2] points to the same list [3, 4]. So, if we change this list here, it will cause origin and cop1 to change at the same time. This is why after origin[2][0] = вҖңhey!вҖқ above, cop1 also becomes [1, 2, [вҖҳhey!вҖҷ, 4]]. В  During deepcopy, each layer of the complex object will be copied into a separate individual.At this time, although the values вҖӢвҖӢof origin[2] and cop2[2] are equal to [3, 4], they are no longer the same list. That is copying in our ordinary sense.
