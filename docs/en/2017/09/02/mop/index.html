Detailed explanation of multi-objective optimization problems

TechnologyMachine LearningEvolutionary Algorithms多目标优化算法PSO算法
In life, many problems are composed of multiple goals that conflict and influence each other. People often encounter the optimization problem of making multiple objectives as optimal as possible in a given area at the same time, which is a multi-objective optimization problem. If an optimization problem has more than one optimization objective and needs to be processed simultaneously, it becomes a multi-objective optimization problem. Multi-objective optimization problems are very common and occupy a very important position in real life such as engineering applications. These practical problems are usually very complex and difficult and are one of the main research fields. Since the early 1960s, multi-objective optimization problems have attracted the attention of more and more researchers with different backgrounds. Therefore, solving multi-objective optimization problems has very important scientific research value and practical significance. In practice, most optimization problems are multi-objective optimization problems. In general, the various sub-objectives of multi-objective optimization problems are contradictory. The improvement of one sub-objective may cause the performance of another or several other sub-objectives to decrease. That is, it is impossible to achieve the optimal value of multiple sub-objectives at the same time. Instead, coordination and compromise can only be carried out among them to make each sub-objective as optimized as possible. The essential difference from the single-objective optimization problem is that its solution is not unique, but there is a set of optimal solutions composed of many Pareto optimal solutions. Each element in the set is called a Pareto optimal solution or a non-inferior optimal solution.1. Description of multi-objective optimization problems The multi-objective optimization problem is described in words as an optimization problem composed of D decision variable parameters, N objective functions, and m + n constraints. The decision variables, objective functions, and constraints are functional relationships. In the non-inferior solution set, the decision maker can only choose a satisfactory non-inferior solution as the final solution according to the specific problem requirements. The mathematical form of the multi-objective optimization problem can be described as follows [1]: min　y = f( x) = [ f1 ( x) , f2 ( x) , …, fn ( x) ] n = 1, 2, …, N s. t. gi ( x) ≤0 i = 1, 2, …, m hj ( x) = 0 j = 1, 2, …, k x = [ x1 , x2 , …, xd , …, xD ] xd_min ≤xd ≤xd_max　d = 1, 2, …, D Among them: x is the D-dimensional decision vector, y is the objective vector, N is the total number of optimization objectives; gi (x) ≤0 is the i-th inequality constraint, hj (x) = 0 is the j-th equality constraint, fn (x) is the nth objective function; X is the decision space formed by the decision vector, and Y is the target space formed by the objective vector. gi (x) ≤0 and hj (x) = 0 determine the feasible domain of solution, xd_max and xd_min are the upper and lower limits of each dimension vector search.The optimal solution or non-inferior optimal solution in a multi-objective optimization problem can be defined as follows: Definition 1 The dominance relationship of f(x) is consistent with the dominance relationship of x. Definition 2: Pareto optimal solution is not dominated by any solution in the feasible solution set Solution, if x3 is a point in the search space, it is said that x3 is a non-inferior optimal solution if and only if there is no x (in the feasibility domain of the search space) such that fn (x) ≤fn (x3) holds, n = 1, 2, …, N. Definition 3: Given a multi-objective optimization problem f(x), f(x3) is the global optimal solution if and only if for any x (in the search space), f(x3) ≤f(x). Definition 4: The set consisting of all non-inferior optimal solutions is called the optimal solution set (Pareto op timal set) of a multi-objective optimization problem, also known as the acceptable solution set or the effective solution set.Application of different algorithms in multi-objective optimizationThere is no unique global optimal solution to multi-objective optimization problems, and too many non-inferior solutions cannot be directly applied, so when solving, one must find a final solution. There are three main methods for finding the final solution: a) Generative method, that is, first find a large number of non-inferior solutions to form a subset of non-inferior solutions, and then find the final solution according to the intention of the decision-maker; b) It is an interactive method, which does not first find many non-inferior solutions, but gradually finds the final solution through dialogue between the analyst and the decision-maker; c) requires the decision maker to provide the relative importance between goals in advance. Based on this, the algorithm converts the multi-objective problem into a single-objective problem for solution. These are mainly achieved through algorithms. Many experts and scholars have been using different algorithms to solve multi-objective optimization problems, such as multi-objective evolutionary algorithms, multi-objective particle swarm algorithms, ant colony algorithms, simulated annealing algorithms and artificial immune systems.multi-objective evolutionary algorithm Multi-objective evolutionary algorithm (MOEA) is a type of global probabilistic optimization search method formed by simulating biological evolution mechanisms. It began to develop rapidly in the mid-1990s, and its development can be divided into two stages. There are two main methods in the first stage, namely methods not based on Pareto optimization and methods based on Pareto optimization; in the second stage, the concept of external set is proposed on this basis. The external set stores all non-dominated individuals of the current generation, so that the solution set maintains a good distribution degree. The multi-objective evolutionary algorithms proposed during this period placed more emphasis on the efficiency and effectiveness of the algorithm. In these two stages, the more typical multi-objective evolutionary algorithms include NS2 GA2[3], PESA2 and SPEA2. For these three algorithms, they have many advantages but their shortcomings are also obvious. For example, NSGA2 has the advantages of high operating efficiency and good distribution of solution sets, especially for low-dimensional optimization problems. Its disadvantage is that the solution set process has defects in high-dimensional problems and the diversity of solution sets is not ideal. The advantage of PESA2 is that its solution has good convergence and is relatively easy to approach the optimal surface, especially in high-dimensional problems; but its disadvantage is that the selection operation can only select one individual at a time, which consumes a lot of time, and the class diversity is not good. The advantage of SPEA2 is that it can obtain a well-distributed solution set, especially when solving high-dimensional problems. However, its clustering process takes a long time to maintain diversity and its operating efficiency is not high. The basic principle of the multi-objective evolutionary algorithm is described as follows: The multi-objective evolutionary algorithm starts from a group of randomly generated populations and performs evolutionary operations such as selection, crossover and mutation on the population. After multiple generations of evolution, the fitness of individuals in the population continues to improve. Thereby gradually approaching the Pareto optimal solution set of the multi-objective optimization problem. Different from single-objective evolutionary algorithms, multi-objective evolutionary algorithms have a special fitness evaluation mechanism. In order to give full play to the group search advantages of evolutionary algorithms, most MOEAs adopt a fitness evaluation method based on Pareto ranking. In practical applications, in order to make the algorithm better converge to the Pareto optimal solution of the multi-objective optimization problem, existing MOEA usually also adopts key technologies such as elite strategy, niche and setting external sets. The algorithm idea described in the general framework of MOEA is as follows: MOEA generates the next generation population X (t + 1) by performing operations such as selection, crossover and mutation on the population X (t). In the evolution process of each generation, all non-inferior solution individuals in the population At the end of evolution, the external set The non-inferior solution individuals in the solution are output as the optimal solution. At present, MOEA research has achieved a lot of results and has been applied to many fields, such as engineering, industrial and scientific fields. Among them, the engineering field has the most applications, such as electronic engineering, water conservancy engineering, wind power engineering and control, etc.Multi-objective particle swarm algorithm The particle swarm optimization algorithm (PSO) is an evolutionary computing technology invented from the study of the predatory behavior of bird flocks. It was first proposed by Dr. Barnhart and Dr. Kennedy in 1995 [7]. It is an optimization tool based on iteration. The system initializes a set of random solutions and searches for the optimal value through iteration. It not only has global optimization capabilities, but also has strong local optimization capabilities. In the basic particle swarm algorithm [8, 9], the particle swarm consists of n particles, and the position xi of each particle represents the potential solution of the optimization problem in the D-dimensional search space. Particles fly at a certain speed in the search space. This speed dynamically adjusts the next flight direction and distance based on its own flight experience and the flight experience of its companions. All particles have a fitness value determined by the objective function, and know the best position they have found so far (individual extreme value pi) and the current position (xi). In addition, each particle also knows the best position found by all particles in the entire group so far (the global extreme value pg), which is the optimal value among all the best positions. The mathematical description of the particle swarm algorithm is as follows: each particle i contains a D-dimensional position vector xi = (xi1, xi2,..., xiD) and velocity vector vi = (vi1, vi2,..., viD). When particle i searches the solution space, it saves its searched optimal experience position pi = (pi1, pi2,..., piD). At the beginning of each iteration, the particle adjusts its velocity vector to adjust its position based on its own inertia and experience and the group's optimal experience position pg = (pg1, pg2, ..., pgD). c1 and c2 are positive constants, called acceleration factors; r1 and r2 are uniformly distributed random numbers in [0, 1], d is the dimension in D dimension; ω is the inertia weight factor. Because the particle swarm algorithm has efficient search capabilities, it is conducive to obtaining the optimal solution in a multi-objective sense; by representing the entire solution cluster group, multiple non-inferior solutions are searched simultaneously in a parallel manner, that is, multiple Pareto optimal solutions are searched; at the same time, the particle swarm algorithm is relatively versatile, suitable for processing multiple types of objective functions and constraints, and can be easily combined with traditional optimization methods, thereby improving its own limitations and solving problems more efficiently. Therefore, applying particle swarm algorithm to solve multi-objective optimization problems has great advantages. The idea of ​​particle swarm algorithm is described as follows: after initializing the population, the size of the population is recorded as N. Based on the idea of ​​fitness dominance, the population is divided into two subgroups, one is called the non-dominated subset A and the other is called the dominated subset B. The cardinal numbers of the two subsets are n1 and n2 respectively, and the sum of the cardinal numbers of the two subgroups is N [13]. The external elite set is used to store the non-inferior solution subset A generated in each generation. Each iteration process only updates the speed and position of the particles in B, and compares the updated particles in B with the particles in A based on the idea of fitness dominance. If xi ∈B, ϖ xj ∈A, such that If xi dominates xj, delete xj and add xi to A to update the external elite set; and the size of the elite set must be maintained within an upper limit using some technologies, such as density evaluation technology, dispersion technology, etc. Finally, the criterion for algorithm termination can be the maximum number of iterations Tmax, the calculation accuracy ε or the maximum number of stagnation steps Δt of the optimal solution, etc.
