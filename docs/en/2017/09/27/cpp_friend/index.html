Summary of friend functions in C++

高级语言C&C++技术域C++

Friend function purpose

By using the class mechanism, data hiding and encapsulation are realized. The data members of a class are generally defined as private members, and the member functions are generally defined as public, thereby providing a communication interface between the class and the outside world. However, sometimes you need to define some functions that are not part of the class but need to frequently access the data members of the class. In this case, these functions can be defined as friend functions of the function. In addition to friend functions, there are also friend classes, both of which are collectively called friends. The function of friends is to improve the running efficiency of the program (that is, to reduce the time overhead required for type checking and security checking), but it destroys the encapsulation and hiding properties of the class, allowing non-member functions to access the private members of the class.

usage

In fact, there are probably two situations where friend functions need to be used: ①Some situations where operator overloading requires the use of friends. ②When two classes want to share data.

Advantages and Disadvantages of Friend Functions

Advantages: Can improve efficiency, simple and clear expression. Disadvantages: Friend functions break the encapsulation mechanism. Try not to use member functions unless you have to use friend functions.

Use of friend functions

Parameters of friend function:

①Because the friend function does not have this pointer, the parameters must have three situations: ② When accessing non-static members, an object is required as a parameter; ③When accessing static members or global variables, no object is required as a parameter; ④If the object used as a parameter is a global object, the object does not need to be used as a parameter;

Friend function location

Because a friend function is a function outside the class, its declaration can be placed in the private segment or the public segment of the class without distinction.

Classification of friend functions:

Depending on the source of this function, it can be divided into three methods: ① Ordinary function friend function Purpose: To enable ordinary functions to access friends of a class Syntax: Declaration: friend + ordinary function declaration Implementation location: can be outside the class or within the class Implementation code: same as ordinary function Call: similar to ordinary functions, call directly Code:
class INTEGER
{
    friend void Print(const INTEGER& obj);//声明友元函数
};
void Print(const INTEGER& obj）
{
    //函数体
}
void main()
{
    INTEGER obj;
    Print(obj);//直接调用

}
②All member functions of class Y are friend functions of class X—friend classes Purpose: Use a single declaration to make all functions of class Y become friends of class X. It provides a way of cooperation between classes so that objects of class Y can have the functions of class X and class Y. Syntax: Declaration location: public or private, often written as private (think of the class as a variable) Statement: friend + class name (not an object) Code:
class girl;
class boy
{
    public:
    void disp(girl &);
};
void boy::disp(girl &x) 
//函数disp()为类boy的成员函数，也是类girl的友元函数
{
    cout<<"girl's name is:"<<x.name<<",age:"<<x.age<<endl;
    //借助友元，在boy的成员函数disp中，借助girl的对象，直接访问girl的私有变量
}

class girl
{
    private：
    char *name;
    int age;
    friend boy; //声明类boy是类girl的友元
};
The main function is the same as when called normally. ③A member function of class Y is a friend function of class X Purpose: Make a member function of class Y become a friend of class X. Specifically: in this member function of class Y, with the help of parameter Syntax: Declaration location: declared in public (itself a function) Declaration: friend + declaration of member function Call: First define the object y of Y---use y to call your own member function---use the friend mechanism in your own member function Code:
class girl;
class boy
{
    public:
    void disp(girl &);
};
void boy::disp(girl &x) 
//函数disp()为类boy的成员函数，也是类girl的友元函数
{
    cout<<"girl's name is:"<<x.name<<",age:"<<x.age<<endl;
//借助友元，在boy的成员函数disp中，借助girl的对象，直接访问girl的私有变量
}

class girl
{
    private：
    char *name;
    int age;
    friend void boy::disp(girl &); //声明类boy是类girl的友元
};
Summary: In fact, the overloaded implementation of some operators must be implemented outside the class, so usually in this case, it is necessary to declare it as a friend of the class.

The difference between friend functions and class member functions

① Member functions have this pointers, but friend functions do not have this pointers. ② Friend functions cannot be inherited, just like a father’s friends may not be his son’s friends.
