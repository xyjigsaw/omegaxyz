C++ constructors and destructors

Programming LanguagesC & C++TechnologyC++
The constructor is used to initialize variables. It is automatically called by the system. If no constructor is written, the system will generate a default constructor for it. Its call is automatically and implicitly called by the system when the variable is declared. The constructor provided by the C++ language for classes can automatically complete the object initialization task. The constructors of global objects and static objects are called before the main() function is executed. The constructors of local static objects are called when the program executes the corresponding statement for the first time. However, when a reference declaration of an external object is given, the corresponding constructor is not called, because the external object only refers to an object declared elsewhere and does not actually create an object. In essence, if you do not explicitly define a copy constructor in the class, the class will also define a default copy constructor to complete the copying of all members one by one. Speaking of which, let's introduce a very important concept - the default constructor. The default constructor is different from the default copy constructor. The default constructor has no parameters. If the user does not write a constructor, the compiler will generate a default constructor. This constructor does nothing. If a constructor is defined, no default constructor will be generated. Note that the default constructor is different from the default copy constructor. Although both are generated without defining a constructor, they are completely different functions. The default constructor is a parameterless function and does nothing, just an empty title. However, the default copy constructor needs to pass parameters. The formal parameters are class objects, and all members need to be copied one by one. Destructor: The class name is used as the function name, with ~ in front. The destructor does not allow overloading, and the destructor has no parameters. Normally, the object's destructor will be automatically called when the program ends, but what needs to be noted is the dynamic allocation of memory.
#include<iostream>
using namespace std;
unsigned time=0;
class ABC{
    int data;
    string name;
public:
    ABC(int d,const char *ch):data(d),name(ch)
    {
        time++;
        cout<<time<<"调用构造"<<endl;
    }
    ABC(ABC &a)
    {
        time++;
        data=a.data;
        name=a.name;
        cout<<time<<"Copy 构造"<<endl;
    }
    ~ABC(){time++; cout<<time<<"析构函数"<<endl;}
};

ABC func(ABC a)
{
    return a;
}

int main()
{
    ABC b(13,"XUYI");
    ABC a(12,"WHAT");
    b=func(a);
    cout<<"--------pause--------"<<endl;

    return 0;
}
Here, when calling function func, the incoming parameter is in the form of ABC a. In fact, the compiler will create a copy of object a at this time. That is, the copy constructor is called once, and when it returns, a copy object of the returned object is created, so the copy constructor is called again. So the destructor will be called twice when that function ends.
