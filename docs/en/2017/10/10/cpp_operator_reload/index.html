Detailed explanation of C++ operator overloading

高级语言C&C++技术域C++

Why do we need to overload operators?

The operands of operators predefined in C++ are limited to basic built-in data types, but there is no way to operate on our custom types (classes). But most of the time we need to perform similar operations on the types we define. At this time, we need to redefine such operators and give them new functions to meet our own needs.

The essence of C++ operator overloading

The essence of operator overloading is function overloading or function polymorphism. Operator overloading is a form of C++ polymorphism. The purpose is to enable people to use functions with the same name to complete different basic operations. To overload an operator, you need to use a special function form called an operator function. The operator function form is: operatorp (argument-list) // The 'p' after operator is the operator symbol to be overloaded.

Rules for operator overloading

(1) In order to prevent users from overloading operators on standard types, C++ stipulates that at least one of the operands of the overloaded operator must be a user-defined type What does this mean? For example, there are two numbers: int number1, int number2, Then number1+number2 finds the sum of two numbers, But if you overload it so that the sum of two numbers is their product, this is certainly illogical. There may be ambiguity after overloading, causing the program to not know which one to execute (the built-in function or the overloaded function) (2) The use of operators cannot violate the original syntax rules of the operators. If % cannot be overloaded as an operand, For example: int index; %index; this is not allowed (3) The original priority of the operator cannot be modified. (4) A new operator cannot be created. For example, operator** (···) cannot be defined to represent exponentiation. (5) Operators that cannot be overloaded: member operator, scope operator, conditional operator, sizeof operator, typeid (an RTTI operator), const_cast, dynamic_cast, reinterpret_cast, static_cast cast operator (6) Most operators can be overloaded through member functions and non-member functions, but the following four operators can only be overloaded through functions: = assignment operator, () function call operator, [ ] subscript operator, -> operator for accessing class members through pointers. (7) In addition to the above rules, we should also pay attention to following some sensible rules when overloading operators: For example: do not overload the + operator to exchange the values ​​of two objects. Two forms of overloaded operators: There are two ways to overload operators, namely: Overloaded as a member function of the class || Overloaded as a non-member function of the class. When overloaded as a non-member function of a class: Usually we declare it as a friend function, because most of the time the overloaded operator needs to access the private data of the class (of course it can also be set as a non-friend, non-class member function. However, the non-friend is not a member function of the class, so there is no way to directly access the private data of the class). If it is not declared as a friend function of the class, but accesses private data by calling the public function of the class in this function, it will reduce performance. Therefore, it is generally set as a friend function of the class, so that we can access the data in the class in this non-member function.

Can be used as an overloaded operator

Arithmetic operators: +,-,*,/,%,++,--; Bit operation operators: &,|,~,^,＜＜,＞＞ Logical operators: !,&&,||; Comparison operators: ＜,＞,＞=,＜=,==,!=; Assignment operators: =,+=,-=,*=,/=,%=,&=,|=,^=,＜＜=,＞＞=; Other operators: [], (), ->,, (comma operator), new, delete, new [], delete [], -> *. The following operators do not allow overloading: .,.*,::,?:

Detailed code explanation

(Basic operators and << operators are overloaded, the latter is to output the value of class members. The four arithmetic operators provide two overloads, and the friend function overloads are commented out)
#include<iostream>
using namespace std;
class A
{
public:
    friend ostream& operator<<(ostream& os, const A& app)
    {
        os<<app.data;
        return os;
    }//次函数用来重载<<运算符使其输出A类的值

    A(int d):data(d){}
    A operator+(A&);
    A operator-(A&);
    A operator*(A&);
    A operator/(A&);
    A operator%(A&);

   /* friend A operator+(A&,A&);
    friend A operator-(A&,A&);
    friend A operator*(A&,A&);
    friend A operator/(A&,A&);
    friend A operator%(A&,A&);*/

private:
    int data;
};


//作为成员函数形式进行重载
A A::operator+(A &a)
{
    return A(data+a.data);
}

A A::operator-(A &a)
{
    return A(data-a.data);
}

A A::operator*(A &a)
{
    return A(data*a.data);
}

A A::operator/(A &a)
{
    return A(data/a.data);
}

A A::operator%(A &a)
{
    return A(data%a.data);
}
