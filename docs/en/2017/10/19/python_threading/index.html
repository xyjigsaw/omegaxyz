Python multithreading (threading module)

Programming LanguagesTechnologyPythonPython多线程
CPUs are getting faster and faster. We can't satisfy Python's need to do only one thing at the same time. It needs to be half-hearted. Multitasking (mutiltasking) requires the use of multi-threading to let the program do many things at the same time. The following is a brief introduction to the threading module. Thread is one of the most important classes in the threading module, which can be used to create threads. There are two ways to create a thread: one is to inherit the Thread class and override its run method; the other is to create a threading.Thread object and pass the callable object as a parameter in its initialization function (__init__). The following code creates a threading.Thread object, which is enough for beginners.
import threading
from time import ctime, sleep


def event1(func):
    for i in range(2):
        print("时间：%s 我正在学习 %s。" % (ctime(), func))
        sleep(2)


def event2(func):
    for i in range(2):
        print("时间：%s 我正在听%s。" % (ctime(), func))
        sleep(3)


threads = []
t1 = threading.Thread(target=event1,args=('Python',))
threads.append(t1)
t2 = threading.Thread(target=event2,args=('Shape of you',))
threads.append(t2)

if __name__ == '__main__':
    for t in threads:
        t.setDaemon(True)
        t.start()
    print("all over %s" % ctime())
The time to see results is the same.   Initialization function prototype of threading.Thread class: def __init__(self, group=None, target=None, name=None, args=(), kwargs={})
  • The parameter group is reserved for future expansion;
  • The parameter target is a callable object (also called activity), which is executed after the thread starts;
  • The parameter name is the name of the thread. The default value is "Thread-N", where N is a number.
  • The parameters args and kwargs respectively represent the parameter list and keyword parameters when calling target.
The Thread class also defines the following commonly used methods and properties:Thread.getName() Thread.setName() Thread.nameUsed to get and set the name of the thread.Thread.identGet the thread's identifier. The thread identifier is a non-zero integer. This property is only valid after the start() method is called, otherwise it just returns None.Thread.is_alive() Thread.isAlive()Determine whether the thread is active (alive). The thread is activated from the time the start() method is called to start the thread until the run() method completes execution or is interrupted due to an unhandled exception.Thread.join([timeout])Calling Thread.join will block the calling thread until the called thread finishes running or times out. The parameter timeout is a numeric type, indicating the timeout time. If this parameter is not provided, the calling thread will be blocked until the called thread ends.threading.Timerthreading.Timer is a subclass of threading.Thread that can perform an operation after a specified time interval.threading.EventEvent implements similar functions to Condition, but is simpler than Condition. It maintains internal identifiers to achieve synchronization between threads. (Threading.Event and the System.Threading.ManualResetEvent class in .NET implement the same functionality.)Event.wait([timeout])Block the thread until the internal flag bit of the Event object is set to True or times out (if the parameter timeout is provided).Event.set()Set flag bit to TrueEvent.clear()Set the flag companion to False.Event.isSet()Determine whether the flag bit is True.threading.active_count() threading.activeCount()Get the number of currently active (alive) threads.threading.current_thread() threading.currentThread()Get the current thread object (Thread object).threading.enumerate()Get a list of all currently active threads.threading.settrace(func)Set a trace function to be called before run() is executed.threading.setprofile(func)Set a tracking function to be called after run() completes execution.
