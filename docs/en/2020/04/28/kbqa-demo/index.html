Demo of question and answer system based on knowledge graph

Programming LanguagesTechnologyMachine LearningPythonKnowledge GraphsNatural Language ProcessingPythonGithubNatural Language ProcessingKnowledge Graphs

Introduction

A question answering system based on knowledge graph, namely KBQA. One of the simple implementation methods is to convert the natural language questions input by the user into relational queries in the graph database, and finally present the entities and relationships in the database to the user. A demo of KBQA is given below.

Implement dependencies

  • Python 3.6+
  • Apache Jena Fuseki
  • SPARQLWrapper
  • refo
  • jiaba

Instance data

The topic of this knowledge graph is COVID-2019 knowledge query. Triplet data comes fromOpenKGCOVID-19 data set.

Implementation process

  1. Data extraction: Download the json-ld data provided by OpenKG, and convert the json-ld data into RDF format data through a script. The RDF data has been uploaded to this instance and can be used directly by users.
  2. Data Storage: AdoptApache Jena Fuseki, Apache Jena Fuseki is a SPARQL service that supports multiple operating systems. RDF data can be stored and relationships in the database can be queried through SPARQL query statements. Here, the RDF data given above is imported into Jena.
  3. Rule writing: Python writes database query statements and rules. The rules use refo, which can be downloaded directly from Python. This step is the key to converting natural language into structured rules.
  4. Data query: The SPARQL query statement generated by refo is passed to Jena, and the relevant entity relationships can be found, and finally fed back to the user.
It should be noted that this code was implemented in a hurry, the rules written have less content, and the output is not standardized.
