Ajax and jQuery load data asynchronously

Programming LanguagesTechnologyWebJavaScript&HtmlPythonWebJavaScriptPythonDjangoAjax

Introduction

It is unrealistic to read data from the server database and transfer it to the front-end page at once. On the one hand, it will increase the pressure on the server, and on the other hand, the customer's bandwidth resources will also be occupied. Ajax can just solve the problem of asynchronous loading of data. Ajax is "Asynchronous Javascript And XML" (asynchronous JavaScript and XML) refers to a web development technology that creates interactive, fast and dynamic web applications that can update parts of a web page without reloading the entire web page. Since it is relatively simple to implement ajax with jQuery, the following code references the jQuery library to implement Ajax, and uses Django as the framework. jQuery can be manually downloaded and placed in a local folder, or you can quote the following statement.
<script src="http://apps.bdimg.com/libs/jquery/1.11.1/jquery.min.js"></script>

HTML statement

The div with category demo will display Ajax data$(document).ready(function(){})It means that Ajax data will be loaded after other elements of the page are loaded. At this time, the browser will not have a loading bar or circle. Of course, you can also click on an element to load it, for example$('#demo').click(function(){})。 $.getJSON('/ajax_server/',function(ret)Refers to reading JSON data from the function ajax_server in Django's view.py, and the data passes$('#demo').append(ret)Displayed in a div. Can also be useddocument.getElementById("demo").innerHTML = ret;to display data.

defect

The main criticism of using Ajax is that it can break the browser's back and bookmark functionality. In the case of dynamically updated pages, the user cannot go back to the previous page state because the browser can only remember static pages in the history. The possible differences between a page that has been fully read and a page that has been dynamically modified are very subtle; users often want to be able to cancel their previous action by clicking the back button, but in an Ajax application, this is not possible. However, developers have come up with various ways to solve this problem. Most of the methods before HTML5 were to create or use a hidden IFRAME to reproduce the changes on the page when the user clicks the back button to access the history. (For example, when the user clicks back in Google Maps, it searches in a hidden IFRAME and then reflects the search results onto an Ajax element to restore the application state to what it was at that time).
