Terminal pip and pip3 configuration under Mac (soft link)

Programming LanguagesTechnologyPython操作系统Pythonmac

origin

The Python environment on Mac today is an absolute asshole. 1. The system comes with Python2.7 2. Download a 3.6 version from my official website 3. homebrew quietly downloaded 3.x 4. anaconda comes with a 3.x 5. I updated the Xcode command line tool the day before yesterday and secretly downloaded version 3.7 for me. By the way, I wrote a certain soft connection variable to the system disk and the installed package to the data disk.The MacOS Catalina file system is really awkward and definitely not as easy to use as Windows.In order to correctly point to the required Python when the terminal runs the pip command, the soft link needs to be reconfigured.

Mac two bin directories

Similarities

/usr/bin and /usr/local/bin are both soft links used to store terminal command binaries or commands. These two bin directories are directories that are already included in the environment variables. Programs placed in them or commands linked to them can be executed directly in the terminal.

Differences

Mac's /usr/bin directory does not allow adding or deleting files. To add or delete files in /usr/local/bin, you only need administrator rights to run them directly in the terminal.Note that the leading "/" cannot be missing when searching for directories.

Mac Terminal User Configurable Files

Configurable files are divided into two types according to the terminal type. These files are hidden and have the same syntax structure. They can be used to configure environment variables, etc. and require "Command+Shift+." to display them.bash terminal:/Users/yourusername/.bash_profilezsh terminal:/Users/yourusername/.zsh_profile /Users/yourusername/.zshrc

view location command

which pip and which pip3:Check the startup path of python's pip package management tool (the location of the soft link), which is usually between the two bin directories mentioned above.pip --version and pip3 --version:Used to display the real address storage location of the command

Example

The following takes pip3 as an example. The specific operations for pip3 in zsh are:In the same way, you need to change the 2.7 version of pip in the terminal to the pip version you downloaded, and directly change all the following pip3 to pipAll commands need to be modified according to your own Python version and real location① First, you need to ensure that the environment variable position of /usr/local/bin is in front of /usr/bin, so that the data of /usr/local/bin can be read first, because the former data can be changed.Execute under zsh terminal:
echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.zshrc
Note that you don’t need to do this step yet. If permission denied or command not find appears, it means you have encountered /usr/bin. Then perform the first step.②Find the real location of pip3Generally speaking, the pip of python 3.x you download is in /Library/Frameworks/Python.framework/Versions/3.x/lib/python3.x/site-packages/pip (python 3.x)③Delete existing redundant data 
rm -rf /usr/local/bin/pip3
④Create the soft link of pip3 in /usr/local/bin/ to the real location of pip3 above. 
ln -s /Library/Frameworks/Python.framework/Versions/3.x/bin/pip /usr/local/bin/pip3
At this time, entering pip3 on the command line will automatically point to the real location of your Python version.⑤Verification 
pip3 --version
My terminal shows: pip 19.0.3 from /Library/Frameworks/Python.framework/Versions/3.6/lib/python3.6/site-packages/pip (python 3.6)
which pip3
My terminal shows: /usr/local/bin/pip3
