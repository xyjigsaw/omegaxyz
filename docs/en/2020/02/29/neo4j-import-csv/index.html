Neo4j-import imports CSV data

TechnologyDatabasesDatabasesNeo4j
Recently, data with hundreds of millions of relationships/nodes needs to be imported into Neo4j. There are several tools that can be used to import it:
  • Cypher CREATE statement, write a CREATE for each piece of data
  • Cypher LOAD CSV statement, convert the data into CSV format, and read the data through LOAD CSV.
  • Official Java API - Batch Inserter
  • Batch Import tool
  • neo4j-import tool (comes with Neo4j)
So, I chose the last option without hesitation. Neo4j-import available parameter commands:

Pitfalls encountered

1. This tool is generally placed in the bin folder of the Neo4j installation directory, or it may be in the hidden folder of the user on the C drive. Under the database file in Neo4jDesktop (relatively hidden) 2.The header needs to be written in advance in the CSV file., for example: The general form is name:type, separated by commas. There must be one name and type. ID is also a type, and its name can be ignored. The included types are: int, long, float, double, boolean, byte, short, char, string, point, date, localtime, time, localdatetime, datetime and duration to specify the data type of the attribute. If no data type is provided, it defaults to string. To define an array type, append [] to the type. 3. If there are multiple entity tables, they may have different IDs, so the namespace must be added to the header in the CSV file, otherwise the import will report an error. ID spaces are defined in the ID field of the node file using the syntax ID (<ID space identifier>). To reference an ID of an ID space in a relational file, we use the syntax START_ID(<ID space identifier>) and END_ID(<ID space identifier>). For example, I have two tables that need to be imported, and I need to specify the namespace of the entity after the ID.
  • movieId:ID(Movie-ID),title,year:int,:LABEL
  • personId:ID(Actor-ID),name,:LABEL
4. If the following problems occur: Neo4j bulk import "neo4j-admin import" OutOfMemoryError: Java heap space and OutOfMemoryError: GC overhead limit exceeded (Neo4j bulk import “neo4j-admin import” OutOfMemoryError: Java heap space and OutOfMemoryError: GC overhead limit exceeded) It means that the allocated memory is not enough, and the configuration of neo4j.conf needs to be modified. This file can be found from a subfolder in the upper-level folder of neo4j-import. Two values ​​need to be modified
  • dbms.memory.heap.initial_size=4096m
  • dbms.memory.heap.max_size=16384m
In the end, my computer got stuck on the fourth problem. There was not enough computer memory, so I chose to use CYPHER's LOAD statement to import CSV. You can use syntax like USING periodic commit 10000 to improve import efficiency.
