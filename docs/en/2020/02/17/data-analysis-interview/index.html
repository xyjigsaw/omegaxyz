business logic

Data analysis follows a certain process, which not only ensures that the work content at each stage of data analysis is followed, but also makes the final results of the analysis more accurate and convincing. Generally, data analysis is divided into the following steps:
  1. Understand business, determine goals, and clearly analyze needs;
  2. Data understanding, collecting original data, describing data, exploring data, and testing data quality;
  3. Data preparation, selecting data, cleaning data, structuring data, integrating data, and formatting data;
  4. Establish models, select modeling technology, parameter tuning, generate test plans, and build models;
  5. Evaluate the model, conduct a relatively comprehensive evaluation of the model, evaluate the results, and review the process;
  6. Deploy results and apply analysis results.

feature engineering

Including feature extraction, feature construction, and feature selection. The purpose of feature engineering is to screen out better features and obtain better training data. Because good features have greater flexibility, simple models can be used for training, and good results can be obtained.

Data collection/cleaning/sampling

Data collection

Before data collection, it is necessary to clarify what data to collect. The general idea is: Which data is helpful for the final result prediction? Can we collect the data? Is it fast to obtain it during online real-time calculation?Example 1:Now I want to predict users’ orders for products, or I want to recommend products to users, so what information do I need to collect?
  • Store: store rating, store category...
  • Product: product rating, number of buyers, color, material, collar shape...
  • User: historical information (lowest and highest price of purchased goods), spending power, time of stay of goods...

Data cleaning

Data cleaning is also a very important step. Most of the time, a machine learning algorithm is just a processing machine. As for the final product, it depends on the quality of the raw materials. Data cleaning is to remove dirty data, such as the order data of certain products.So how to determine dirty data?
  • Simple attribute determination: a person is 3 meters tall or more; a person buys 100,000 cards in a month.
  • Combination or statistical attribute determination: It is said that it is in the United States, but its IP has always been a news reading user in mainland China? You want to determine whether a person will buy basketball shoes. 85% of the sample users are female?
  • Fill in the corresponding default values: Untrustworthy samples are discarded, and fields with an extremely large number of default values ​​are not considered to be used.
Data cleaning standards:
  • Integrity of data ---- for example, the lack of gender, place of origin, age, etc. in the person's attributes
  • Uniqueness of data—for example, duplication of data from different sources
  • Authoritativeness of data - for example, the same indicator appears with data from multiple sources, and the values ​​are different
  • Legality of data - for example, the data obtained is inconsistent with common sense and the age is greater than 150 years old
  • Data consistency - for example, different indicators from different sources have the same actual connotation, or the same indicator has inconsistent connotations.

Data sampling

After collecting and cleaning the data, the positive and negative samples are unbalanced, so data sampling is required. Sampling methods include random sampling and stratified sampling. However, random sampling has hidden dangers, because the data obtained from a certain random sampling may be very uneven, and stratified sampling is more often used based on characteristics. Data standardization and normalization

max-min:A drawback of this method is that when new data is added, max and min may change and need to be redefined.Z-score: The most common standardization method is Z standardization, which is also the most commonly used standardization method in SPSS. The default standardization method of spss is z-score standardization. Also called standard deviation standardization, this method standardizes the data by giving the mean and standard deviation of the original data. The processed data conforms to the standard normal distribution, that is, the mean is 0 and the standard deviation is 1. The transformation function is: x∗=x−μσ, where μ is the mean of all sample data and σ is the standard deviation of all sample data. The z-score standardization method is suitable for situations where the maximum and minimum values ​​of attribute A are unknown, or when there are outliers that exceed the value range.

ROC and AOC

The ROC (Receiver Operating Characteristic) curve is the receiver operating characteristic curve (ROC curve for short), also known as the sensitivity curve (sensitivity curve), which is used to evaluate the quality of a binary classifier. AUC (Area Under Curve) is defined as the area under the ROC curve, and the value of this area will not be greater than 1. And since the ROC curve is generally above the straight line y=x, the value range of AUC is between 0.5 and 1. The AUC value is used as the evaluation criterion because in many cases the ROC curve cannot clearly illustrate which classifier performs better. As a numerical value, the classifier with a larger AUC performs better. Why? Because the closer the ROC curve is to the upper left corner, the larger the AUC area and the better the classifier performance.
