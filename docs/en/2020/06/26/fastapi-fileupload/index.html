FastAPI builds file upload server

Programming LanguagesTechnologyWebPythonWebPython
FastAPI is a high-performance web framework and a Python package used to build APIs. It is suitable for building server backends with very little code and achieving front-end and back-end separation. A task is given below: use FastAPI to build a file upload server, provide an upload interface, and save it to the specified location on the server. Python packages required: fastapi and uvicorn.

server code

Among them, with open(file.filename, "wb") saves the file uploaded by the customer. The uploaded URL address is host:port/file_upload, and the port and host can be customized. Note that if cross-domain problems occur during front-end configuration, you need to add FastAPI cross-domain rules.
import time
import uvicorn
from fastapi import FastAPI, File, UploadFile

app = FastAPI()


@app.post("/file_upload")
async def file_upload(file: UploadFile = File(...)):
    start = time.time()
    try:
        res = await file.read()
        with open(file.filename, "wb") as f:
            f.write(res)
        return {"message": "success", 'time': time.time() - start, 'filename': file.filename}
    except Exception as e:
        return {"message": str(e), 'time': time.time() - start, 'filename': file.filename}


if __name__ == '__main__':
    uvicorn.run(app=app, host="127.0.0.1", port=8000, workers=1)

Client code example

The file that needs to be uploaded by this client is in the path, and the url is the API for file upload.
import requests
url = "http://127.0.0.1:8000/file_upload"
path = "C:\\Users\\me\\Desktop\\pic.jpeg"
files = {'file': open(path, 'rb')}
r = requests.post(url, files=files)
print(r.url)
print(r.text)
PS: In addition to the Python client, you can also write post requests in JS (the server-side code does not need to be modified) to upload files to the server in the browser.
