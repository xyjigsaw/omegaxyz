ACM-IEEE-arXiv paper information crawler

Programming LanguagesTechnologyPythonPythonGithub

ACM-IEEE-arXiv Info Spider (under development)

The purpose of this project is to capture structured information of papers from digital libraries. This information can be used for scholar analysis, building knowledge graphs, etc. Code address:https://github.com/xyjigsaw/ACM-IEEE-arXiv-Spider

Supported databases

  • ACM (Completed, Support Digital Library Search Result)
  • IEEE (under development, Support Single Page)
  • arXiv (Completed, Support All Categories)
  • AAAI (Completed, Support 2009-2019 AAAI Conferences)
Keywords: Python, Scrapy, MySQL, Papers

rely

  • Python 3.6
  • MySQL 8.0.17
  • scrapy
  • selenium
  • PhantomJS (optional only for IEEE_Spider)
  • scrapy_proxies
  • logging
  • re
  • pymysql
  • twisted
  • json
  • fake_useragent

Databases and data structures

You can execute papers.sql to initialize the database. MYSQL_DBNAME = 'papers' TABLE_NAME = {'ACM_Data', 'IEEE_Data', 'arXiv_Data'}

characteristic

1. A script runs automatically to obtain free proxies (HTTP only) and will soon be integrated into the main cut-and-paste based program. 2. For each request, it will generate a random proxy and user-agent. 3. TXT files, raw json (not exact json) and MySQL are provided to store data. 4. Optional layer-based logging is given. 5. Use asynchronous mode as the data storage mechanism of the MySQL pipeline to solve the problem of crawler data flooding and improve the operating efficiency and reliability of the program.

Install and run

Before you start scrapy, you should first customize the settings. When you start IEEE_Spider, you need to add js middleware based on selenium and PhantomJS. terminal
scrapy crawl ACM_Spider
Or... 
scrapy crawl IEEE_Spider
...
