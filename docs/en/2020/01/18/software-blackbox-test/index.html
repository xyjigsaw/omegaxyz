Software testing - black box testing

TechnologySoftware EngineeringSoftware Engineering软件测试

1. Test Overview

1.1 Overview

This test report is a black box test of the computer program ability online assessment system. Black box testing can be conducted without knowing the internal structure and code of the program. It is used to test whether the software functions meet user needs, whether it achieves user expectations, and whether it has a good human-computer interaction experience.

picture1.1 Black box testing process

1.2 Test tasks and test scope

The main functions include: ① login module, ② online examination module, ③ test question management module, ④ test paper generation module, ⑤ score inquiry module. The test mainly tests the above functions based on the user's demand specifications, software demand specifications and corresponding documents.

1.3 Defect level classification

surface 1.2 Defect grading

grade describe illustrate
5-Emergency Discover repeatable fatal problems ——causing system crash; ——Resulting in the loss of program modules; ——The main business process has a breakpoint; ——Memory leak; ——Result in crash
4-very high Discover serious and repeatable problems ——The function under test cannot be implemented correctly; ——Software errors lead to data loss; ——Error in processing of measured data; ——User needs are not realized.
3-high General errors or imperfections in function implementation ——Operation interface error; ——Print content and format errors; ——Simple input restrictions are not placed in the foreground for control; ——No prompt is given for deletion operation.
2-medium small mistakes ——Interface is not standardized; ——The auxiliary instructions are unclear; ——Irregular input and output; ——No prompts are given to the user for long operations; ——The prompt window text does not use industry terminology.
1-low Suggestion error It is not stated in the requirements specification and user manual, but it affects the user's convenience in using the software, etc.
