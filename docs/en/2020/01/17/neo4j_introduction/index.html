Neo4j data import and visualization

TechnologyDatabasesDatabasesNeo4j

Introduction

Neo4j is a high-performance NoSQL graph database that stores structured data on the network instead of tables. It is an embedded, disk-based, fully transactional Java persistence engine, but it stores structured data on the network (called a graph from a mathematical perspective) rather than in tables. Neo4j can also be viewed as a high-performance graph engine that has all the features of a mature database. Programmers work in an object-oriented, flexible network structure rather than strict, static tables - but they can enjoy all the benefits of a fully transactional, enterprise-class database.

Neo4j property graph data model

Neo4j graph database follows the property graph model to store and manage its data. Property graph model rules
  • Represent data in nodes, relationships and attributes
  • Both nodes and relationships contain attributes
  • relationship connection node
  • Properties are key-value pairs
  • Nodes are represented by circles and relationships are represented by directional keys.
  • Relationships have directions: one-way and two-way.
  • Each relationship contains a "start node" or "from node" and a "to node" or "end node"
In a property graph data model, relationships should be directional. If we try to create a relationship without direction then it will throw an error message. In Neo4j, relationships should also be directional. If we try to create a relationship without direction, then Neo4j will throw an error message, "Relationship should be directional". Neo4j graph database stores all its data in nodes and relationships. We don't need any additional RRBMS database or SQL-less database to store Neo4j database data. It stores its data in a native format in the form of graphs. Neo4j uses the native GPE (Graphics Processing Engine) engine to use its native graph storage format. The main building blocks of the graph database data model are:
  • node
  • relation
  • property

The command language used is: Cypher Cypher is a declarative graph database query language that can efficiently query and update graph data. Cypher statements can be divided into three categories, including read statements, write statements and general statements:
  • Read statements: MATCH, OPTIONAL MATCH, WHERE, START, AGGREGATION, LOAD CSV
  • Write statements: CREATE, MERGE, SET, DELETE, REMOVE, FOREACH, CREATE UNIQUE
  • Common statements: RETURN, ORDER BY, LIMIT, SKIP, WITH, UNWIND, UNION, CALL
