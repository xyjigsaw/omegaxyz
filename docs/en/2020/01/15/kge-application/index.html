Application scenarios of knowledge graph embedding

TechnologyMachine LearningKnowledge GraphsKnowledge Graphs

In-KG applications (applications within the KG range)

Link prediction

The link prediction task, sometimes called entity prediction or entity ranking, is used to predict whether there is a specific relationship between two entities. That is, given the head entity h and relationship r, predict the tail entity t; or the tail entity t and relationship r, predict the head entity h. Its essence is a KG completion task, which is to add missing knowledge to the graph. At the same time, the relationship between two given entities can also be predicted, that is, the head and tail entities are known, and r is found. Taking the prediction head entity as an example, each entity h in KG can be used as a candidate answer, and then the f(h,t) score is calculated for each entity. For evaluation purposes, it is common practice to record the order in which the correct answers are ranked in an ordered list, in order to see if the correct answers can be ranked before the incorrect answers. A smaller number of correct answer rankings indicates better performance. Various evaluation criteria were designed based on these rankings, such as average ranking (the average of predicted ranking numbers), average reciprocal ranking (the average of reciprocal rankings), Hits@n (the proportion of n correct rankings), and AUC-PR (area under the precision-recall curve).

Triple Classification

Triplet classification is used to determine whether the facts (h, r, t) represented by the triplet are true or false. This task can also be regarded as a KG input completion. Set a threshold for the scoring function to determine whether the fact is correct.

Entity Classification

Entity classification can be viewed as a specific link prediction task, which aims to classify entities into different semantic categories. Similar prediction and evaluation methods are the same as for link prediction.

Entity Resolution

Determine whether two entities refer to the same object. For example, the two entities Peking University and Peking University are actually the same object. Entity resolution can remove these duplicate nodes. If a relation declares whether two entities are equal (denoted as EqualTo), and the embedding of the relation has been learned. In this case, entity parsing degenerates into a triple classification problem, that is, determining whether the triple (x,EqualTo,y) is correct.

Out-of-KG applications (applications that break through the boundaries of KG input and expand to a wider range of fields)

Relation Extraction

The aim is to extract relational facts from plain text where entities have been detected. Currently, there is a method that combines TransE with text-based extractors for relation extraction. In addition, plain text and KG relationships can be jointly embedded, that is, text and KGs are represented in the same matrix. Each row of the matrix represents a pair of entities, and each column represents a mentioned text or KG relationship, and the corresponding entry is set to 1 if two entities co-occur with a mention in the plain text or with a relationship in KGs, and 0 otherwise. From the figure, you can see that the training instance can see both text mentions and KG relationships, but in the test instance, only text mentions are included. Therefore, relation extraction is used to predict the missing KG relations for test instances.

Question Answering

Given a question expressed in natural language, the task is to retrieve the correct answer supported by triples or triplets from KG. One possible approach is to learn low-dimensional vector embeddings of words and KG components such that the representation of the question and the corresponding answer are close to each other in the embedding space.

Recommender Systems

Recommendation systems are widely used. One method is collaborative filtering technology that models the interaction between users and items as an instant representation of products between users and items. However, the interactions between users and items may be sparse. Some scholars have proposed a hybrid recommendation framework that utilizes heterogeneous information within KG to improve the quality of collaborative filtering - using three types of information stored in KG, including structural knowledge (fact triples), textual knowledge (for example, a summary of the textuality of a book or a movie), and visual knowledge (for example, a book cover or a movie poster image), to generate a semantic representation for each item.
