Vue rapid development notes

Programming LanguagesTechnologyWebJavaScript&HtmlWebJavaScriptVue
Vue.js (pronounced /vjuː/, similar to view) is a progressive framework for building user interfaces. Vue only focuses on the view layer and adopts a bottom-up incremental development design.

Vue project initialization

Use vue-cli tool
$ npm install -g vue-cli // vue-cli installed globally

$ vue init webpack project_name // init project

? Project name project_name // 项目名称
? Project description A Vue.js project // Project description
? Author NathanYang // author
? Vue build standalone  // 
? Install vue-router? Yes  // 
? Use ESLint to lint your code? Yes  // Not recommendated
? Pick an ESLint preset Standard
? Setup unit tests with Karma + Mocha? No
? Setup e2e tests with Nightwatch? No // end2end

   vue-cli · Generated "project_name".

   To get started:

     cd first-test-vue
     npm install
     npm run dev

   Documentation can be found at https://vuejs-templates.github.io/webpack
Directory structure, etc. reference:https://www.jianshu.com/p/45b5ce2ab1c5

tool

When using backends such as FastAPI, it is recommended to use axios tools
npm install axios -S
Quickly build UI, use ElementUI
npm install element-ui -S
Front-end display PDF
npm install vue-pdf -S
refer to:https://www.jianshu.com/p/6464eee4d0c1Differences in suffixes such as -g, -s
npm i module_name  -S  = >  npm install module_name --save    写入到 dependencies 对象

npm i module_name  -D  => npm install module_name --save-dev   写入到 devDependencies 对象

npm i module_name  -g  全局安装
refer to:https://www.cnblogs.com/web-record/p/10904907.html

run

# install dependencies
npm install

# serve with hot reload at localhost:8080
npm run dev

# build for production with minification
npm run build

# build for production and view the bundle analyzer report
npm run build --report

other

It is recommended to use fastapi for the backend to cooperate with the frontend, where fastapi needs to write cross-domain requests.
