It also shows load averages at intervals of 1 minute, 5 minutes, and 15 minutes.
# uptime
 08:16:26 up 22 min,  1 user,  load average: 0.00, 0.03, 0.22
Check uptime versionThe uptime command has no options other than uptime (uptime) and version (version). If the time is less than 1 day, it only gives information in the form of hours:mins.
[tecmint@linuxprobe ~]$ uptime -V
 procps version 3.2.8
2.wOrderThis command will display the currently logged in users and their processes, as well as the load average. Additionally, it shows login name, tty name, remote host, login time, idle time, JCPU, PCPU, commands and processes.
# w
08:27:44 up 34 min,  1 user,  load average: 0.00, 0.00, 0.08
USER    TTY    FROM     LOGIN@   IDLE   JCPU   PCPU WHAT
tecmint  pts/0    192.168.50.1     07:59    0.00s  0.29s  0.09s w

 Available options
 -h：不显示标题。

 -s：不显示JCPU和PCPU。

 -f：不显示字段信息。

 -V：（大写V）－显示版本。
3.usersOrderThe users command displays the currently logged in users. This command has no other parameters except help and version.
# users
 Tecmint
4.whoOrderThe who command only returns the user name, date, time and host information; the who command is similar to the w command, but unlike the w command, who does not output information about the operations performed by the user. You may wish to take a closer look at the difference between the two commands who and w.
# who
 tecmint  pts/0        2010-09-18 07:59 (192.168.50.1)

# w
 08:43:58 up 50 min,  1 user,  load average: 0.64, 0.18, 0.06
 USER    TTY    FROM    LOGIN@   IDLE   JCPU   PCPU WHAT
 tecmint  pts/0    192.168.50.1     07:59    0.00s  0.43s  0.10s w

 who command options

 -b：显示上一次系统重启日期和时间。

 -r：显示当前的运行级别。

 -a，–all：显示累积的所有信息。
5.whoamiOrderThe whoami command outputs the name of the current user; you can also use the "who am i" command to display the current user. If you use the sudo command to log in as the root user, the "whoami" command returns that the root user is the current user. If you want to know which user is logged in, use the "who am i" command.
# whoami
 tecmint

7.crontabOrderYou can use the crontab command and the -l option to list the current user's scheduled tasks.
# crontab -l
 00 10 * * * /bin/ls >/ls.txt
You can use the -e option to edit crontab. In the following example, you will use the VI editing tool to open the scheduled task, make the necessary changes, and press the :wq key to exit. This will automatically save the settings.
# crontab -e

10.cpOrderCopies files from source to destination, keeping the same mode.
# cp -p fileA fileB
You will be prompted before overwriting the file.
# cp -i fileA fileB
11.mvOrderRename fileA to fileB; the -i option will prompt before overwriting; if the file already exists, confirmation will be asked.
# mv -i fileA fileB
12.catOrderThe cat command is used to view multiple files at the same time.
# cat fileA fileB
If a file cannot be displayed on one screen/page, you can use the cat command to combine the more and less commands to view the file content.
# cat install.log | less
      or
# cat install.log | more
13.cdCommand (switch directory)With the help of the cd command (switch directory), it will go to the fileA directory.
# cd /fileA
14.pwdCommand (output working directory)The pwd command returns the current working directory.
# pwd
/root
15.sortOrderSorts a text file line by line in ascending order. If the -r option is used, it will be sorted in descending order.
#sort&nbsp;fileA.txt
#sort&nbsp;-r&nbsp;fileA.txt
16.viOrderFor most UNIX-like operating systems, vi is the most popular text editor. The following example uses the -R option to open a file in read-only mode and press ":q" to exit the vi window.
# vi -R /etc/shadows
17.sshCommand (Secure Shell)The ssh command is used to log in to a remote host; for example, the following ssh example will use the user as narad to connect to the host (192.168.50.2).
# ssh narad@192.168.50.2
To check the ssh version, use the option -V (uppercase) to display the ssh version.
# ssh -V
OpenSSH_5.3p1, OpenSSL 1.0.0-fips 29 Mar 2010
18.ftpor sftpOrderThe ftp or sftp command is used to connect to a remote ftp host; ftp refers to the file transfer protocol, and sftp is the secure file transfer protocol. For example, the following command will connect to the ftp host (192.168.50.2).
# ftp 192.168.50.2
# sftp 192.168.50.2
Just as you can use mput to upload multiple files to a remote host, we can also use mget to download multiple files from a remote host.
# ftp > mput *.txt
# ftp > mget *.txt
19.serviceOrderThe service command calls the script located in the /etc/init.d/ directory and executes the script. There are two ways to start any service; for example, we use the service command to start a service named httpd.
# service httpd start
# /etc/init.d/httpd start

22.tarOrderThe tar command is used to compress files and folders under Linux; for example, the following command will create a compressed archive in the /home directory with the file name archive-name.tar.
# tar -cvf archive-name.tar /home
To decompress a tar archive, use the options shown below.
# tar -xvf archive-name.tar
23.grepOrdergrep searches files for specific strings; only displays the tecmint user in the /etc/passwd file. We can use the -i option to ignore case.
# grep tecmint /etc/passwd
 tecmint:x:500:500::/home/tecmint:/bin/bash
24.findOrderThe find command is used to search for files, strings, and directories; in the following example, the find command searches for the tecmint word in the "/" partition and returns the output.
# find / -name tecmint
 /var/spool/mail/tecmint
 /home/tecmint
 /root/home/tecmint

26.lastOrderWith the last command, we can observe user activities in the system; this command can also be executed as a normal user. It will display complete user information, such as terminal, time, date, system restart or startup, and kernel version. This is a practical command for troubleshooting.

27.psOrderThe ps command displays information about processes running on the system; the following example only displays the init process.
# ps -ef | grep init
 root         1     0  0 07:53 ?        00:00:04 /sbin/init
 root      7508  6825  0 11:48 pts/1    00:00:00 grep init
28.killOrderUse the kill command to terminate the process; first use the ps command to find the process ID, as shown below, and then use the kill -9 command to terminate the process.
# ps -ef | grep init
 root         1     0  0 07:53 ?        00:00:04 /sbin/init
 root      7508  6825  0 11:48 pts/1    00:00:00 grep init
# kill- 9 7508
29.rmOrderThe rm command is used to clear or delete files without prompting for confirmation.
# rm filename
Use the -i option to confirm before deletion; use the "-r" and "-f" options to forcefully delete the file without confirming it.
# rm -i test.txt
rm: remove regular file `test.txt'?
30.mkdirOrderThe mkdir command is used to create directories under Linux.
