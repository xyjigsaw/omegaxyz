Decision tree overview

技术域机器学习

Summary

Decision Tree is a decision analysis method that based on the known probability of occurrence of various situations, forms a decision tree to calculate the probability that the expected value of the net present value is greater than or equal to zero, evaluates project risks, and determines its feasibility. It is a graphical method that intuitively uses probability analysis. Because this kind of decision branch is drawn graphically like the branches of a tree, it is called a decision tree. In machine learning, a decision tree is a prediction model that represents a mapping relationship between object attributes and object values. Entropy = How messy the system is, using algorithm ID3, C4.5 and C5.0 spanning tree algorithms use entropy. This measure is based on the concept of entropy in informatics theory. A decision tree is a tree structure in which each internal node represents a test on an attribute, each branch represents a test output, and each leaf node represents a category. Classification tree (decision tree) is a very commonly used classification method. It is a kind of supervised learning. The so-called supervised learning is to give a bunch of samples, each sample has a set of attributes and a category. These categories are determined in advance, and then through learning, a classifier can be obtained that can correctly classify new objects. Such machine learning is called supervised learning.

Decision tree popular explanation

In layman's terms, the idea of ​​decision tree classification is similar to finding objects. Now imagine that a girl’s mother wants to introduce a boyfriend to the girl, so she has the following dialogue:Daughter: How old are you? Mother: 26. Daughter: Are you handsome? Mother: Very handsome. Daughter: Is the income high? Mother: Not very high, just average. Daughter: Are you a civil servant? Mother: Yes, I work at the tax bureau. Daughter: Okay, I’ll go meet you.This girl's decision-making process is a typical classification tree decision-making. It is equivalent to dividing men into two categories based on age, appearance, income and whether they are civil servants: seen and not seen. Decision tree construction

Unlike the Bayesian algorithm, the decision tree construction process does not rely on domain knowledge and uses an attribute selection metric to select attributes that best divide tuples into different classes. The construction of the so-called decision tree is to perform attribute selection measurement to determine the topological structure between each feature attribute. The key step in constructing a decision tree is splitting attributes. The so-called split attribute is to construct different branches according to different divisions of a certain characteristic attribute at a certain node. The goal is to make each split subset as "pure" as possible. To be as "pure" as possible is to try to make the items to be classified in a split subset belong to the same category. Split properties are divided into three different cases: 1. The attributes are discrete values and do not require the generation of a binary decision tree. At this time, each division of attributes is used as a branch. 2. The attributes are discrete values ​​and require the generation of a binary decision tree. At this time, a subset of the attribute division is used for testing, and it is divided into two branches according to "belongs to this subset" and "does not belong to this subset". 3. The attribute is a continuous value. At this time, a value is determined as the split point split_point, and two branches are generated according to >split_point and <=split_point. The key content of constructing a decision tree is the attribute selection metric. The attribute selection metric is a selection splitting criterion. It is a heuristic method to "best" divide the data D of the training set of given class labels into individual classes. It determines the topology and the choice of split_point. There are many attribute selection metric algorithms, generally using the top-down recursive divide-and-conquer method and adopting a greedy strategy without backtracking. There are two common algorithms, ID3 and C4.5. The specific algorithm will be explained in detail in another article.

Additional information

This situation may occur during the construction of a decision tree: all attributes are used up as splitting attributes, but some subsets are not pure sets, that is, the elements in the set do not belong to the same category. In this case, since no more information is available, these subsets are generally "majority vote”, that is, use the category with the most occurrences in this subset as this node category, and then use this node as a leaf node.
