Python course design Windows system resource detector

1. Requirements analysis

(1) Folder dynamic monitoring:Windows system C drive is the system disk, which stores many important data files. The available space is sufficient at first, but after using the computer for a period of time, we will find that the C drive space is getting larger and larger. This is a common problem in Windows systems, so I hope to make a program that can monitor selected folders in real time. This program is different from the resource manager that comes with Windows. It can dynamically and automatically monitor the creation, modification and deletion of files in the folder (of course the program needs to be hung in the background), and feedback the time and path of the file changes to the user, so that the user can know at the first time which files are the initiator of the system disk (or other disks) becoming larger.(2) Temporary file detection and CPU monitoring:Our commonly used system optimization software often tends to be bundled with installation and deliver advertisements. This problem is particularly serious with 360 Security Guard, Master Lu, and QQ Computer Manager. Here I imagine making a small GUI program that can easily integrate our commonly used Master Lu real-time system monitoring and system temporary file detection and cleaning to avoid problems like 360 ​​bundled installation and self-starting.(3) Resource manager:Sometimes, we need to know the size of a folder, the number and size of stored files, Windows' own resource manager can do this, but it has a flaw, that is, it can only find large files in the current folder at a time, but cannot find large files in subfolders in the current folder. Considering that there is a tool for finding large files in 360 Security Guard, I wanted to use Python to implement it.(4) File exploration:For a student who is coding, we often need to obtain the MD5 code value and other related information of a file. By right-clicking on the file properties, you can see most of the information, but you cannot view its encryption information, so I integrated the MD5 code and SAH1 code query functions in the GUI program to facilitate file verification.(5) Automatic shutdown:At night, the speed of 4G drops from hundreds of kb/s to dozens of kb/s, so we often need to turn on computer hotspots to preserve the battery life of our mobile phones. However, those of us on the upper bunk don’t want to come down and turn off the computer after turning off the lights. At this time, we need to shut down the computer regularly. In fact, Windows computers can use console commands and create scheduled tasks to implement scheduled shutdown of the computer. However, for console commands, many people do not know how to enter the command (at 03:00 /every:M,T,W,Th,F,S,Su shutdown -s -t 30 From Monday to Sunday, shut down at 3 am every day, and there will be a 30-second prompt before shutting down!); for scheduled tasks, you first have to open the control panel, create a new task, and then enter the instructions. Especially on a win10 computer, I spent a long time looking for the scheduled task panel, which was very labor-intensive. In addition, if we want to cancel the scheduled shutdown, it is also a very troublesome thing for Windows computers. So in the GUI program, I added a scheduled shutdown function, which can set and cancel the scheduled shutdown, and more importantly, can save before shutting down. 2. Program analysis and design

(1) Program running and event triggering mechanism

The GUI program to be done contains multiple functions, and more importantly, it needs to dynamically monitor the system. Considering that when a GUI program executes a dynamic update function, if the execution time of the function is too long, it will cause the program to enter a state of suspended animation, making it impossible to operate and move the window. At this time, multi-threaded programming is required. Through research on the threading module, I divided the event triggering modules of each different function into blocks, and finally integrated them into one main interface. In addition, the functions triggered by the event are placed in another class, and an external class (class WorkerThread(Thread)) is defined externally to execute events in the main interface usage class. The events that need to be executed in the main function will be passed into the external multi-threaded class through parameters and executed synchronously. The message generated by the external multi-threaded class will be delivered to the main function and its module that have subscribed to the message through pub in wx.lib.pubsub.legend:  

(2) Folder dynamic monitoring module

If a worker wants to do his job well, he must first sharpen his tools. For dynamic monitoring, I first chose the watchdog and logging modules. First, I use watchdog to monitor changes in specified directories/files, such as adding or deleting files or directories, modifying file contents, renaming files or directories, etc. Each change will generate an event, and there will be a specific event class corresponding to it. However, after searching the watchdog manual, I found that watchdog messages can only be sent to the console through system warnings, and cannot be sent to the console as ordinary str string type data. Therefore, I used the logging.info and other messages in the logging module to write to the external database file first, and then the program read the data from the database file and sent it to the GUI interface using multi-threading. In addition, in order to prevent repeated output of information in the database, I use a function to screen and filter the opposite messages. And a user selection function is added, allowing users to select the file operations that need to be monitored: file creation, file modification or file deletion. legend:

(3) File detection module

The part of searching for file resources is slightly simpler. Use the knowledge of os.walk() learned in class to traverse the folder. In order to obtain the size of the file, I added two additional arrays, FileSize and BigFile, to save the size and path of the file during the traversal process, so that if the user wants to find a large file, they can directly extract it from FileSize based on the file size specified by the user. In order for users to obtain the values ​​of MD5 code and SHA1 code, I imported the hashlib module, so that I don’t have to worry about writing the MD5 algorithm, and it is very efficient (so life is short, I use python).

(4) Temporary file detection and deletion module

At the beginning of this part, I want to use os.walk() to screen the file name at the same time. If it is a temporary file, then delete it, otherwise skip it. I searched on Baidu for scripts on how to clean up junk files in Python (reference materials are attached). I saw that everyone had the same idea. I studied a script that is more popular on the Internet (pictured)I found that some files cannot be deleted due to system permission issues, which will cause exceptions. In addition, users cannot self-select junk files that need to be deleted. Therefore, I first added exception handling in the GUI program, and also added a user selection function to protect the user's important files from being deleted.

(5) Hardware ranking module

First, I selected a hardware ladder website for crawling operation (pictured)The website is static and clearly structured. I directly use BeautifulSoup to crawl.Use these two lines of code to obtain the required data (soup = BeautifulSoup(wb_data, 'lxml') data = soup.find_all('tr')) In addition, the data format is optimized and output in the GUI interface through simple processing. I added a user selection feature so the user can choose the size of the rankings they want to crawl. Of course, I also added a network exception handling mechanism.

(6) Implementation of dynamic update of CPU and memory

Similar to the dynamic monitoring of folders, I first chose the psutil module, which can easily obtain information about the processes running on the system and system utilization (including CPU, memory, disk, network, etc.). Using multi-threaded messaging, I start this process when initializing the interface. Originally this was just a digital display, but I hope to beautify the interface. I thought of using the matplotlib module to draw pictures, but to achieve dynamic updates of drawings, even using multi-threads will cause the interface to enter a state of suspended animation, and the code that needs to be modified is very large (the entire GUI interface needs to embed the matplotlib GUI interface again). Therefore, I gave up this idea and used the progress bar provided in the GUI interface to dynamically beautify the display of CPU and memory.

(7) Scheduled shutdown setting

Use the os.system module to directly write the console commands shutdown -s -t %d and shutdown -a Command to control. This part is not the focus, so I won’t explain it in detail.
