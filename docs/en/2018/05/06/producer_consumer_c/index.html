C language implementation of producer and consumer issues

高级语言C&C++技术域操作系统C语言多线程操作系统原理

Experiment purpose

① Implement the simulation of the producer-consumer problem to better understand this classic process synchronization problem. The producer-consumer problem is a typical PV operation problem. Assume that there is a relatively large buffer pool in the system. The producer's task is to put produced products into it as long as the buffer pool is not full, and the consumer's task is to take products from the buffer pool as long as the buffer pool is not empty. While the buffer pool is occupied, no process can access it. ②Each producer puts the products he produces into the buffer pool, and each consumer takes the products from the buffer pool for consumption. In this case, the producer and consumer processes are synchronized, because only by communicating messages can they know whether products can be deposited or removed. There is also mutual exclusion between them, that is, the producer and consumer must mutually access the buffer pool, that is, no more than two processes can be performed at the same time.

Experimental principle

Execute two threads within the same process address space. The producer thread produces items and then places the items in an empty buffer for consumption by the consumer thread. The consumer thread gets items from the buffer and then releases the buffer. When a producer thread produces an item, if no empty buffer is available, the producer thread must wait for the consumer thread to release an empty buffer. When the consumer thread consumes items, if the buffer is not full, the consumer thread will be blocked until new items are produced.

Note

①This experiment is about the mutual exclusion and synchronization between producers and consumers. The problem refers to P and V operations. The experiment assumes a shared buffer, and the producer and consumer are mutually exclusive. When one thread uses the buffer, the other thread waits until the previous thread releases the buffer. ② Producers and consumers are an empirical problem related to reality. Other similar problems can be solved by drawing inferences from one example through this principle. Through this experimental design, we can further understand the P and V of the operating system, and gain an in-depth understanding of the essence and importance of P and V operations. The theoretical knowledge in the textbook further elaborates on practical problems in reality. ③Writing variant C language in the Linux environment is slightly different from Windows. Note that when compiling the source program with the thread <pthread.h> header file in Linux, you need to add the parameter -lpthread. And if you want to output Chinese on the Linux console, you have to change it to utf-8 encoding. ④When writing the producer-consumer function, it is best to embed the P and V operations as a whole without setting up a separate function body. Otherwise, if like: Semaphore mutex=1//At this time mutex is a global variable If you write this in the producer (consumer) function body: Producer() { … Wait(mutex); … } At this time, the value of the global variable cannot be changed at all, that is, the mutex value has not changed. If you must use a function, you need to set a P function, a V function for each semaphore or not set mutex as a global variable.

code

#include <stdio.h>
#include <pthread.h>
#include <windows.h>
#define N 100
#define true 1
#define producerNum  10
#define consumerNum  5
#define sleepTime 1000

typedef int semaphore;
typedef int item;
item buffer[N] = {0};
int in = 0;
int out = 0;
int proCount = 0;
semaphore mutex = 1, empty = N, full = 0, proCmutex = 1;

void * producer(void * a){
    while(true){
        while(proCmutex <= 0);
        proCmutex--;
        proCount++;
        printf("生产一个产品ID%d, 缓冲区位置为%d\n",proCount,in);
        proCmutex++;

        while(empty <= 0){
            printf("缓冲区已满！\n");
        }
        empty--;

        while(mutex <= 0);
        mutex--;

        buffer[in] = proCount;
        in = (in + 1) % N;

        mutex++;
        full++;
        Sleep(sleepTime);
    }
}

void * consumer(void *b){
    while(true){
        while(full <= 0){
            printf("缓冲区为空！\n");
        }
        full--;

        while(mutex <= 0);
        mutex--;

        int nextc = buffer[out];
        buffer[out] = 0;//消费完将缓冲区设置为0

        out = (out + 1) % N;

        mutex++;
        empty++;

        printf("\t\t\t\t消费一个产品ID%d,缓冲区位置为%d\n", nextc,out);
        Sleep(sleepTime);
    }
}

int main()
{
    pthread_t threadPool[producerNum+consumerNum];
    int i;
    for(i = 0; i < producerNum; i++){
        pthread_t temp;
        if(pthread_create(&temp, NULL, producer, NULL) == -1){
            printf("ERROR, fail to create producer%d\n", i);
            exit(1);
        }
        threadPool[i] = temp;
    }//创建生产者进程放入线程池


    for(i = 0; i < consumerNum; i++){
        pthread_t temp;
        if(pthread_create(&temp, NULL, consumer, NULL) == -1){
            printf("ERROR, fail to create consumer%d\n", i);
            exit(1);
        }
        threadPool[i+producerNum] = temp;
    }//创建消费者进程放入线程池


    void * result;
    for(i = 0; i < producerNum+consumerNum; i++){
        if(pthread_join(threadPool[i], &result) == -1){
            printf("fail to recollect\n");
            exit(1);
        }
    }//运行线程池
    return 0;
}
