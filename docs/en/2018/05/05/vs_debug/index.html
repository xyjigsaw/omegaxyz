Visual Studio breakpoint debugging displays variable information

Programming LanguagesC & C++TechnologyC++C
Visual Studio is a very useful IDE. Using debug for debugging can not only quickly find errors, but also improve programming efficiency.The following uses step into in Visual Studio debug to debug the Fibonacci sequence and track the internal variables of the function. ①First of all, be sure to create a project file ②After editing the code, double-click the side of the code to set a breakpoint ③Right-click the breakpoint and select the hit condition (when hit) ④A selection box will appear to specify what operation to perform when the break point is hit. Fill in the curly braces {n} with the variable value that needs to be displayed. Since there is only one variable n at this time, the called function and the value of n are displayed every time the execution reaches this point.⑤At this time, only the first three values ​​of the Fibonacci sequence are calculated for analysis. The calling function is Fibonacci(int), and the value of N is 1 at this time. The first time n=1 F(1)=1 displays 1 The calling function is Fibonacci(int), and the value of N is 2 at this time. The second time n=2 F(2)=1 displays 1 The calling function is Fibonacci(int), and the value of N is 1 at this time. When n=3 for the third time, the first call of the recursion is 1 The calling function is Fibonacci(int), and the value of N is 2 at this time. When n=3 for the third time, the second recursive call is 2 The calling function is Fibonacci(int), and the value of N is 3 at this time. The final value for the third time is 3 To sum up, F(3) is calculated twice and F(2) is calculated three times during the recursive call. F(1) was called 5 times, and Fib(0) was called 3 times.
