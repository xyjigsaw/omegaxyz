PSO feature selection algorithm based on training set dynamic surrogate model

TechnologyMachine LearningEvolutionary AlgorithmsPSO算法特征选择

question

①The Wrapper feature selection algorithm based on evolutionary computing is very computationally expensive. ②The feature selection algorithm based on PSO evolutionary calculation has significantly improved the evolutionary efficiency, but the evaluation process still takes a long time.

contribute

The author proposes a dynamic proxy model that applies clustering to the training set, which helps to obtain features on the data set to make the selected features better.

DROP3 algorithm

Purpose: retain instances on the class boundary, remove instances within the class, and construct an agent training set Main idea: Use KNN algorithm and sorting to remove noisy instances

Algorithm steps

①First remove all features misclassified by the KNN algorithm on the training set ②For two categories, calculate the distance between each instance and the nearest instance of other classes, and sort them ③Remove the above distant instances ④Remove instances that do not affect the prediction results of other instances

Example

 

Agglomerative Clustering

Purpose:Solve the problems of the DROP3 algorithm to construct a proxy model (abandoning the KNN algorithm) bottom-up clusteringFirst, each instance forms a class of its own, and the two nearest ones are merged into one class. The central instance of each class is selected and added to the agent training set model. (The number of classes is equal to the agent training set instance size, user settings)

dynamic agent model

Real fitness: fitness value on the original training setSurrogate fitness:fitness value on the surrogate model

Purpose

Since the feature subset will change every iteration, in order to maintain the consistency of the above two values, it must be dynamically adjusted in a timely manner.

Main steps

① Use AGG algorithm to generate multiple proxy models ②Use the original training set to calculate the fitness value and find the individual with the optimal fitness value f0X. ③Calculated in the agent poolXThe fitness value of {f1, f2,...,fm} is obtained, and the |fi-f0| with the smallest calculated difference is obtained, and this agent is selected. ④Evolution begins, everyISThe next generation uses the selected agent model for particle evaluation and update. If gbest has the best evaluation on the original training set, if gbest does not improve, the agent with the smallest gap |fi-f0| is selected. ⑤ISAfter iteration, all particles are evaluated and updated using the original training set.
