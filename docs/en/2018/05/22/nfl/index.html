NFL (no free lunch) theorem

TechnologyMachine Learning
What NLF talks about is that without considering the specific problem, no algorithm is better than another algorithm, not even random guessing is better.
  • There is no universally applicable "optimal classifier" that is independent of specific applications.
  • The learning algorithm must make a "hypothesis" related to the problem domain, and the classifier must be adapted to the problem domain.
However, the premise of the NFL theorem is that all problems have an equal chance of occurring, or that all problems are equally important. But in reality, we often get specific data, specific distributions, and solve specific problems, so we only need to solve the problems we are concerned about, and do not need to consider whether this model can solve other problems well. It only makes sense for us to compare different models for a specific problem. This is not only true for machine learning, but also when we do other algorithms. Without considering the actual problem solved, it is difficult to say whether the algorithm is good or bad.

for example:

d= training set;
m=The number of elements in the training set;
f= Input-output relationship of target;
h= Assume that the input parameter of algorithm f is d and the result is h
C=Offline training loss function for f, h
All algorithms are identical if measured by the following parameters: E(C|d), E(C|m), E(C|f,d), or E(C|f,m).

in conclusion:

Let’s compare two algorithms A and B:
1. For all problems, A is not always better than B
2. For all problems, a specific algorithm is not always better than a random algorithm.

The most important implication of the NFL theorem is that it makes us clearly realize that it is meaningless to talk about "what learning algorithm is better" without specific problems, because if all potential problems are considered, all algorithms are equally good. To talk about the relative advantages and disadvantages of algorithms, we must focus on specific problems; a learning algorithm that performs well on some problems may not be satisfactory on another problem. Whether the inductive preference of the learning algorithm itself matches the problem often plays a decisive role. ——Zhou Zhihua
