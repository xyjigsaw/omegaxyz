Summary of C++ pointers

Programming LanguagesC & C++TechnologyC++C

Pointers and variables

  • int p; //This is an ordinary integer variable.
  • int *p; //Start from P first and combine it with * first, so it means that P is a pointer, and then combine it with int, which means that the type of the content pointed to by the pointer is int type. So P is a pointer that returns integer data.

Pointers and arrays

  • int p[3]; //Start from P first, first combine it with [] to show that P is an array, and then combine it with int to show that the elements in the array are integer, so P is an array composed of integer data.
  • int *p[3]; //Start from P, first combine it with [], because its priority is higher than *, so P is an array, and then combine it with *, indicating that the elements in the array are pointer types, and then combine it with int, indicating that the type of the content pointed to by the pointer is integer, so P is an array composed of pointers that return integer data.
  • int (*p)[3]; //Start from P, first combine it with *, indicating that P is a pointer, and then combine it with [] (this step with "()" can be ignored, just to change the priority), indicating that the content pointed to by the pointer is an array, and then combine it with int, indicating that the elements in the array are integers. So P is a pointer to an array composed of integer data.

Pointers and functions

  • int p(int); //Start from P, first combine it with (), indicating that P is a function, and then enter () for analysis, indicating that the function has a parameter of an integer variable, and then combine it with the external int, indicating that the return value of the function is an integer data.
  • Int (*p)(int); //Start from P, first combine it with the pointer, indicating that P is a pointer, then combine it with (), indicating that the pointer points to a function, then combine it with the int in (), indicating that the function has an int type parameter, and then combine it with the outermost int, indicating that the return type of the function is an integer type, so P is a pointer to a function with an integer parameter and an integer return type.
  • Call: p (actual parameters)

Pointers and objects

  • Point p; //Declare a Point class object
  • Point *Ptr=&p; //Declare a pointer Ptr pointing to Point class object p to call object members through the pointer
  • Ptr->data member name ((*Ptr).data member name) Ptr->function member name ((*Ptr).function member name)

Pointer to non-static member of class

  • 1 statement
–Declares a pointer type specifier pointing to a public data member Class name::*Pointer name; – Declare a pointer type specifier pointing to a public function member (class name::*pointer name) (parameter list);
  • 2 points
--Pointer to data member pointer name =& class name::data member name; --Pointer to function member pointer name = class name::function member name;
  • 3 calls
– Access data members by combining the object name (or object pointer) with the member pointer Object name.* Class member pointer name or: Object pointer name—>*Class member pointer name – Access function members by combining the object name (or object pointer) with the member pointer (Object name.* Class member pointer name) (Parameter list) Or: (Object pointer name—>* class member pointer name) (parameter list)

Pointer to static member of class

  • Access to static members of a class does not depend on the object
  • You can use ordinary pointers to point to and access static members

this pointer (pointer to the current object)– When calling a member function through an object, the system first assigns the address of the object to the this pointer, and then calls the member function. When the member function operates on the data members of the object, the this pointer is implicitly used.
