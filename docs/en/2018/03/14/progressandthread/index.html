The difference and connection between process and thread

Technology操作系统操作系统原理

concept

process, is the basic unit for allocating and managing resources during the execution of concurrently executed programs. It is a dynamic concept and the basic unit for competing for computer system resources. Each process has its own address space, which is process space or (virtual space).threadA thread is a part of a process, and a process without threads can be considered single-threaded. Threads are sometimes called light-weight processes or lightweight processes and are also a basic unit of CPU scheduling.

the difference

The execution of the process is linear, although interruptions or pauses may occur in the middle, the resources owned by the process only serve the linear execution process. Once a process context switch occurs, these resources must be protected. This is the macro execution process of the process. Processes can be divided into two types: single-threaded processes and multi-threaded processes. We know that a process has three parts: a process control block PCB, related program segments, and a set of data structures that the program segment operates on. The execution process of a single-threaded process is linear at the macro level, and there is only a single execution process at the micro level; while the execution process of a multi-threaded process is also linear at the macro level, but there can be multiple execution operations (threads) at the micro level, such as different code fragments and related data structure sets.The change of the thread only represents a change of the CPU execution process, but does not change the resources owned by the process. removeIn addition to the CPU,The allocation of software and hardware resources within the computer has nothing to do with threads, a thread can only share resources of the process to which it belongs. Similar to the process control table and PCB, each thread also has its own thread control table TCB, and the thread status information saved in this TCB is much less than the PCB table. This information is mainly the relevant pointer stack (system stack and user stack), and the status data in the registers.The process has a complete virtual address space and exists independently of the thread; on the contrary, the thread is part of the process and does not have its own address space. It shares all resources allocated to the process with other threads in the process.. Right now:1. Thread is the smallest unit of program execution, and process is the smallest unit of resources allocated by the operating system; 2. A process consists of one or more threads. Threads are different execution routes of code in a process; 3. Processes are independent of each other, but threads in the same process share the program's memory space (including code segments, data sets, heaps, etc.) and some process-level resources (such as open files and signals). Threads in a process are not visible to other processes; 4. Scheduling and switching: Thread context switching is much faster than process context switching.

Intuitive explanation

①A process is like a factory floor, it represents a single task that the CPU can handle. At any time, the CPU is always running one process, and other processes are in a non-running state. ② There can be many workers (threads) in a workshop (process). They work together to complete a mission. ③Threads are like workers in a workshop. A process can include multiple threads. ④The space in the workshop is shared by workers. For example, many rooms can be entered and exited by every worker. This symbolizes that the memory space of a process is shared, and each thread can use these shared memories.
