Detailed explanation of C++ container Set

Programming LanguagesC & C++TechnologyC++
The set collection is an associative container that comes with the C++ STL library. It uses a very efficient balanced search binary tree: red-black tree (RBT). set has the following two characteristics: 1. The elements in the set are all sorted. 2. There are no duplicate elements in the set collectionCommon operations:

Construction, copying, destruction

operate

Effect

set c

Produces an empty set/multiset with no elements

set c(op)

Using op as the sorting criterion, generate an empty set/multiset

set c1(c2)

Produce a copy of a set/multiset, all elements are copied

set c(beg,end)

Generate a set/multiset with all elements in the interval [beg, end)

set c(beg,end, op)

Using op as the sorting criterion, the elements in the interval [beg, end) generate a set/multiset

c.~set()

Destroy all elements and release memory

set<Elem>

Generate a set with (operator <) as the sorting criterion

set<Elem,0p>

Generate a set, using op as the sorting criterion

non-volatile operations

operate

Effect

c.size()

Returns the current number of elements

c.empty ()

Determine whether the size is zero, which is equivalent to 0 == size() and is more efficient.

c.max_size()

Returns the maximum number of elements that can be accommodated

c1 == c2

Determine whether c1 is equal to c2

c1 != c2

Determine whether c1 is not equal to c2 (equal to! (c1==c2))

c1 < c2

Determine whether c1 is less than c2

c1 > c2

Determine whether c1 is greater than c2

c1 <= c2

Determine whether c1 is less than or equal to c2 (equal to! (c2<c1))

c1 >= c2

Determine whether c1 is greater than or equal to c2 (equivalent to!(c1<c2))

Special search functions

Sets and multisets are optimized for fast search of elements and provide special search functions. Therefore, these search functions should be used first to obtain logarithmic complexity instead of linear complexity of STL. For example, when searching for 1000 elements, the logarithmic complexity takes an average of ten times, while the linear complexity requires an average of 500 times.

operate

Effect

count (elem)

Returns the number of elements whose value is elem

find(elem)

Returns the first element whose element value is elem, if not return end()

lower _bound(elem)

Returns the first insertable position where the element value is elem, that is, the first element position where the element value >= elem

upper _bound (elem)

Returns the last insertable position where the element value is elem, that is, the first element position where the element value > elem

equal_range (elem)

Returns the first and last position where elem can be placed, that is, the interval where the element value == elem

Assignment

operate

Effect

c1 = c2

Give all elements of c2 to c1

c1.swap(c2)

Swap the elements of c1 and c2

swap(c1,c2)

Same as above, global function

Iterator related functions

The iterators of sets and multisets are bidirectional iterators. For iterator operations, all elements are treated as constants, which ensures that you will not artificially change the element values ​​to disrupt the established order, so you cannot call variability algorithms such as remove().

operate

Effect

c.begin()

Returns a random access iterator pointing to the first element

c.end()

Returns a random access iterator pointing to the position next to the last element

c.rbegin()

Returns a reverse iterator pointing to the first element of the reverse iteration

c.rend()

Returns a reverse iterator pointing to the position next to the last element of the reverse iteration

Inserting and removing elements

The parameters must be guaranteed to be valid, the iterator must point to a valid position, the starting point of the sequence cannot be located after the end point, and elements cannot be deleted from an empty container.

operate

Effect

c.insert(elem)

Inserts a copy of elem and returns the position of the new element, regardless of whether the insertion was successful or not.

c.insert(pos, elem)

Insert a copy of the elem element and return the position of the new element. POS is the starting point for retrieval, which improves the insertion speed.

c.insert(beg,end)

Insert all elements in the interval [beg, end) into c, no return value.

c.erase(elem)

Delete all elements equal to elem and return the number of elements removed.

c.erase(pos)

Remove the element at the position pointed by the iterator pos, no return value.

c.erase(beg,end)

Removes all elements in the interval [beg, end), no return value.

c.clear()

Remove all elements and empty the container
Example: 
#include<iostream>
#include<set>
//#include<multiset>

using namespace std;

int main()
{
    set<int> s;
    s.insert(1);
    s.insert(2);
    s.insert(3);
    s.insert(1);
    s.insert(4);
    s.insert(5);
    s.insert(6);
    cout<<"SIZE:"<<s.size()<<endl;
    cout<<"max_size:"<<s.max_size()<<endl;
    cout<<"how many 1 in set? "<<s.count(1)<<endl;
    cout<<"lower_bound:"<<*s.lower_bound(-1)<<endl;
    for(set<int>::iterator it=s.begin();it!=s.end();it++){
        cout<<*it<<" ";
    }
    cout<<endl;
    s.erase(1);
    s.erase(s.begin());

    cout<<"//////////////"<<endl;
    for(set<int>::iterator it=s.begin();it!=s.end();it++){
        cout<<*it<<" ";
    }
    cout<<endl;
    s.clear();
    if(s.empty()){
        cout<<"set is empty"<<endl;
    }
    cout<<"SIZE:"<<s.size()<<endl;
    cout<<"max_size:"<<s.max_size()<<endl;
    return 0;
}
