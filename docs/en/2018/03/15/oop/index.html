Encapsulation, inheritance, polymorphism and abstraction

Programming LanguagesTechnologyProgramming
We know that encapsulation can hide implementation details and make code modular; inheritance can extend existing code modules (classes); their purpose is: code reuse. Polymorphism is to achieve another purpose-interface reuse! The function of polymorphism is to ensure the correct call when using a certain attribute of an instance of any class in the "family tree" when the class inherits and derives.

Package:

Encapsulation is the first step in realizing object-oriented programming. Encapsulation is to collect data or functions into units (we call them classes). Encapsulated objects are often called abstract data types.The meaning of encapsulation:The meaning of encapsulation is to protect or prevent code (data) from being inadvertently destroyed by us. In object-oriented programming data is considered a central element and is closely integrated with the functions that use it, thus protecting it from accidental modification by other functions. 1. Protect data members from direct access or modification by programs outside the class. They can only be accessed through the provided public interface ==> data encapsulation. 2. The details of the method are hidden from users. As long as the interface remains unchanged, modification of the content will not affect external callers ==> method encapsulation. 3. When an object contains complete attributes and corresponding methods, it is called encapsulation. 4. The properties of the object cannot be directly accessed from outside the object, but can only be accessed through the method corresponding to the property. 5. The method of the object can receive messages from outside the object. Access modifiers for class members:

That is, the access control characters of class methods and member variables. If a class is not visible as a whole object, it does not mean that all its fields and methods are also invisible to other parts of the program. Their access modifiers need to be judged.

inherit:

Inheritance mainly implements code reuse and saves development time. Inheritance is a method that allows an object of one type to obtain the properties of an object of another type. It supports the concept of hierarchical classification. Inheritance refers to the ability to use all the functionality of an existing class and extend it without having to rewrite the original class. The new class created through inheritance is called a "subclass" or "derived class", and the inherited class is called a "base class", "parent class" or "super class". The process of inheritance is the process from general to special. To achieve inheritance, you can achieve it through "Inheritance" and "Composition". There are two types of ways to implement the concept of inheritance: implementation inheritance and interface inheritance. Implementation inheritance refers to the ability to directly use the properties and methods of a base class without additional coding; interface inheritance refers to the ability to use only the names of properties and methods, but subclasses must provide implementations;

Polymorphism:

1. “One interface, multiple methods” The same operation applied to different objects can have different interpretations and produce different execution results. Three conditions for polymorphism: a. The existence of inheritance (inheritance is the basis of polymorphism, there would be no polymorphism without inheritance). b. The subclass overrides the method of the parent class (calling the method overridden by the subclass under polymorphism). c. The parent class reference variable points to the subclass object (type conversion from subclass to parent class). Overloading and overriding are the two main ways to implement polymorphism. 2. Implement polymorphism: interface polymorphism. Inheritance polymorphism. Polymorphism via abstract classes.

abstract:

It represents an abstract concept derived from the analysis and design of a problem domain. It is an abstraction of a series of specific concepts that look different but are essentially the same. Abstraction is modified with the abstract keyword in Java. When using abstract to modify a class, this class cannot be instantiated. It can be seen from this that abstract classes exist for inheritance. If you define an abstract class without inheriting it, then you have created this abstract class in vain, because you cannot use it to do anything. Use abstract When modifying a method, this method is an abstract method. Abstract methods must exist in abstract classes. Abstract methods have no method body. For a parent class, if one of its methods is implemented in the parent class, it does not make any sense. Different implementations must be carried out according to the actual needs of the subclass. Then this method can be declared as an abstract method. The modifier of the abstract method must be public or protected. If it should be private, it cannot be inherited by the subclass, and the subclass cannot implement the method. By default, it defaults to public.   References:http://blog.csdn.net/c_w_d/article/details/52982191
