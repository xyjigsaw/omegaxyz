Python single and double underline

Programming LanguagesTechnologyPythonPython
__foo__: A convention, an internal name in Python, used to distinguish other user-defined names to prevent conflicts._foo:A convention for specifying that a variable is private. A way for programmers to specify private variables.__foo:This has real meaning: used by the parser_classname__footo replace this name to distinguish it from other classes with the same name. First, it starts with a single underscore. This is commonly used in modules. In a module, variables and functions that start with a single underscore are treated as internal functions by default. If you use from a_module import * to import, these variables and functions will not be imported. However, it is worth noting that if you use import a_module to import a module, you can still access such an object in the form of a_module._some_var. Among the officially recommended code styles of Python, there is also a style that ends with a single underscore, which has no special meaning when parsing, but is usually used to distinguish it from Python keywords. For example, if we need a variable called class, but class is a keyword of Python, we can end with a single underscore and write class_. Naming forms starting with double underscores are used to represent name mangling (Name Mangling) in Python's class members. That is, if there is a member __x in the Test class, then dir(Test) will see _Test__x instead of __x. This is to avoid the member's name conflicting with names in subclasses. But note that this requires that the name does not have an underscore at the end. What starts with a double underscore and ends with a double underscore are some Python "magic" objects, such as __init__, __del__, __add__, __getitem__, etc. for class members, and global __file__, __name__, etc. Python officially recommends that you never apply this naming method to your own variables or functions, but use it according to the documentation. In addition, there is another situation where a single underscore is not generally used. An extension library written in C is sometimes named with an underscore, and then packaged using a Python module with the underscore removed. For example, the struct module is actually a Python wrapper of the C module _struct.
