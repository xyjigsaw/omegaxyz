A/B testing

A statistical method used to compare two or more technologies, usually a currently employed technology versus a new technology. A/B testing is not only designed to determine which technique performs better, but also helps understand whether the difference is statistically significant. A/B testing typically compares two technologies using one measure, but can work with any limited number of technologies and measures.

bucketing

Convert a feature (usuallycontinuousfeatures) into multiple binary features (called buckets or bins), typically based on value intervals. For example, instead of representing temperature as a single continuous floating-point feature, you can split temperature intervals into discrete bins. Assuming that the temperature data is accurate to one decimal place, you can place all temperatures between 0.0 and 15.0 degrees in one bin, all temperatures between 15.1 and 30.0 degrees in a second bin, and all temperatures between 30.1 and 50.0 degrees in a third bin.

calibration layer

A post-forecast adjustment, usually to reduceForecast bias. The adjusted predictions and probabilities should be consistent with the distribution of the observed label set.

candidate sampling

A training-time optimization that uses a function (such as softmax) to calculate probabilities for all positive class labels, but only for random samples of negative class labels. For example, if a sample is labeled "beagle" and "dog", candidate sampling will calculate the predicted probability and corresponding loss term for the "beagle" and "dog" category outputs, as well as a random subset of other categories (cat, lollipop, fence). This sampling is based on the idea that as long aspositive categoryAlways receive appropriate positive reinforcement,negative categoryIt is possible to learn from less frequent negative reinforcement, which is indeed what is observed in practice. The purpose of candidate sampling is to increase computational efficiency by not computing predictions for all negative classes.

Confusion matrices for multi-class classification problems help identify error patterns. The confusion matrix contains sufficient information to calculate various performance metrics, including precision and recall.
