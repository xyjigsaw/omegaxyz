Principle of record semaphore (operating system)

Technology操作系统操作系统原理
In 1965, the Semaphores mechanism proposed by Dutch scholar Dijkstra is an effective process synchronization tool. In the long-term and extensive application, the semaphore mechanism has been greatly developed. It has developed from an integer semaphore through a record semaphore to a "semaphore set" mechanism. Now, the semaphore mechanism has been widely used in single-processor and multi-processor systems and computer networks.

The wait operation in the integer semaphore mechanism will continue to test as long as the semaphore S<=0. Therefore, this mechanism does not follow the "yield waiting" principle, but leaves the process in a "busy waiting" state. The record semaphore mechanism is a process synchronization mechanism that does not have the "busy wait" phenomenon.
However, after adopting the "give and wait" strategy, there will be multiple processes waiting to access the same critical resource. To this end, in the semaphore mechanism, in addition to an integer variable value representing the number of resources, a process list pointer L should also be added to link all the waiting processes mentioned above. The record semaphore is named because it uses a record data structure.
Data structure:
typedef struct 
{
    int value;
    struct process *L;
} semaphore;
The value here has the same meaning as S above, both indicating the number of available resources. The linked list L represents the ready queue, so the core is still PV, but how to treat the process and the record-type semaphore are more complete. Solve the problem of keeping the process busy waiting.
void P(semaphore s)
{
    s.value--; // 申请资源，边界是value已经为0了，那么现在变-1，表示有一个进程在等待
    if(s.value < 0)
    {
        将此进程加入就绪队列，等待；
        block(s.L);
    }
}

void V(semaphore s)
{
    s.value++;
    if(s.value <= 0)
    {
        将进程P从就绪队列中移出；
        wakeup(P);// 唤醒
    }
}
When s.value = 0, it changes from when s.value = -1. Therefore, although s.value = 0, you must not forget to remove the last process from the ready queue and wake it up.
