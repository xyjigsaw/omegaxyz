Maximum correlation minimum redundancy (mRMR) algorithm

TechnologyMachine Learning特征选择信息论
When doing feature selection, you may face two problems: how relevant the features are to category prediction, and how much redundancy there is between features. In feature selection, "the best m features are not necessarily the m best features". From the perspective of correlation and redundancy, the best m features refer to the features most relevant to classification. However, since there may be redundancy between the best m features, the most relevant m features do not necessarily produce better classification accuracy than the other m features. It can be seen that feature selection can be divided into two processes: 1. How to measure feature correlation. 2. How to solve the redundancy between features.

mutual information

feature selection process

The purpose of feature selection is to select a streamlined feature set with good classification effect. In order to achieve this goal, it can be carried out in two steps: the first step is to use mrmr to select a candidate feature set; the second step is to use other methods to select a streamlined feature set.

Main steps:

  • The process of processing and converting data (Note: In order to calculate the joint distribution and marginal distribution of two features, the data needs to be normalized to between [0, 255], and each dimensional feature is stored using a reasonable data structure)
  • Calculate distribution and mutual information between features, features and response variables
  • Perform mrmr scores on features and sort them
