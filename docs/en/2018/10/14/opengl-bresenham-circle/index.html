OpenGL implements Bresenham circle drawing algorithm

Programming LanguagesC & C++Technology计算机图形学C++C计算机图形学openGL

Midpoint Bresenham algorithm

The Bresenham circle drawing algorithm, also known as the midpoint circle drawing algorithm, is the same as the Bresenham straight line algorithm. Its basic method is to use the discriminant variable to judge and select the nearest pixel point. The value of the discriminant variable can be calculated using only some addition, subtraction and shift operations. For the sake of simplicity, consider a circle with the center at the origin of the coordinates, and only calculate the points on the eighth circle, and the other points on the circle can be obtained by using symmetry. Why is it enough to just calculate the points on the eighth circle? Similar to the straight line algorithm above, the circle also has an "eight symmetry", as shown in the figure below.Obviously, we only need to know the coordinates (x, y) of one point on the circle. Using eight symmetries, we can immediately get the coordinates of the other seven symmetry points. Similar to the straight line algorithm, the Bresenham circle drawing algorithm also uses a series of discrete points to approximately describe a circle, as shown in the figure below.Consider one-eighth arc of a circle with a center at the origin and radius R in the first quadrant. Determine this arc in a clockwise direction from point (0, R) to point (R', R'). Suppose a certain point Pi(xi, yi) is already the point on the arc closest to the actual arc, then the next point of Pi can only be one of P1 on the right or P2 on the lower right, as shown in the following figure:

Construct the discriminant function:

F(x, y）= x2 + y2 – R2

When F(x, y) = 0, it means the point is on the circle. When F(x, y) > 0, it means the point is outside the circle. When F(x, y) < 0, it means the point is inside the circle. if M is P1and P2The midpoint of M, then the coordinates of M are (xi + 1, yi– 0.5), when F(xi + 1, yi– 0.5) < 0, point M is within the circle, indicating that P1The point is closer to the actual arc and should be P1as the next point of the circle. Analyzing in the same way, when F(xi + 1, yi– 0.5) > 0, P2Closer to the actual arc, P should be chosen2as the next point. When F(xi + 1, yi– 0.5) = 0, P1and P2can be used as the next point of the circle, and the algorithm agrees to take P2as the next point. Now set the coordinates of point M (xi + 1, yi– 0.5) Bring in the discriminant function F(x, y) to get the discriminant d:

d = F（xi + 1, yi – 0.5）= (xi + 1)2 + (yi – 0.5)2 – R2

If d < 0, then take P1is the next point, at this time P1The discriminant of the next point is:

d’ = F（xi + 2, yi – 0.5）= (xi + 2)2 + (yi – 0.5)2 – R2

After expansion, d is brought into the recursion relationship to obtain the discriminant:

d’ = d + 2xi + 3

If d > 0, then take P2is the next point, at this time P2The discriminant of the next point is:

d’ = F（xi + 2, yi – 1.5）= (xi + 2)2 + (yi – 1.5)2 – R2

After expansion, d is brought into the recursion relationship to obtain the discriminant:

d’ = d + 2(xi - yi) + 5

In particular, at the first point (0, R) of the first quadrant, the initial value d of the discriminant d can be deduced0：

d0 = F(1, R – 0.5) = 1 – (R – 0.5)2 – R2 = 1.25 - R

Improved midpoint Bresenham algorithm

In the midpoint circle drawing method, floating point operations are used to calculate the discriminant d, which affects the efficiency of circle generation. If the discriminant can be reduced to integer operations, calculations can be simplified and efficiency improved. So people have made various improvements to the midpoint circle drawing method. One of the ways is to change the initial value of d by1.25 – RChange to1 – R, considering that the radius R of the circle is always greater than 2, this modification will not affect the sign of the initial value of d, and floating point operations can be avoided.

// Bresenham_circile.cpp : 定义控制台应用程序的入口点。
#include<stdlib.h>
#include <GL/glut.h>

/* initialization: */
void myinit(void)
{

	/* attributes */

	glClearColor(1.0, 1.0, 1.0, 0.0); /* white background */
	glColor3f(1.0, 0.0, 0.0); /* draw in red */

	/* set up viewing: */
	/* 500 x 500 window with origin lower left */

	glMatrixMode(GL_PROJECTION);
	glLoadIdentity();
	gluOrtho2D(0.0, 500.0, 0.0, 500.0);
	glMatrixMode(GL_MODELVIEW);
}
void  plot_circle_points(int xc, int yc, int x, int y)
{
	glBegin(GL_POINTS);
	glVertex3f(xc + x, yc + y, 0);
	glVertex3f(xc - x, yc + y, 0);
	glVertex3f(xc + x, yc - y, 0);
	glVertex3f(xc - x, yc - y, 0);
	glVertex3f(xc + y, yc + x, 0);
	glVertex3f(xc - y, yc + x, 0);
	glVertex3f(xc + y, yc - x, 0);
	glVertex3f(xc - y, yc - x, 0);
	glEnd();
}
void drawcircle(int xc, int yc, int radius)
{
	int x, y, p;
	x = 0;
	y = radius;
	p = 3 - 2 * radius;
	glClear(GL_COLOR_BUFFER_BIT);
	glBegin(GL_POINTS);
	while (x<y)
	{
		plot_circle_points(xc, yc, x, y);
		if (p<0)
			p = p + 4 * x + 6;
		else
		{
			p = p + 4 * (x - y) + 10;
			y -= 1;
		}
		x += 1;
	}
	if (x == y)
		plot_circle_points(xc, yc, x, y);
}


/* the display callback: */
void display(void)
{
	glClear(GL_COLOR_BUFFER_BIT);  /*clear the window */

	/*----------------------------------------*/
	/*  viewport stuff                        */
	/*----------------------------------------*/
	/* set up a viewport in the screen window */
	/* args to glViewport are left, bottom, width, height */
	glViewport(0, 0, 500, 500);
	/* NB: default viewport has same coords as in myinit, */
	/* so this could be omitted: */


	drawcircle(200, 200, 100);

	/* and flush that buffer to the screen */
	glFlush();
}

int main(int argc, char** argv)
{

	/* Standard GLUT initialization */

	glutInit(&argc, argv);
	glutInitDisplayMode(GLUT_SINGLE | GLUT_RGB); /* default, not needed */
	glutInitWindowSize(500, 500); /* 500 x 500 pixel window */
	glutInitWindowPosition(0, 0); /* place window top left on display */
	glutCreateWindow("Bresenham  circile"); /* window title */
	glutDisplayFunc(display); /* display callback invoked when window opened */
	myinit(); /* set attributes */
	glutMainLoop(); /* enter event loop */
}
