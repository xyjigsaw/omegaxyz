Compiled, interpreted and scripting languages

Programming LanguagesTechnologyProgramming
Compiled languages вҖӢвҖӢhave a separate compilation process before the program is executed, and there is no need to translate the program into machine language. Interpreted languages вҖӢвҖӢtranslate programs into machine language when running, so the running speed is compared to programming. C/C++, etc. are all compiled languages, while Java, C#, etc. are all interpreted languages. Although a Java program also has a compilation process before running, the program is not compiled into machine language, but into bytecode (which can be understood as an intermediate language). At run time, the JVM translates the bytecode into machine language. Scripting languages вҖӢвҖӢgenerally have corresponding script engines to interpret and execute. They generally require an interpreter to run. JAVASCRIPT, ASP, PHP, PERL, and Nuva are all scripting languages. After C/C++ is compiled and linked, an independently executed exe file can be formed.

compiled language

  1. One of the biggest advantages of compiled languages вҖӢвҖӢis their execution speed. Programs written in C/C++ run 30%-70% faster than the same programs written in Java.
  2. Compiled programs consume less memory than interpreted programs.
  3. The downside - compilers are much harder to write than interpreters.
  4. Compilers don't provide much help when debugging a program - how many times have you encountered a "null pointer exception" in your C code and spent hours trying to figure out where the error was in the code.
  5. Executable compiled code is much larger than the same interpreted code. For example, a C/C++ .exe file is much larger than a Java .class file with the same functionality.
  6. Compiled programs are platform-specific and therefore platform-dependent.
  7. Compiled programs do not allow security to be implemented in the code - for example, a compiled program can access any area of вҖӢвҖӢmemory and do anything it wants to your PC (most viruses are written using compiled languages)
  8. Due to loose security and platform dependencies, compiled languages вҖӢвҖӢare not suitable for developing Internet or Web-based applications.

Interpreted languages вҖӢвҖӢ(including scripting languages)

  1. Interpreted languages вҖӢвҖӢprovide excellent debugging support. It only takes a few minutes for a Java programmer to locate and fix a "null pointer exception" because the Java runtime environment not only indicates the nature of the exception, but also gives the specific line number and function call sequence where the exception occurred (the famous stack trace information). Such convenience is not provided by compiled languages.
  2. Another advantage is that interpreters are easier to implement than compilers
  3. One of the biggest advantages of interpreted languages вҖӢвҖӢis their platform independence
  4. Interpreted languages вҖӢвҖӢcan also ensure a high degree of security - something that Internet applications urgently need
  5. The size of the intermediate language code is much smaller than the compiled executable code
  6. Platform independence, and tight security are the two most important factors that make interpreted languages вҖӢвҖӢideal for Internet and Web applications.
  7. Interpreted languages вҖӢвҖӢhave some serious drawbacks. Interpreted applications take up more memory and CPU resources. This is because, in order to run a program written in an interpreted language, the associated interpreter must first be run. Interpreters are complex, intelligent, resource-intensive programs and they take up a lot of CPU cycles and memory.
  8. Interpreted applications are much slower than compiled programs due to their decode-fetch-execute cycle.
  9. The interpreter also does a lot of code optimization and runtime security checks; these extra steps take up more resources and further slow down the application.

scripting language

Definition: A computer programming language created to shorten the traditional edit-compile-link-run process. Its characteristics are: the program code is the final execution file, but this process requires the participation of the interpreter, so scripting languages вҖӢвҖӢare closely related to interpreted languages. Scripting languages вҖӢвҖӢare usually interpreted and programs are text files. Typical scripting languages вҖӢвҖӢinclude JavaScript, Python, etc. Corresponding to scripting language is system language. Their differences are: 1. Level of abstraction: This is the most important and obvious difference. Scripting languages вҖӢвҖӢprovide higher-level abstractions to programmers. This is evident in the fact that in the language itself, there are advanced data structures, such as list and dictionary structures, as well as simple and convenient nesting and operations on such structures. This can create a very successful program. 2. Type definition: System languages вҖӢвҖӢare usually strongly typed and static type defined. This means that the types of all variables must be specified in the program and checked at compile time. In contrast, scripting languages вҖӢвҖӢare the loosest type definitions, with no type declarations at all, and dynamic type checking at runtime. 3. Execution: The system language is characterized by compilation. Programs are compiled into executable binaries. On the other hand, scripting languages вҖӢвҖӢare characterized by interpretation, that is, instructions are executed immediately without an intermediate state of compilation. This means that the scripting language is interactive (you can type commands at the prompt and see the results), which is another huge win. This completely removes the compilation process from the edit-compile-run cycle. 4. Speed: The above three points are typical examples of the compromise between speed and efficiency of scripting languages вҖӢвҖӢon the one hand, and ease of use and power of expressions on the other. This makes the execution speed of script languages вҖӢвҖӢan order of magnitude slower than system languages. This is the most maligned aspect of scripting languages. The performance penalty is not really a problem because the idea is to use a scripting language to assemble components that are written in a fast systems language like C. So everything that needs to run fast will run fast because it's implemented in a fast language. Scripting languages вҖӢвҖӢare only used to tie things together, and these are usually not performance bottlenecks (or if they are, you need to re-examine your design).The difference between interpreted languages вҖӢвҖӢand scripting languagesScripting language is an interpreted language, that is, interpreted language includes scripting language. An interpreted language refers to a language that is interpreted and executed, but the executed code is not necessarily in text format.
