Java Data Access Object (DAO)

Programming LanguagesTechnologyJavaDatabasesJavaSQL

What is DAO

DAO (Data Access Object), as the name suggests, is an object that provides an abstract interface for a database or other persistence mechanism. It provides various data access operations without exposing the implementation details of the underlying persistence solution. In actual development, all access operations to the data source should be abstracted and encapsulated in a public API. In terms of programming language, it is to establish an interface, which defines all transaction methods that will be used in this application. In this application, this interface is used when interacting with the data source, and a separate class is written to implement this interface, which logically corresponds to a specific data storage. The DAO pattern actually includes two patterns, one is Data Accessor, and the other is Data Object. The former solves the problem of how to access data, while the latter solves the problem of how to encapsulate data with objects.

The role of DAO model

1Isolate business logic code and data access code 2.Isolate the implementation of different databases Business logic layer, data access layer (Oracle, SQLServer, MySQL)

Components of the DAO pattern

  • DAO interface
  • DAO implementation class
  • Entity class
  • Database connection and shutdown tool class
