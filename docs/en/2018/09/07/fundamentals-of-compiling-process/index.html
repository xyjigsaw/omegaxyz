Compilation process

The working process of a compiler can generally be divided into 6 stages: 1. Lexical analysis 2. Syntax analysis 3. Semantic analysis 4. Intermediate code generation 5. Optimization 6. Target code generation

Lexical analysis

The task of lexical analysis is to input the source program, scan and decompose the strings that make up the source program, and identify each word (definition, identifier, operator, delimiter, constant). What is followed in the lexical analysis stage is the grammatical rules of the language (or word formation rules). Effective tools for describing grammar rules are formal forms and finite automata.

syntax analysis

The task of grammatical analysis is: on the basis of lexical analysis and according to the grammatical rules of the language, decompose the word symbol string into various grammatical units (grammatical categories) (phrases, clauses, sentences, program segments, programs), and determine whether the entire input string constitutes a grammatically correct program. Grammatical analysis follows the grammatical rules of the language. Grammar rules are usually described using context-free grammars. Lexical analysis is a linear analysis, while syntactic analysis is a hierarchical analysis.

Semantic analysis and intermediate code generation

The task at this stage is to analyze the meaning of various grammatical categories identified by syntax analysis and perform preliminary translation (generating intermediate codes). This stage usually includes two aspects of work. 1. Perform static semantic checks on each syntax category, for example, whether the variables are defined, whether the type is correct, etc. 2. If the semantics are correct, translate the intermediate code. This stage follows the semantic rules of the language, and attribute grammar is usually used to describe the semantic rules.

optimization

Process and transform the code (mainly intermediate code) in order to produce more efficient (time and space saving) target code. The main aspects of optimization include: extraction of common subexpressions, loop optimization, deletion of useless code, etc. Optimization follows the equivalent transformation rules of the program.

Object code generation

The task of this stage is to convert the intermediate code (after optimization) into low-level language code (absolute instructions, relocatable instructions, assembly instructions) on a specific machine.
