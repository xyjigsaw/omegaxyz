Human-computer interaction - EEC model

TechnologyAPP开发Software EngineeringappGUISoftware EngineeringHCI
A framework is a structure that provides an understanding or definition of something.

What is an interactive framework?

  • ①Help define the interaction design process
  • ②Identify the main problems in the design process
  • ③Help define problem domain
 

Execution/evaluation activity cycle EEC model

Basic components (structure) of activities
  • ①Goal: What you want to do, this is the most important
  • ②Execution: the operations that must be performed to achieve the goal
  • ③Objective factors (World): Objective factors that need to be considered when executing activities
  • ④Evaluation: Measuring the gap between the results of activity execution and the goals
  Note:
  • ① Goals are not intentions. There may be two intentions between one goal. Similarly, each intention can include a series of activities.
  • ②The goal does not describe the activities in detail
The interaction model is as aboveThere are seven stages:
  • ①Establish goals
  • ②Form operation intention
  • ③Clear the action sequence
  • ④Execute action
  • ⑤Perception of system status
  • ⑥Explain system status
  • ⑦Evaluate system status against goals
Among them, ① to ④ are executed, and ⑤ to ⑦ are evaluated. Note that new operations can be performed based on ⑦.

The EEC model can explain some problems with the use of interfaces

Enforcement barriers

Definition: The difference between the actions specified by the user to achieve a goal and the actions allowed by the system. Simply put, it is whether your interface can perform the functions that users want.

Assessing the gap

Definition: The difference between the actual performance of the system state and the user's expectation. For example, mode phenomena will cause evaluation gaps, upper and lower case CapsLock input method switching, etc.Disadvantages: Cannot describe the communication between humans and systems through the interface.

Extended EEC framework

The interactive system mainly consists of four parts
  • ① System (S): Use the system’s kernel language (computing attributes related to the system state)
  • ②User (U): Use the user’s task language (psychological attributes related to the user’s state)
  • ③Input (I): Use input language
  • ④Output (O): Use output language
The interaction loop consists of two phases:① Execution stage: definition, execution, performance. Designers should ensure that translation from input to system is easy
  • Definition: A user articulates a goal, which is then coordinated and connected through input language
  • Execution: Convert from input language to kernel language (operations to be performed by the system)
  • Performance: Use output language to express the execution results of the kernel language.
②Evaluation stage
  • Observation: The interactive process introduces iterative feedback loops, whereby vague goals become clear through alternating approaches and refinements.
