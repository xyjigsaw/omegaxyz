Detailed explanation of One-Hot coding and Python implementation

Programming LanguagesTechnologyMachine LearningPythonPython
The basic idea of ​​one-hot: Treat each value of a discrete feature as a state. If you have N different values ​​in this feature, then we can abstract the feature into N different states. One-hot coding ensures that each value will only make one state "activated", which means that only one of the N states has a state bit value of 1, and the other state bits are 0. For example, assuming we take browsers as an example, the category we want to study is
  • Browser: ["Firefox", "Chrome", "Safari", "Internet Explorer"]
There are 4 categories in total. If we use one-hot to encode them, we will get: Firefox: [1, 0, 0, 0] Chrome: [0, 1, 0, 0] Safari: [0, 0, 1, 0] IE: [0, 0, 0, 1] If multiple features appear For example:
  • Gender: ["male", "female"]
Then the encoding for the instance (Firefox, female) is [1, 0, 0, 0, 0, 1], that is, the One-Hot encoding of the two categories is assembled together.Of course, the disadvantage of this is that the encoding vector is too sparse.Python implementation:
browser = ["Firefox", "Chrome", "Safari", "Internet Explorer"]
gender = ["male", "female"]


def onehotEncoding(instance1, instance2, class1, class2):
    temp1, temp2 = [0] * len(class1), [0] * len(class2)
    temp1[class1.index(instance1)] = 1
    temp2[class2.index(instance2)] = 1
    return temp1 + temp2

for i in browser:
    for j in gender:
        print(onehotEncoding(i, j, browser, gender))
