Introduction to TensorFlow

It is used in many fields of machine deep learning such as speech recognition or image recognition, and it can run on a variety of devices ranging from a smartphone to thousands of data center servers.

Why learn TensorFlow?

TensorFlow was originally developed by researchers and engineers at the Google Brain Group (part of Google's machine intelligence research organization) for research in machine learning and deep neural networks, but the system's versatility makes it applicable to a wide range of other computing fields. TensorFlow can undoubtedly be regarded as one of the most useful libraries in neural networks. The task it is good at is training deep neural networks. By using TensorFlow, we can quickly get started with neural networks, greatly reducing the development cost and difficulty of deep learning (that is, deep neural networks); the open source nature of TensorFlow allows everyone to use, maintain, and consolidate it, so that it can be quickly updated and developed. Although some people may saycaffebetter suited for images,mxnetMore efficient and so on, but in fact these frameworks are all the same, only the syntax is different, so we don't have to worry too much here. So let's start with tensorflow.

Computational graph

TensorflowPrograms are usually organized into a build phase and an execution phase. In the build phase,opThe execution steps are described as a graph. In the execution phase, sessions are used to execute theop. For example, it is common to create a graph to represent and train a neural network during the build phase, and then iteratively perform the training on the graph during the execution phaseop。 TensorflowSupport C/C++, python programming language. at present,TensorFlowThe python library is easier to use and provides a large number of helper functions to simplify the work of building graphs, which are not yet supported by the C/C++ library. Start a graph in a session

The diagram cannot be started until the construction phase is complete. The first step in starting a graph is to create aSessionObject without any creation parameters, the session constructor will fail to start the default graph. For the complete session API, please readSessionkind。 SessionObjects need to be closed to release resources after use, unless explicitly calledcloseIn addition, you can also usewithCode to automatically complete the closing action:
with tf.Session() as sess:
  result = sess.run([product])
  print result
In terms of implementation,TensorflowConvert graph definitions into operations that are executed in a distributed manner to fully utilize available computing resources (such as CPU or GPU). Generally you don't need to explicitly specify whether to use CPU or GPU.TensorflowCan be detected automatically. If GPU is detected,TensorflowWhenever possible, the first GPU found will be used to perform the operation. If there is more than one available GPU on the machine, the other GPUs except the first one are not involved in the calculation. in order to letTensorflowTo use these GPUs you have toopExplicitly assign them to perform.with...DeviceStatements are used to assign specific CPU or GPU operations:
with tf.Session() as sess:
  with tf.device("/gpu:1"):
    matrix1 = tf.constant([[3., 3.]])
    matrix2 = tf.constant([[2.], [2.]])
    product = tf.matmul(matrix1, matrix2)
Devices are identified with strings. Currently supported devices include:
  • /cpu:0:Machine’s CPU
  • /gpu:0:The machine's first GPU, if one is available
  • /gpu:1:The second GPU of the machine, and so on

variable

existVariablesSee more details in . Variables maintain status information during graph execution. The following example demonstrates how to use variables to implement a simple counter:
# 创建一个变量，初始为标量0
state = tf.Variable(0, name="counter")

# 创建一个op，其作用是使`state`增加1
one = tf.constant(1)
new_value = tf.add(state, one)
update = tf.assign(state, new_value)

# 启动图后，变量必须先经过init op初始化
# 首先先增加一个初始化op到图中
init_op = tf.initialize_all_variables()

# 启动图
with tf.Session() as sess:
  # 运行init op
  sess.run(init_op)
  # 打印 state 的初始值
  print sess.run(state)
  # 运行op， 更新state 并打印
  for _ in range(3):
    sess.run(update)
    print sess.run(state)

# 输出：
# 0
# 1
# 2
# 3
in codeassign()An operation is part of the expression described by the graph, asadd()The operation is the same, so callingrun()It does not actually perform the assignment until the expression is executed. It is common to express the parameters in a statistical model as a set of variables. For example, you can store the weights of a neural network as a variable in atensormiddle. During the training process, by repeatedly training the graph, update thistensor。
