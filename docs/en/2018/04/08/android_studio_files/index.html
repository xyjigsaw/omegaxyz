About Android Studio project directory structure analysis

TechnologyAPP开发Android
There are many directories in the project after Android Studio automatically generates the first Hello World file. Similarly, other project files will also include these directories for the hello world program. The following uses Hello Android as a prototype to introduce the directory structure.It starts with the Android file framework. Change the directory framework to Project for easier analysis (the project shown in the upper row of dark blue) Files and directories that do not require excessive care are ① .gradle ②.idea ③build, etc.The directories that need attention are app. This directory stores the code and resources of your App.The files and directories to pay attention to in the app directory are: ①libs: If you use third-party jar packages in your project, you need to place these jar packages in this directory. The jar packages placed in this directory will be automatically added to the build path. ②The res directory in src contains all images, layouts, strings and other resources. ③Place pictures in drawable, place layout under layout, and put strings under values. Note that layout can be visually laid out and the layout code is automatically generated, which is much more convenient than hand-written code. The picture below is the activity_example.xml file in the layout④AndroidManifest.xml is the entry file of the Android application. It describes the components exposed in the package (activities, services, etc.), their respective implementation classes, various data that can be processed, and the startup location. In addition to declaring Activities, ContentProviders, Services, and Intent Receivers in the program, permissions and instrumentation (security control and testing) can also be specified. ⑤The folders starting with mipmap are all used to store application icons. The reason why there are so many folders is to be compatible with multiple devices. ⑥The JAVA directory contains the source code of the program. Reference: Guo Lin. The first line of code [M]. People's Posts and Telecommunications Publishing House, 2014.
