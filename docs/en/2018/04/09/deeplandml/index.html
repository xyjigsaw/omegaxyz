Comparison between deep learning and classic machine learning

In recent years, deep learning has become the technology of choice for most AI problems, dwarfing classic machine learning. The reason is obvious, deep learning has shown excellent performance on many tasks such as speech, natural language, vision and games. However, despite such good performance of deep learning, classical machine learning methods still have some advantages, and in some specific cases it is better to use classical machine learning methods, such as linear regression or decision trees, instead of using a large deep network. This article will compare deep learning and classic machine learning, introducing the advantages and disadvantages of each technology and the problems/how they are best used.

①Why deep learning is better than classic machine learning

Best-in-class performance:In many fields, deep networks have achieved accuracy far exceeding that of classical ML methods, including speech, natural language, vision, games, etc. In many tasks, classic ML methods cannot even compare with deep learning. For example, the figure below shows the image classification accuracy of different methods on the ImageNet dataset; blue represents the classic ML method and red represents the deep convolutional neural network (CNN) method. The classification error rate of deep learning methods is much lower than that of classical ML methods.
Efficient scaling with data:Compared to classic ML algorithms, deep networks scale better if there is more data. The picture below is a simple example. Many times, the best advice for improving accuracy with deep networks is to use more data! But when using classic ML algorithms, this quick and simple method has little effect, and more complex methods are usually required to improve accuracy.
No feature engineering required:Classic ML algorithms often require complex feature engineering. Typically, exploratory data analysis needs to be performed on the dataset first. Then, the dimensionality can be reduced for easier processing. Finally, the best features must be carefully selected to pass to the ML algorithm. When using deep learning, such feature engineering is not needed because good performance is usually achieved immediately by simply passing the data directly to the network. This completely eliminates the arduous and challenging feature engineering phase of the entire process.Adaptable and easy to migrate:Compared with classic ML algorithms, deep learning technology can be more easily adapted to different fields and applications. First, transfer learning can make pre-trained deep networks effective for different applications in the same domain. For example, in computer vision, pretrained image classification networks are often used as feature extraction front-ends for object detection and segmentation networks. Using these pre-trained networks as front-ends simplifies training of the entire model and often helps achieve higher performance in less time. In addition, the basic ideas and techniques used in deep learning in different fields are often transferable. For example, once you understand the basic theory of deep learning in the field of speech recognition, it is not too difficult to learn how to apply deep networks to natural language processing because the basic knowledge required for both is very similar. But this is not the case for classic ML, as building high-performance ML models requires domain-specific and application-specific ML techniques and feature engineering. The knowledge base of classical ML is very different for different domains and applications, and often requires extensive specialized research in each individual domain.

②Why classic machine learning is better than deep learning

Works better on small data:To achieve high performance, deep learning requires very large data sets. The pre-trained network mentioned earlier was trained on 1.2 million images. For many applications, such large data sets are not readily available, expensive, and time-consuming. For smaller data sets, classic ML algorithms often outperform deep learning.Financially and computationally cheaper:There is a large amount of data and it needs to be trained within a reasonable time. Deep learning requires the use of high-end GPUs. These GPUs are very expensive, but without them it is difficult to implement high-performance deep networks. To effectively use such a high-end GPU, you also need a fast CPU, SSD storage, and fast and large RAM. Classic ML algorithms only require a decent CPU to train well and do not require the best hardware. Because they are computationally inexpensive, they can iterate faster and try many different techniques in a shorter amount of time.Easier to explain:Since classical ML involves direct feature engineering, these algorithms are easy to explain and understand. In addition, it is easier to tune parameters and change the model design because we have a deeper understanding of the data and underlying algorithms. On the other hand, deep learning is a "black box" and even now, researchers cannot fully understand the "inside" of deep networks. Hyperparameters and network design are also a big challenge due to the lack of theoretical foundation.
