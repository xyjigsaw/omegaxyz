Python implementation of sleep sorting, counting sorting and monkey sorting

Programming LanguagesTechnologyPythonPython

sleep sorting

The inventor of sleep sequencing is really a man with a lot of imagination!Thought:Construct n threads, which correspond to these n numbers one-to-one. After initialization, the threads start to sleep and wake up after the corresponding number of time units. Then output its corresponding number. In this way, the thread corresponding to the smallest number wakes up earliest, and this number is output first. When all threads wake up, sorting is completed.accomplish:Using python multi-threading module threading Simple sleep sorting cannot sort negative numbers. In order to make it sort negative numbers, I use an exponential function to map the number to be sorted to a number greater than 0, that is, pow(1.1, num) Note that the larger the base, the longer the sorting time. Therefore, I added a coefficient of 0.01 before the time. Of course, if the coefficient is too small, the threadparallelThis will lead to sorting errors.Time complexity:It has a best time complexity of O(n) and a worst-case complexity of infinity, since this constant may be much larger than n.Code: 
import time
import threading


def event(func):
    co = 0.02
    time.sleep(co * pow(1.1, float(func)))
    print(func)

num = [2, 4, 9, 7, -1, 6, 7, 0, 8, -3, 3, 5, 20]
positive = []
for i in range(len(num)):
    temp = threading.Thread(target=event, args=(str(num[i]),))
    positive.append(temp)

if __name__ == '__main__':
    for t in positive:
        t.start()

counting sort

1. Counting sorting is a very fast and stable sorting method with a time complexity of O(n+k), where n is the number of numbers to be sorted and k is the maximum value of the numbers to be sorted. Counting sort is very fast when sorting a certain amount of integers, generally faster than other sorting algorithms. However, counting sorting has great limitations and is limited to sorting integers. Counting sorting is a fast sorting method that consumes space complexity. Its space development degree is O(K). Similarly, K is the maximum value to be sorted. 2. The basic idea of ​​counting sorting is to count the number of other numbers in the set of numbers that are smaller than this number before sorting a set of numbers, and then the position of this number can be determined. For example, the numbers to be sorted are 7 4 2 1 5 3 1 5; then there are 7 numbers smaller than 7. All 7s should be in the eighth position of the sorted sequence. Similarly, 3 is in the fourth position. For repeated numbers, 1 is in the 1st and 2nd positions (tentatively, the first 1 is smaller than the second 1), and 5 and 1 are in the 6th and 7th positions.Simple non-perfect counting sorting implementation (I was lazy and didn’t count the number of duplicates): 
num = [2, 4, 9, 7, 1, 6, 7, 8, 3, 5]

result = [None] * (len(num)-1)

for i in range(len(num)):
    result[num[i]-1] = num[i]

print(result)

Monkey sorting (Bogo sorting, quantum sorting)

Bogo sort (Bogo-sort), also known as monkey sort, is a spoof sorting algorithm. Its algorithm randomly shuffles the elements, and then closely checks whether they conform to the sorting order. If not, continue to randomly shuffle and continue to check the results until they match the sorting order. The worst time complexity of Bogo sorting is O(∞), and the sorting result cannot be output in a lifetime, and the average time complexity is O(n·n!). However, there is a seemingly joke method that claims to be able to achieve Bogo sorting in O(n). According to the parallel universe interpretation of quantum theory, elements are randomly rearranged using quantum randomness. Different possibilities will unfold in different universes. There is always a possibility that the monkey got the correct order. After the quantum computer finds this universe, it starts to destroy other universes that are unsuccessful in sorting, leaving a universe in the correct order that observers can see. If you want to take the first step towards this seemingly absurd but extremely exciting "efficient algorithm", please first prove the correctness of the "parallel universe explanation".Code: 
import random


def monkeySort(num):
    for i in range(len(num)):
        if i+1 == len(num):
            return False
        if num[i] > num[i+1]:
            return True
    return False

count = 0
num = [2, 6, 7, 8, 3, 5]
while monkeySort(num):
    random.shuffle(num)
    count += 1
print(num)
print(count)
