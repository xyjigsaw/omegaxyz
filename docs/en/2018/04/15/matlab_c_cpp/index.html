MATLAB calls C/C++

Programming LanguagesC & C++TechnologyMATLABC++CMATLAB
Matlab is a matrix language. If the operation can be implemented using matrices, its operation speed is very fast. But if a large number of loops are involved in the operation, the speed of Matlab will be unbearable. When a for loop must be used and the corresponding matrix operation cannot be found to be equivalent, the time-consuming function can be implemented in C language and compiled into a Mex file. Matlab can then call the function written in C like a built-in function. The Mex file is actually a dynamic link library. Older versions of Matlab can directly call .dll, while new versions need to call .mexw32 or .mexw64 files. The compilation process requires a C language compiler. Type mex –setup in Matlab to install and configure.Source code composition of MEX file: (1) Function subroutine. This process contains the code of the Mex file to implement the calculation function, which is a standard C language subroutine. (2) Entry subroutine. This process provides the interface between the functional subroutine and Matlab, implemented with the mexFunction function.Note that the name of the entry process must be mexFunction and contain four parameters, namely void mexFunction(int nlhs,mxArray*plhs[],int nrhs,const mxArray *prhs[])；nrhs(left hand side): the number of input parameters; prhs is an input array whose content is a pointer, pointing to mxArray type data (all data in MATLAB is saved in the form of a matrix mxArray). nlhs, plhs have similar meanings. Specifically, if [a,b]=test(c,d,e) is executed in Matlab, then nlhs=2, nrhs=3, prhs[0] points to c, prhs[1] points to d, prhs[2] points to e (can be understood as: prhs[0]=&c, prhs[1]=&d, prhs[2]=&e), note that prhs is a const pointer array, so the content pointed to cannot be changed; when the function returns, the contents pointed to by plhs[0] and plhs[1] are assigned to a and b (can be understood as a=*plhs[0], b=*plhs[1]).Example:C language function is established as above.
#include "mex.h"    
double add(double x, double y)  
{  
    return x + y;  
}   
void mexFunction(int nlhs,mxArray *plhs[], int nrhs,const mxArray *prhs[])  
{  
    double * a;  
    double b, c;  
    plhs[0] = mxCreateDoubleMatrix(1, 1, mxREAL);  
    a = mxGetPr(plhs[0]);  
    b = *(mxGetPr(prhs[0]));  
    c = *(mxGetPr(prhs[1]));  
    *a = add(b, c);  
}
Next, enter in the MATLAB console:
mex add.c
Format: mex C file.c At this time, the path directory appears:Finally, call the add function in the MATLAB console.add(2,3)Of course, this simple function cannot reflect the speed of C language. When there are for loops or large calculation functions, calling C language becomes very efficient.References:https://blog.csdn.net/u010839382/article/details/42463237
