Programming Language Chronicles

Programming LanguagesTechnologyProgramming
There are thousands of programming languages, but only about 10 are popular. Who created the programming languages ​​we often use and when?1800Joseph Marie Jacquard taught a loom to read punched cards, creating the first highly multi-threaded processing unit. His invention met with fierce opposition from textile workers who had foreseen the birth of Skynet.1842Ada Lovelace (daughter of the English poet Lord Byron) was disgusted by the nobility and scrawling of the notebooks that were later recognized as the first computer programs to be released, because, slightly inconveniently, there were no computers yet.1936Alan Turing invented everything, but the British court did not recognize it and sentenced him to chemical castration. The queen came to forgive him, but unfortunately he was long dead by then.1936Alonzo Church (an important founder of algorithm theory), like Turing, also invented everything, but he was from the other side and was not castrated.1957John Backus created FORTRAN, the first language used by real programmers.1959Grace Hopper invented the first business-oriented programming language for enterprises, and called this language "common business-oriented language", or COBOL for short.1964John Kemeny and Thomas Kurtz believed that programming was too difficult and needed to return to its roots. They called their programming language BASIC.1970Niklaus Wirth developed several languages, the last one that became popular was PASCAL. He loves developing languages. He also invented Wirth's Law, which made Moore's Law obsolete (software slows down faster than hardware gets faster) because software developers would write bloated software that even mainframes couldn't keep up with. This proved to be true later - after Electron.js was invented.1972Dennis Ritchie got bored while working at Bell Labs, so he decided to write the C language with curly braces. This language was a huge success. He then added developer-friendly features such as segmentation faults to help improve productivity. After tinkering with the language, he still had time, so he and his guys at Bell Labs decided to make a demonstration program for the C language. As a result, they came up with an operating system, UNIX.1980Alan Kay invented an object-oriented language, which he called Smalltalk. In Smalltalk, everything is an object, even an object is an object. No one really knows what small talk means.1987Larry Wall had religious experience, became a pastor, and turned Perl into a doctrine.1983Jean Ichbiah noticed that Ada Lovelace's program had never actually run, and decided to develop a language in her name, but the language still did not run.1986Brac Box and Tol Move decided to make an unreadable version of the C language based on Smalltalk. They called the language Objective-C, but no one could figure out its syntax.1983Bjarne Stroustrup went back to the future and noticed that C didn't take enough time to compile, so he added every feature he could think of and called it C++. Programmers everywhere embraced it because it made them look more sincere in their excuses to watch Cats and Dogs videos and xkcd comics while working.1991Guido van Rossum didn't like curly braces, so he invented Python. His grammatical choices were inspired by Monty Python and Flying Circus.1993Roberto Ierusalimschy and his friends thought they needed a scripting language localized in Brazil. An error occurred during localization that would cause the pointer to start counting from 1 instead of 0. They called the language Lua.1994Rasmus Lerdorf built a template engine for the CGI script of his personal homepage, and later put all his information online. The world decided to use these things for everything, so Rasmus hastily threw in some database bindings and called the language PHP.1995Yukihiro Matsumoto was not very happy because he noticed that other programmers were not very happy. He created Ruby to make programmers happy. After he created Ruby, "Matz" was happy, the Ruby community was happy, and everyone was happy.1995Brendan Eich spent his weekends designing a language that he plans to use to power every major browser in the world and eventually Skynet. He first went to Netscape and said the language was called LiveScript, but during the code review period Java became popular, so they decided it was better to use curly braces and renamed it JavaScript. It turned out that Java was a trademark that would get them into trouble. JavaScript was later renamed ECMAScript, but everyone still called it JavaScript.1996James Gosling invented Java, the first truly overly cumbersome object-oriented language where design patterns completely trumped pragmatism. So the super effective manager provider, container provider, service provider, single manager provider model was born.2001Anders Hejlsberg reinvented Java and called it C# because programming in C felt cooler than Java. Everyone loves this new version of Java because it doesn't look like Java at all.2005David Hanselmeyer Hansen created a web framework called Ruby on Rails. Since then, everyone no longer remembers that Ruby and Rails are two separate things.2006John Resig wrote a help library for JavaScript, and everyone thought it was a language, and copying and pasting jQuery code from the Internet became a profession.2009Ken Thompson and Rob Pike decided to make a language similar to C, but with safer devices, better sales, and a Gopher as the mascot. They called the language Go, made it open source, and sold Gopher-branded kneepads and helmets as a source of income.2010Graydon Hoare also wants to make the language like C, which he calls Rust. Everyone demanded that every piece of the software be rewritten in Rust immediately. Graydon wanted to do something more exciting, so he started developing Swift for Apple.2012Anders Hjelsberg wanted to write C# in a web browser, so he designed TypeScript, which is actually JavaScript, but with more Java stuff in it.2013Jeremy Ashkenas wanted to be as happy as a Ruby developer, so he created CoffeeScript, which compiles like JavaScript but looks more like Ruby. Jerry never became as truly happy as Matz and the Ruby developers.2014When Chris Lattner was working on Swift, its main design goal was not to become Objective-C. In the end, it looked like Java.
