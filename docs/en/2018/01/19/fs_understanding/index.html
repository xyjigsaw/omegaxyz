Feature selection methods and understanding

TechnologyRepostsMachine Learning特征选择
Reprinted from the author:Edwin JarvisChinese version:https://www.cnblogs.com/tonglin0325/p/6214978.htmlFeature selection (ranking) for data scientists,machine learningVery important for practitioners. Good feature selection can improve the performance of the model and help us understand the characteristics and underlying structure of the data, which is helpful for further improving the model andalgorithmAll play an important role. Feature selection mainly has two functions:
  1. Reduce the number of features and dimensionality to make the model more general and reduce overfitting.
  2. Enhance understanding of features and eigenvalues
Once you get the data set, it is often difficult for a feature selection method to accomplish these two purposes at the same time. Under normal circumstances, we often ignore everything and choose the feature selection method that we are most familiar with or the most convenient (often the purpose is dimensionality reduction, while ignoring the purpose of understanding features and data). In many machine learning-related books, it is difficult to find content about feature selection, because the problems to be solved by feature selection are often regarded as a side effect of machine learning and are generally not discussed separately. This article will combineExamples provided by Scikit-learnIntroducing several commonly used feature selection methods, their respective advantages, disadvantages and problems.

1Removing features with low variance

This should be the simplest feature selection method: assuming that the eigenvalues ​​of a feature are only 0 and 1, and in all input samples, 95% of the instances have a value of 1, then this feature can be considered to have little effect. If 100% of them are 1, then this feature is meaningless. This method can only be used when the feature values ​​​​are discrete variables. If it is a continuous variable, the continuous variable needs to be discretized before it can be used. In practice, it is generally unlikely that more than 95% of features take a certain value, so this method is simple but not very easy to use. It can be used as preprocessing for feature selection, first removing those features with small value changes, and then selecting appropriate ones from the feature selection methods mentioned next for further feature selection.

2Univariate feature selection Univariate feature selection

Univariate feature selection can test each feature, measure the relationship between the feature and the response variable, and discard bad features based on the score. For regression and classification problems, features can be tested using chi-square tests and other methods. This method is relatively simple, easy to run, and easy to understand. It usually has a better effect on understanding the data (but it may not be effective for feature optimization and improving generalization ability); this method has many improved versions and variants.

2.1 Pearson Correlation Pearson Correlation

The Pearson correlation coefficient is the simplest method that can help understand the relationship between features and response variables. This method measures the linear correlation between variables. The value range of the result is [-1, 1], -1 indicates a complete negative correlation (when this variable decreases, that one will increase), +1 indicates a complete positive correlation, and 0 indicates no linear correlation. Pearson Correlation is fast and easy to calculate, and is often executed as soon as the data (after cleaning and feature extraction) is obtained. scipypearsonrThe method can calculate the correlation coefficient and p-value at the same time,
import numpy as np
from scipy.stats import pearsonr
np.random.seed(0)
size = 300
x = np.random.normal(0, 1, size)
print "Lower noise", pearsonr(x, x + np.random.normal(0, 1, size))
print "Higher noise", pearsonr(x, x + np.random.normal(0, 10, size))
Lower noise (0.71824836862138386, 7.3240173129992273e-49) Higher noise (0.057964292079338148, 0.31700993885324746) In this example, we compare the differences between variables before and after adding noise. When the noise is relatively small, the correlation is strong and the p-value is low. Provided by Scikit-learnf_regrssionThis method can calculate the p-value of features in batches, which is very convenient. Please refer to sklearn'spipelineAn obvious flaw of the Pearson correlation coefficient is that, as a feature ranking mechanism, it is only sensitive to linear relationships. If the relationship is nonlinear, the Pearson correlation may be close to 0 even if the two variables have a one-to-one correspondence.

2.2 Mutual information and maximal information coefficient (MIC)

The above is the classic mutual information formula. It is not very convenient to directly use mutual information for feature selection: 1. It does not belong to a measurement method, and there is no way to normalize it, so the results on different data cannot be compared; 2. It is not very convenient to calculate continuous variables (X and Y are both sets, and x and y are discrete values). Usually the variables need to be discretized first, and the results of mutual information are very sensitive to the discretization method. The maximum information coefficient overcomes both problems. It first looks for an optimal discretization method, and then converts the mutual information value into a measurement method, with the value range being [0, 1].minepyProvides MIC function. Looking back at the example of y=x^2, the mutual information value calculated by MIC is 1 (the maximum value).
from minepy import MINE
m = MINE()
x = np.random.uniform(-1, 1, 10000)
m.compute_score(x, x**2)
print m.mic()
1.0 The statistical power of MIC sufferssome doubts, when the null hypothesis does not hold, the statistics of MIC will be affected. This problem does not exist in some data sets, but it does exist in some data sets.

2.3 Distance correlation

The distance correlation coefficient was developed to overcome the weaknesses of the Pearson correlation coefficient. In the example of x and x^2, even if the Pearson correlation coefficient is 0, we cannot conclude that the two variables are independent (it may be nonlinear correlation); but if the distance correlation coefficient is 0, then we can say that the two variables are independent. R'senergyThe package provides the implementation of distance correlation coefficient. In addition, this isPython gistrealization.
#R-code
> x = runif (1000, -1, 1)
> dcor(x, x**2)
[1] 0.4943864
Although there are MIC and distance correlation coefficients, when the relationship between variables is close to linear correlation, the Pearson correlation coefficient is still irreplaceable. First, the calculation speed of Pearson correlation coefficient is fast, which is very important when processing large-scale data. Second, the value range of the Pearson correlation coefficient is [-1, 1], while the MIC and distance correlation coefficients are both [0, 1]. This feature enables the Pearson correlation coefficient to represent richer relationships. The symbol represents the positive and negative of the relationship, and the absolute value can represent the intensity. Of course, the premise for the Pearson correlation to be effective is that the changing relationship between the two variables is monotonic.

2.4 Feature ranking based on learning model (Model based ranking)

The idea of ​​this method is to directly use the machine learning algorithm you want to use and build a predictive model for each individual feature and response variable. In fact, the Pearson correlation coefficient is equivalent to the standardized regression coefficient in linear regression. If the relationship between a feature and the response variable is nonlinear, tree-based methods (decision trees, random forests), or extended linear models can be used. Tree-based methods are easier to use because they model nonlinear relationships better and do not require much debugging. But we should pay attention to the problem of over-fitting, so it is best not to make the depth of the tree too large, and then use cross-validation. existBoston house price data setUsing sklearn onrandom forest regressionGive an example of univariate selection:

3 Linear models and regularization

Univariate feature selection methods independently measure the relationship between each feature and the response variable. Another mainstream feature selection method is a method based on machine learning models. Some machine learning methods have their own mechanism for scoring features, or they can be easily applied to feature selection tasks, such as regression models, SVM, decision trees, random forests, etc. As an aside, this method seems to be called the wrapper type in some places. It probably means that the feature ranking model and the machine learning model are coupled together, and the corresponding non-wrapper type feature selection method is called the filter type. Next, we will introduce how to use the coefficients of the regression model to select features. The more important the feature is, the larger the corresponding coefficient will be in the model, and the more irrelevant the feature is to the output variable, the corresponding coefficient will be closer to 0. On data with little noise, or data whose amount of data is much larger than the number of features, if the features are relatively independent, even the simplest linear regression model can achieve very good results.
from sklearn.linear_model import LinearRegression
import numpy as np

np.random.seed(0)
size = 5000

#A dataset with 3 features
X = np.random.normal(0, 1, (size, 3))
#Y = X0 + 2*X1 + noise
Y = X[:,0] + 2*X[:,1] + np.random.normal(0, 2, size)
lr = LinearRegression()
lr.fit(X, Y)

#A helper method for pretty-printing linear models
def pretty_print_linear(coefs, names = None, sort = False):
    if names == None:
        names = ["X%s" % x for x in range(len(coefs))]
    lst = zip(coefs, names)
    if sort:
        lst = sorted(lst,  key = lambda x:-np.abs(x[0]))
    return " + ".join("%s * %s" % (round(coef, 3), name)
                                   for coef, name in lst)

print "Linear model:", pretty_print_linear(lr.coef_)
Linear model: 0.984 * X0 + 1.995 * X1 + -0.041 * X2 In this example, although there is some noise in the data, this feature selection model can still well reflect the underlying structure of the data. Of course, this is also because the problem in the example is very suitable to be solved by a linear model: there is a linear relationship between the features and the response variables, and the features are independent. In many actual data, there are often multiple interrelated features. At this time, the model will become unstable. Subtle changes in the data may lead to huge changes in the model (changes in the model are essentially coefficients, or parameters, which can be understood as W). This will make the prediction of the model difficult. This phenomenon is also called multicollinearity. For example, suppose we have a data set, and its real model should be Y=X1+X2. When we observe it, we find that Y’=X1+X2+e, and e is noise. 3.1 Regularization model

Regularization is to add additional constraints or penalties to the existing model (loss function) to prevent overfitting and improve generalization ability. The loss function changes from the original E(X,Y) to E(X,Y)+alpha||w||, w is a vector composed of model coefficients (also called parameters, coefficients in some places), ||·|| is generally the L1 or L2 norm, and alpha is an adjustable parameter that controls the strength of regularization. When used on linear models, L1 regularization and L2 regularization are also called Lasso and Ridge.

3.2 L1 regularization/Lasso

L1 regularization adds the l1 norm of the coefficient w as a penalty term to the loss function. Since the regular term is non-zero, this forces the coefficients corresponding to those weak features to become 0. Therefore, L1 regularization often makes the learned model very sparse (the coefficient w is often 0). This feature makes L1 regularization a good feature selection method. Scikit-learn provides Lasso for linear regression and L1 logistic regression for classification. The following example runs Lasso on Boston house price data, where the parameter alpha is optimized through grid search.
from sklearn.linear_model import Lasso
from sklearn.preprocessing import StandardScaler
from sklearn.datasets import load_boston

boston = load_boston()
scaler = StandardScaler()
X = scaler.fit_transform(boston["data"])
Y = boston["target"]
names = boston["feature_names"]

lasso = Lasso(alpha=.3)
lasso.fit(X, Y)

print "Lasso model: ", pretty_print_linear(lasso.coef_, names, sort = True)
Lasso model: -3.707 * LSTAT + 2.992 * RM + -1.757 * PTRATIO + -1.081 * DIS + -0.7 * NOX + 0.631 * B + 0.54 * CHAS + -0.236 * CRIM + 0.081 * ZN + -0.0 * INDUS + -0.0 * AGE + 0.0 * RAD + -0.0*TAX It can be seen that the coefficients of many features are 0. If you continue to increase the value of alpha, the resulting model will become increasingly sparse, that is, more and more feature coefficients will become 0. However, L1 regularization, like non-regularized linear models, is also unstable. If there are correlated features in the feature set, subtle changes in the data may lead to large model differences.

3.3 L2 regularization/Ridge regression

L2 regularization adds the L2 norm of the coefficient vector to the loss function. Since the coefficient in the L2 penalty term is quadratic, there are many differences between L2 and L1. The most obvious point is that L2 regularization will average the values ​​of the coefficients. For correlated features, this means that they can obtain closer corresponding coefficients. Taking Y=X1+X2 as an example, assuming that X1 and X2 have a strong correlation, if L1 regularization is used, the penalty will be the same regardless of whether the learned model is Y=X1+X2 or Y=2X1, which is 2alpha. But for L2, the penalty term of the first model is 2alpha, but the penalty term of the second model is 4*alpha. It can be seen that when the sum of the coefficients is a constant, the penalty is the smallest when the coefficients are equal, so L2 has the characteristic that the coefficients will tend to be the same. It can be seen that L2 regularization is a stable model for feature selection. Unlike L1 regularization, the coefficients will fluctuate due to subtle data changes. Therefore, the values ​​provided by L2 regularization and L1 regularization are different. L2 regularization is more useful for feature understanding: the coefficient corresponding to a feature with strong representation ability is non-zero. Let's look back at the example of 3 interrelated features, and run them 10 times with random initialization with 10 different seeds to observe the stability of L1 and L2 regularization.

4 Random Forest

Random forest has the advantages of high accuracy, good robustness, and ease of use, making it one of the most popular machine learning algorithms currently. Random forest provides two methods of feature selection: mean decrease impurity and mean decrease accuracy.

4.1 mean decrease impurity mean decrease impurity

Random forest consists of multiple decision trees. Each node in the decision tree is a condition related to a certain feature in order to divide the data set into two according to different response variables. Impurity can be used to determine nodes (optimal conditions). For classification problems, usuallyGini Impurityorinformation gain, for regression problems, usually usedvarianceOr least squares fitting. When training a decision tree, you can calculate how much each feature reduces the impurity of the tree. For a decision tree forest, you can calculate how much impurity is reduced by each feature on average, and use its average reduced impurity as the value of feature selection. The following example is the feature importance measurement method based on random forest in sklearn:

from sklearn.ensemble import RandomForestRegressor
import numpy as np
#Load boston housing dataset as an example
boston = load_boston()
X = boston["data"]
Y = boston["target"]
names = boston["feature_names"]
rf = RandomForestRegressor()
rf.fit(X, Y)
print "Features sorted by their score:"
print sorted(zip(map(lambda x: round(x, 4), rf.feature_importances_), names), 
             reverse=True)

When using impurity-based methods, remember: 1. This method existsbias, it will be more beneficial for variables with more categories; 2. 4.2 Mean decrease accuracy

Another commonly used feature selection method is to directly measure the impact of each feature on the model accuracy. The main idea is to disrupt the order of the feature values ​​of each feature and measure the impact of order changes on the accuracy of the model. Obviously, for unimportant variables, shuffling the order will not have a great impact on the accuracy of the model, but for important variables, shuffling the order will reduce the accuracy of the model. This method is not directly provided in sklearn, but it is easy to implement. Let’s continue to implement it on the Boston housing price data set.
from sklearn.cross_validation import ShuffleSplit
from sklearn.metrics import r2_score
from collections import defaultdict

X = boston["data"]
Y = boston["target"]

rf = RandomForestRegressor()
scores = defaultdict(list)

#crossvalidate the scores on a number of different random splits of the data
for train_idx, test_idx in ShuffleSplit(len(X), 100, .3):
    X_train, X_test = X[train_idx], X[test_idx]
    Y_train, Y_test = Y[train_idx], Y[test_idx]
    r = rf.fit(X_train, Y_train)
    acc = r2_score(Y_test, rf.predict(X_test))
    for i in range(X.shape[1]):
        X_t = X_test.copy()
        np.random.shuffle(X_t[:, i])
        shuff_acc = r2_score(Y_test, rf.predict(X_t))
        scores[names[i]].append((acc-shuff_acc)/acc)
print "Features sorted by their score:"
print sorted([(round(np.mean(score), 4), feat) for
              feat, score in scores.items()], reverse=True)
Features sorted by their score: [(0.7276, ‘LSTAT’), (0.5675, ‘RM’), (0.0867, ‘DIS’), (0.0407, ‘NOX’), (0.0351, ‘CRIM’), (0.0233, ‘PTRATIO’), (0.0168, ‘TAX’), (0.0122, ‘AGE’), (0.005, ‘B’), (0.0048, ‘INDUS’), (0.0043, ‘RAD’), (0.0004, ‘ZN’), (0.0001, ‘CHAS’)] In this example, the two features LSTAT and RM have a great impact on the performance of the model. Disturbing the eigenvalues of these two features reduces the performance of the model by 73% and 57%. Note that although we trained the model on all features and then obtained the importance test of each feature, this does not mean that the performance of the model will definitely drop a lot after we throw away one or some important features, because even after a feature is deleted, its associated features can still play a role, leaving the model performance basically unchanged.

5 Two top-level feature selection algorithms

They are called top-level because they are all based on model-based feature selection methods, such as regression and SVM, which build models on different subsets and then aggregate them to finally determine feature scores.

5.1 Stability selection Stability selection

Stability selection is a newer method based on a combination of subsampling and selection algorithms. The selection algorithm can be regression, SVM or other similar methods. Its main idea is to run the feature selection algorithm on different data subsets and feature subsets, repeat it continuously, and finally summarize the feature selection results. For example, you can count the frequency of a feature being considered an important feature (the number of times it is selected as an important feature divided by the number of times the subset it is in is tested). Ideally, scores for important features would be close to 100%. A slightly weaker feature will have a non-zero score, while the least useful feature will have a score close to 0. sklearn inrandom lassoandstochastic logistic regressionThere is an implementation of stability selection in .

For feature selection tasks, stability selection is often one of the best-performing methods on many datasets and environments.

5.2 Recursive feature elimination (RFE)

The main idea of ​​recursive feature elimination is to repeatedly build a model (such as SVM or regression model) and then select the best (or worst) features (can be selected based on coefficients), put the selected features once, and then repeat this process on the remaining features until all features have been traversed. The order in which features are eliminated in this process is the ranking of features. Therefore, this is a greedy algorithm to find the optimal subset of features. The stability of RFE largely depends on which underlying model is used during iteration. For example, if RFE uses ordinary regression, the regression without regularization is unstable, then RFE is unstable; if Ridge is used, and the regression regularized by Ridge is stable, then RFE is stable. Sklearn providesRFEpackage, which can be used for feature elimination, also providesRFECV, the features can be sorted through cross-validation.
from sklearn.feature_selection import RFE
from sklearn.linear_model import LinearRegression

boston = load_boston()
X = boston["data"]
Y = boston["target"]
names = boston["feature_names"]

#use linear regression as the model
lr = LinearRegression()
#rank all features, i.e continue the elimination until the last one
rfe = RFE(lr, n_features_to_select=1)
rfe.fit(X,Y)

Summary

  1. Univariate feature selection is a very good choice for understanding data, data structure, and characteristics. Although it can be used to rank features to optimize the model, it cannot detect redundancy (for example, if there is a strong correlation between features in a feature subset, it is difficult to consider redundancy when selecting the optimal features).
  2. Regularized linear models are very powerful tools for feature understanding and feature selection. L1 regularization can generate sparse models and is very useful for selecting feature subsets; compared to L1 regularization, L2 regularization is more stable. Since useful features often correspond to non-zero coefficients, L2 regularization is very suitable for data understanding. Since the relationship between response variables and features is often nonlinear, basis expansion can be used to transform the features into a more appropriate space. On this basis, a simple linear model can be considered.
  3. Random forest is a very popular feature selection method. It is easy to use and generally does not require tedious steps such as feature engineering and parameter adjustment. Moreover, many toolkits provide average impurity reduction methods. Its two main problems are: 1. Important features may have low scores (associated feature problem), and 2. This method is more beneficial to features with many feature variable categories (bias problem). Still, this approach is well worth trying in your application.
  4. Feature selection is very useful in many machine learning and data mining scenarios. When using it, figure out what your goals are, and then find which method is suitable for your task. When selecting optimal features to improve model performance, cross-validation methods can be used to verify whether a certain method is better than other methods. When using feature selection methods to understand data, be careful. The stability of the feature selection model is very important. A model with poor stability can easily lead to wrong conclusions. It can help to subsample the data and then run a feature selection algorithm on a subset. If the results are consistent across subsets, then the conclusions drawn on this data set are credible and the results of this feature selection model can be used to understand the data.
