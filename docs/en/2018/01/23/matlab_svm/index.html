How to use SVM (support vector machine) in MATLAB

Programming LanguagesTechnologyRepostsMATLABMachine LearningMATLABSVM
LIBSVM is a simple, easy-to-use, fast and effective SVM pattern recognition and regression software package developed and designed by Professor Lin Chih-Jen of National Taiwan University. He not only provides compiled executable files that can be used on Windows series systems, but also provides source code to facilitate improvement, modification and application on other operating systems. The software has relatively few parameter adjustments involved in SVM and provides many default parameters. Many problems can be solved by using these default parameters; and provides interactive verification (Cross Validation) function. The software can solve problems such as C-SVM, ν-SVM, ε-SVR and ν-SVR, including multi-class pattern recognition problems based on one-to-one algorithms. Note that it is not the svm implementation function that comes with matlab. The built-in svm implementation function only supports classification problems and does not support regression problems; libsvm not only supports classification problems but also regression problems, with adjustable parameters and more powerful functions. The configuration of libsvm is very simple. You only need to download the advanced version of matlab and libsvm. The VC or VS compilation and implementation is very simple, so I won’t go into details here. Two steps: training modeling -> model prediction Classification model = svmtrain(trainlabel, traindata, '-s 0 -t 2 -c 1.2 -g 2.8'); Regression model = svmtrain(trainlabel, traindata, '-s 3 -t 2 -c 2.2 -g 2.8 -p 0.01');Parameter description: -s svm type: SVM setting type (default 0)0 -- C-SVC 1 --v-SVC 2 – One-class SVM 3--e-SVR 4 -- v-SVR-t kernel function type: kernel function setting type (default 2)0 – Linear: u'v 1 – Polynomial: (r*u'v + coef0)^degree 2 – RBF function: exp(-r|u-v|^2) 3 –sigmoid：tanh(r*u'v + coef0)-g r(gama): gamma function settings in kernel function (for polynomial/rbf/sigmoid kernel function) -c cost: Set the parameters (loss function) of C-SVC, e-SVR and v-SVR (default 1), penalty coefficient -n nu: Set v-SVC, first-class SVM and v-SVR parameters (default 0.5) -p p: Set the value of loss function p in e-SVR (default 0.1) -d degree: degree setting in kernel function (for polynomial kernel function) (default 3) -wi weight: Set the parameter C of which category to weight*C (C in C-SVC) (default 1) -v n: n-fold interactive testing mode, n is the number of folds, which must be greater than or equal to 2 Misunderstandings in using libsvm----------------------(1) Simply normalizing the training set and test set directly to the [0,1] interval may lead to poor experimental results. (2) If the number of features of the sample is very large, then there is no need to use the RBF kernel to map the sample into a high-dimensional space.
  • a) When the number of features is very large, using linear kernel, the results are already very good, and only parameter C needs to be selected.
  • b) Although the results of the RBF kernel are at least better than the linear kernel, the entire space must be searched.
(3) When the number of samples << the number of features: it is recommended to use linear kernel, which can achieve the same performance as RBF. (4) The number of samples and features is very large: it is recommended to use liblinear, which takes less time and memory and has comparable accuracy. (5) Number of samples>>Number of features: If you want to use a linear model, you can use liblinear and use the -s 2 parameter   Detailed explanation:

1. Training

Format:
model = libsvmtrain(training_label_vector, training_instance_matrix [, 'libsvm_options']);This function has three parameters, among which
  • -training_label_vector: The class label of the training sample. If there are m samples, it is a matrix of m x 1 (the type must be double). There can be two categories and multiple categories here. The class labels are (-1,1), (1,2,3) or any other numbers used to represent different categories. They must be converted to double type.
  • -training_instance_matrix: Characteristics of training samples. If there are m samples and each sample feature is n-dimensional, it will be an m x n matrix (the type must be double).
  • -libsvm_options: training parameters, detailed in point 3.
The libsvmtrain function returns the trained SVM classifier model model which is a structure containing the following members:
  • -Parameters: A 5 x 1 matrix, expressed from top to bottom: -s SVM type (default 0); -t kernel function type (default 2) -d degree setting in kernel function (for polynomial kernel function) (default 3); -g r(gamma) function setting in kernel function (for polynomial/rbf/sigmoid kernel function) (reciprocal of the default number of categories); -r coef0 setting in kernel function (for polynomial/sigmoid kernel function) ((default 0)
  • -nr_class: Indicates how many categories there are in the data set. For example, this value is 2 for two classifications.
  • -totalSV: ​​Indicates the total number of support vectors.
  • -rho: The opposite number (-b) of the constant term in the decision function wx+b.
  • -Label: represents the label of the category in the data set, such as 1 and -1, which are common in binary classification.
  • -ProbA: The value used for probability estimation when the -b parameter is used, otherwise it is empty.
  • -ProbB: The value used for probability estimation when the -b parameter is used, otherwise it is empty.
  • -nSV: Indicates the number of support vectors for each type of sample, corresponding to the category label of Label. For example, Label=[1; -1], nSV=[63; 67], then the sample with label 1 has 63 support vectors, and the sample with label -1 has 67 support vectors.
  • -sv_coef: Indicates the coefficient of each support vector in the decision function.
  • -SVs: represents all support vectors. If the feature is n-dimensional and there are m support vectors in total, it is an m x n sparse matrix.
In addition, if the -v parameter is used for cross-validation during training, what is returned is not a model, but the accuracy of the cross-validation classification or the root mean square error of the regression.
After building the model, you still need to select appropriate values ​​for the above parameters. The main methods are Gridsearch. Others are not commonly used. Gridsearch is simply exhaustive.Grid parameter optimization function (classification problem):SVMcgForClass[bestCVaccuracy,bestc,bestg]=SVMcgForClass(train_label,train,cmin,cmax,gmin,gmax,v,cstep,gstep,accstep) Input:
  • train_label: The label of the training set, the format requirements are the same as svmtrain.
  • train: training set, the format requirements are the same as svmtrain.
  • cmin, cmax: The variation range of the penalty parameter c, that is, finding the best parameter c within the range of [2^cmin, 2^cmax]. The default value is cmin=-8, cmax=8, that is, the default range of the penalty parameter c is [2^(-8),2^8].
  • gmin,gmax: The variation range of RBF kernel parameter g, that is, find the best RBF kernel parameter g within the range of [2^gmin,2^gmax]. The default value is gmin=-8, gmax=8, that is, the default range of RBF kernel parameter g is [2^(-8),2^8].
  • v: Parameters in the Cross Validation process, that is, v-fold Cross Validation is performed on the training set. The default is 3, which means that the 3-fold CV process is performed by default.
  • cstep, gstep: The step size of c and g is used for parameter optimization, that is, the value of c is 2^cmin, 2^(cmin+cstep),...,2^cmax, and the value of g is 2^gmin,2^(gmin+gstep),...,2^gmax. The default values ​​are cstep=1, gstep=1.
  • accstep: The step interval size (a number between [0,100]) displayed by the accuracy discretization in the final parameter selection result graph, the default is 4.5.
Output:
  • bestCVaccuracy: The best classification accuracy in the final CV sense.
  • bestc: best parameter c.
  • bestg: the best parameter g.
  Grid parameter optimization function (regression problem):SVMcgForRegress[bestCVmse,bestc,bestg]=SVMcgForRegress(train_label,train,cmin,cmax,gmin,gmax,v,cstep,gstep,msestep) Its input and output are similar to SVMcgForClass, so I won’t go into details here.How can SVM get good results?1. Normalize the data (simple scaling) 2. Apply RBF kernel 3. Use cross-validation and grid-search to get the optimal c and g 4. Use the optimal c and g to train the training data 5. TestingAbout svm’s C and kernel function parameter settings----------------------C can generally be chosen as: 10^t, t=- 4..4 is 0.0001 to 10000 The larger the selection, the greater the punishment for erroneous cases, which may lead to model overfitting.   In LIBSVM -t is used to specify the kernel function type (the default value is 2). 0) Linear kernel function (no other parameters) 1) Polynomial kernel function (The key point is the choice of order, that is, d. Generally, 1-11 is selected: 1 3 5 7 9 11. You can also choose 2, 4, 6...) 2) RBF kernel function (Radial basis RBF kernel, exp{-|xi-xj|^2/mean square error}, where the mean square error reflects the size of data fluctuations. The parameters can usually be selected from the reciprocal of the following numbers: 0.1 0.2 0.4 0.6 0.8 1.6 3.2 6.4 12.8. The default is the reciprocal of the number of categories, that is, 1/k, and for 2 categories, it is 0.5) 3) sigmoid kernel function, also called S-shaped kernel Two parameters g and r: g generally chooses 1 2 3 4, r chooses 0.2 0.4 0.60.8 1 4) Custom kernel function   Libsvm parameters corresponding to the kernel function: 1) For linear kernel functions, there are no parameters that specifically need to be set. 2) For the polynomial kernel function, there are three parameters. -d is used to set the highest degree of the polynomial kernel function, which is d in the formula. The default value is 3. -g is used to set the gamma parameter setting in the kernel function, which is the first r (gamma) in the formula. The default value is 1/k (k is the number of categories). -r is used to set coef0 in the kernel function, which is the second r in the formula. The default value is 0. 3) For the RBF kernel function, there is a parameter. -g is used to set the gamma parameter setting in the kernel function, which is the first r (gamma) in the formula. The default value is 1/k (k is the number of categories). 4) For the sigmoid kernel function, there are two parameters. -g is used to set the gamma parameter setting in the kernel function, which is the first r (gamma) in the formula. The default value is 1/k (k is the number of categories). -r is used to set coef0 in the kernel function, which is the second r in the formula. The default value is 0.

2. Prediction

Format:
[predicted_label, accuracy, decision_values/prob_estimates] = libsvmpredict(testing_label_vector, testing_instance_matrix, model [, 'libsvm_options']);This function includes four parameters, among which
  • -testing_label_vector: The class label of the test sample. If there are m samples, it is an m x 1 matrix (the type must be double). If the class label is unknown, it can be initialized to any m x 1 double array.
  • -testing_instance_matrix: Characteristics of the test sample. If there are m samples and each sample feature is n-dimensional, it will be an m x n matrix (the type must be double).
  • -model: use the model returned by libsvmtrain
  • -libsvm_options: Predicted parameters, in the same form as training parameters.

The libsvmpredict function has three return values. Unnecessary values ​​can be replaced with ~ in Matlab.
  • -predicted_label: The first return value represents the predicted class label of the sample.
  • -accuracy: The second return value, a 3 x 1 array, represents the accuracy of classification, the root mean square error of regression, and the square correlation coefficient of regression.
  • -decision_values/prob_estimates: The third return value, a matrix containing the values ​​or probability estimates of the decision. For the problem of n prediction samples and k categories, if the "-b 1" parameter is specified, it will be an n x k matrix, with each row representing the probability that the sample belongs to each category; if the "-b 1" parameter is not specified, it will be an n

3. Training parameters

There are many parameters that can be selected during LIBSVM training, including:
  • -s svm type: SVM setting type (default 0) 0 — C-SVC; 1 — v-SVC; 2 — Type I SVM; 3 — e-SVR; 4 — v-SVR
  • -t kernel function type: kernel function setting type (default 2) 0 – linear kernel function: u’v 1 – Polynomial kernel function: (r*u’v + coef0)^degree 2 – RBF (radial basis) kernel function: exp(-r|u-v|^2) 3 – sigmoid kernel function: tanh(r*u’v + coef0)
  • -d degree: degree setting in kernel function (for polynomial kernel function) (default 3)
  • -g r(gamma): gamma function setting in kernel function (for polynomial/rbf/sigmoid kernel function) (default 1/k, k is the total number of categories)
  • -r coef0: coef0 setting in kernel function (for polynomial/sigmoid kernel function) ((default 0)
  • -c cost: Set the parameters (loss function) of C-SVC, e-SVR and v-SVR (default 1)
  • -n nu: Set the parameters of v-SVC, one-type SVM and v-SVR (default 0.5)
  • -p p: Set the value of the loss function p in e-SVR (default 0.1)
  • -m cachesize: Set cache memory size in MB (default 40)
  • -e eps: Set the allowed termination criteria (default 0.001)
  • -h shrinking: whether to use heuristics, 0 or 1 (default 1)
  • -wi weight: Set the parameter C of which category to weight*C (C in C-SVC) (default 1)
  • -v n: n-fold interactive testing mode, n is the number of folds, which must be greater than or equal to 2
The above parameter settings can be combined in any combination according to the SVM type and the parameters supported by the kernel function. If the set parameter does not have an impact in the function or SVM type, the program will not accept the parameter; if the required parameter setting is incorrect, the parameter will use the default value.

4. Read or save

The libsvmread function can read data files stored in LIBSVM format.[label_vector, instance_matrix] = libsvmread(‘data.txt’);The input of this function is the name of the file, and the output is the class label and corresponding features of the sample. The libsvmwrite function can store Matlab's matrix as a LIBSVM format file.libsvmwrite(‘data.txt’, label_vector, instance_matrix]This function has three inputs, which are the saved file name, the class label of the sample, and the corresponding features (must be a sparse matrix of double type).   The above content is excerpted from CSDN:http://blog.csdn.net/sherry_gp/article/details/51823380
