How to plagiarize code gracefully? There is a lot of code in the world, this is the correct posture

Do you know which are the two shortcut keys that programmers are most familiar with, most proficient in, and most commonly used?That's right, I guess what you are thinking about right now is: ctrl+c and ctrl+v, commonly known as: copy and paste. For most programmers: copying and pasting are their left and right arms when typing code and writing programmers to create great products, and copying and pasting are even more inseparable good brothers. The so-called copy of the code in the world is nothing more than copying and pasting in the minds of programmers. What's more, we live in a world that loves open source, and programmers are active participants in creating and creating the open source world. At present, we are in the sharing economy, but we do not know that we are in an era of pseudo-sharing. Shared bicycles, shared cars, and even shared power banks are not true sharing. The so-called sharing is to use idle and remaining resources and share them to achieve resource utilization and maximize the value of resources. The current sharing economy is not about reusing idle and surplus resources, but about spending money to buy them, which is causing greater waste of resources every day and every month. However, the open source of code and technology can be said to be true sharing, but it is not yet economic. After all, too much open source is free. Therefore, the open source of our technology is the real originator of sharing. It's a bit far gone, so I'll try to pull it back. The reason why we talk about this topic is that a recent investigation by a team of researchers from the University of California, Irvine, Czech Technical University, Microsoft Research, and Northeastern University found that most source code files hosted on GitHub are actually copies of previously created files.
The international team of eight researchers didn’t initially set out to study duplication on GitHub. The original goal was to try to define the "granularity" of code replication (i.e. how many files changed between different replicas), but they later discovered "alarmingly high file duplication rates" and changed direction. The researchers looked at 4.5 million original (non-forked) GitHub projects with a total of 428 million unique files. They found that only 85 million files were unique, accounting for approximately 17.63% of all files analyzed.
In fact, one of GitHub's missions is to share code, so it is not surprising to find a high code duplication rate on the GitHub platform. But there is one thing that should alarm you very much, and that is:
Git, the source code control system used to build GitHub, encourages project forking. However, there is a large amount of code duplication in GitHub. These codes have not experienced the fork mechanism, but have been entered by copying and pasting files, or even copying and pasting the entire library.
Many researchers have also expressed doubts. Although it can be forked or relied on, most people now copy and paste files, or even copy and paste the entire library into their own projects, and then share it. It is really surprising.Correct usage of open source librariesFrom this, we should feel and be aware of the use of some open source licenses, and we should use open source libraries and open source code correctly. I wonder how many developers are aware of the correct use of open source libraries? I believe that almost every programmer now uses open source libraries on GitHub in projects developed, but do you respect open source and copyright? Generally, the application should declare which open source libraries are used, as well as information such as open source licenses. I found that Alipay does a good job in this regard. You may also be able to use them in the future. This is learning. I believe that everyone is not using and declaring open source libraries correctly.SummaryIn fact, through research, we can actually find: First, GitHub itself seems to be able to greatly compress the code base, so that it will not be as huge as it is now. Second, more and more research is being done today using a host of open source projects readily available from GitHub. But when we study and use it, we should use it correctly and not misuse it.
Do you know which language has the highest code duplication rate on GitHub? JavaScript projects contained the most duplicate files, and the research team only looked at source code projects written in C++, Java, JavaScript, and Python. Of the four types of projects, the JavaScript project contained the most duplicate code, with 94% of files being 100% identical clones (based on file hashes) of another file hosted on GitHub. C++ ranks second, with 73% of all files being copies of other files, while Python also has 71% code reuse and Java only 40%. JavaScript actually contains the most reused code examples, and it’s easy to explain why: NPM, the de facto package manager for all client-side and server-side JavaScript projects.It is said that there is a lot of code in the world, but it should be copied correctly, used correctly, and played under the rules and permission.Don't copy and paste the entire code into your code base without declaring it. It disrespects the author and makes it look like you did it.
