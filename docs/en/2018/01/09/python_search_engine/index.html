Python English search engine (fuzzy search)

Programming LanguagesTechnologyMachine LearningPythonPython
Suppose there are several .txt files under C:\Record, all of which are pure English documents. Use these documents as content to implement a local search engine that lists relevant search results when the user gives a certain input. You can decide on your own to change the functionality of the search engine and provide relevant documentation. (NLTK can be considered)Documentation:Main steps 1: How to design a search engine? The easiest way is to directly use the simplest pattern matching algorithm such as the KMP algorithm to search in the document list. Of course, this can be done in just one line in Python. 2: Next I thought of using regular expressions for pattern matching, which can enhance the accuracy of matching. So I wrote a pattern matching function of about 10 lines（fuzzy_finder_by_interval(key, name_list)）Ability to perform fuzzy searches based on user input and lexicographic order. 3: I thought that everything would be fine, but during testing I found that using regular expression search The results of learning python and python learning are different, so I usednltk.tokenizeThe module divides the input words (this is advantageous when the search file name is relatively long) so that no matter which word comes first, consistent results can be obtained. 4: Considering that we are sometimes too lazy to type spaces when searching, that is, input learningpython directly, in this case, even using NLTK's simple functions and regular expressions will not get the correct results. So I thought of dividing the input words. At this time, an external dictionary file will be used. I imported commonly used words and computer professional vocabulary into a txt file (I am not sure whether NTLK has a similar function. I will study it during the winter vacation. I will implement what I want to do first) as shown in the figure. In this way, each continuous string can be divided into natural languages. The forward maximum matching algorithm is used here.5: In this way, a correct search engine is completed. Finally, in order to make the search results more accurate, I downloaded thehttp://blog.csdn.net/sky_money/article/details/7957996Learn the basic principles of the spell checker, and use the Naive Bayes algorithm to train common words in the dictionary txt file and correct spelling errors (of course, if the user does not want to turn on automatic correction), for example, I entered learningpkthonThis incorrectly entered string can be automatically changed to learning python so that the search is more accurate. Others: Use os.walk() to perform file traversal operations. In addition, I wrote a function to generate a large number of irrelevant txt files. Code: 
import os
import re
import collections
from nltk.tokenize import TreebankWordTokenizer

all_file = []
word_list = []
get_list = []
outcome = []
alphabet = 'abcdefghijklmnopqrstuvwxyz'
file_path = '**********'
dictionary = '**********'
NWORDS = []


def visit_dir(path):
    '''文件读取'''
    if not os.path.isdir(path):
        print("ERROR")
        return
    list_dirs = os.walk(path)
    for root, dirs, files in list_dirs:
        for f in files:
            all_file.append(os.path.join(root, f)[len(path)+1:])


def load_dic():
    global NWORDS
    '''字典读取'''
    f = open(dictionary, 'r')
    NWORDS = train(words(f.read()))
    for line in f:
        nlen = len(line)-1
        word_list.append(line[:nlen])
    f.close()


def divide_str(note, wordlist):
    '''连续字符串划分如iloveyou划分为 i love you'''
    i = 10
    head = 0
    flag = 0
    while head <= len(note) - 1:
        if head >= (len(note)-i):
            i = len(note)-head
        for p in range(i):
            rear = head + i - p
            flag = 0
            for each in wordlist:
                if note[head:rear] == each:
                    get_list.append(each)
                    head = head + len(each)
                    flag = 1
                    break
            if flag == 1:
                break
        if flag == 0:
            head = head + 1


def fuzzy_finder_by_interval(key, name_list):
    '''根据字典序和输入排序的模糊搜索'''
    key = key.strip()
    findings = []
    pattern = '.*?'.join(key)
    regex = re.compile(pattern)
    for item in name_list:
        match = regex.search(item)
        if match:
            findings.append((len(match.group()), match.start(), item))
    return [x for _, _, x in sorted(findings)]


def words(text): return re.findall('[a-z]+', text.lower())


def train(features):
    model = collections.defaultdict(lambda: 1)
    for f in features:
        model[f] += 1
    return model


def edits1(word):
    splits = [(word[:i], word[i:]) for i in range(len(word) + 1)]
    deletes = [a + b[1:] for a, b in splits if b]
    transposes = [a + b[1] + b[0] + b[2:] for a, b in splits if len(b) > 1]
    replaces = [a + c + b[1:] for a, b in splits for c in alphabet if b]
    inserts = [a + c + b for a, b in splits for c in alphabet]
    return set(deletes + transposes + replaces + inserts)


def known_edits2(word):
    return set(e2 for e1 in edits1(word) for e2 in edits1(e1) if e2 in NWORDS)


def known(words): return set(w for w in words if w in NWORDS)


def correct(word):
    candidates = known([word]) or known(edits1(word)) or known_edits2(word) or [word]
    return max(candidates, key=NWORDS.get)


load_dic()
visit_dir(file_path)
tokenizer = TreebankWordTokenizer()
my_input = input("请输入文件名称：")

if ' ' not in my_input:
    divide_str(my_input, word_list)
    my_input = get_list
else:
    my_input = tokenizer.tokenize(my_input)
    print(my_input)
    for i in range(len(my_input)):
        my_input[i] = correct(my_input[i])

print(my_input)

for i in range(len(my_input)):
    temp = fuzzy_finder_by_interval(my_input[i], all_file)
    for j in range(len(temp)):
        if temp[j] not in outcome:
            outcome.append(temp[j])
print(outcome)
