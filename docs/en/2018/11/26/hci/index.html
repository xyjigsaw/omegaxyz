Forms of human-computer interaction

技术域APP开发软件工程appGUI软件工程HCI

Command line interaction

advantage:
  • ①Expert users can complete tasks faster using the command line
  • ② Saves system resources more than the graphical interface
  • ③It is open to users, and there is no problem that user-operable options cannot be dynamically configured in the graphical interface.
  • ④Keyboard operation is more precise than mouse operation, giving you greater control over applications
  • ⑤Support user-defined commands
shortcoming:
  • ① Mastering the command language places higher demands on the user’s long-term memory and short-term memory.
  • ②The interface uses a memory-based approach, which is not as easy to use as a graphical user interface based on recognition.
  • ③Requires users to be familiar with the keyboard layout and make errors more frequently
  • ④Command writing is close to the execution method of the computer, which is contrary to the emphasis of usability theory that users should not be required to understand the underlying implementation of the computer.

Menu driven interface

advantage:
  • ①Based on recognition mechanism, low demand for memory
  • ②Self-explanatory
  • ③Easy to correct errors. In comparison, the command line mode requires parsing the commands entered by the user.
  • ④Suitable for novice users. If better shortcut key functions are provided, it will also be suitable for expert users
shortcoming:
  • ①The navigation method is not flexible enough
  • ②When the menu size is large, the navigation efficiency is not high
  • ③Occupies screen space and is not suitable for small display devices. To save space, usually organized as drop-down or pop-up menus
  • ④Not efficient for expert users

table-based interface

advantage:
  • ① Simplified data entry
  • ② Since the area to be filled in has been pre-defined, it only needs to be recognized without learning.
  • ③ Guide users to complete data input through predefined formats
  • ④Especially suitable for daily document processing and other tasks that require keying in a large amount of data
shortcoming:
  • ① Will take up a lot of screen space
  • ② It may lead to a situation where the business process is more formalized

direct manipulation

advantage:
  • ① Visualize task concepts so that users can easily identify them
  • ②Easy to learn, suitable for novice users
  • ③The operation is based on recognition and does not require high memory, which can effectively avoid errors.
  • ④Support spatial clues to encourage users to explore the interface
  • ⑤ Able to achieve rapid feedback on user operations, with high user subjective satisfaction
shortcoming:
  • ① Difficult to implement
  • ②Not efficient for expert users
  • ③Not suitable for small screen display devices
  • ④High demand for graphics display performance
  • ⑤ Not self-explanatory and may mislead users

Q&A interface (wizard)

advantage:
  • ① Low memory requirements
  • ②Each interface is self-explanatory
  • ③Represent the task process in a simple linear form
  • ④Suitable for novice users
shortcoming:
  • ①Require valid input from the user side
  • ②Require users to be familiar with interface control
  • ③The error correction process may be tedious

metaphorical interface

shortcoming:
  • ①Metaphor cannot be expanded
  • ②Metaphor relies on similar associations between designers and users
  • ③It is difficult to find appropriate metaphors
  • ④Limits our thinking

natural language interaction

For example: Siri, Cortana, etc.
