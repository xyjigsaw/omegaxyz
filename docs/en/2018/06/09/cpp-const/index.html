C++const detailed explanation

高级语言C&C++技术域C++

The difference between const and macro definition

const int r=100; or int const r=100 Standard const variable declaration plus initialization must be initialized because of the default internal connection. Its scope is this file. The compiler directly replaces it with 100 at compile time after type checking. #definePI 3.1415926
  • (1) Different compiler processing methods
The –define macro is expanded during the preprocessing phase. –const constants are used during compilation and runtime.
  • (2) Different types and safety inspections
The –define macro has no type and does not do any type checking, just expansion. –const constants have specific types, and type checking is performed during the compilation phase.
  • (3) Different storage methods
– The define macro is only expanded. It will be expanded as many times as it is used. No memory will be allocated. –const constants will be allocated in memory (either in the heap or the stack).

const and pointers

  • 1.const int *r=&x; //Declare r as a pointer to a constant x. The object pointed to by r cannot be modified, but it can point to a constant at any address.
  • 2.int const *r=&x; //Exactly equivalent to usage 1, there is no difference.
  • 3.int * const r=&x; //Declare r as a constant pointer, which points to x. The pointer pointed by r cannot be modified, but the content of the address it points to can be modified.
  • 4.const int * const r=&x; //Based on usage 1 and 3, r is a constant pointer pointing to a constant.

const and function

  • 1.void Fuction1 (const int r); //The const value is passed here for the parameter, which means that the initial value of the variable cannot be changed by the function
  • 2.const int Fuction1 (int); //The const value is returned here, which means that the initial value of the variable in the returned original function cannot be modified, but the variable returned by the function by value is made into a copy. Whether it can be modified is meaningless. It can be assigned to any const or non-const type variable, and there is no need to add the const keyword. But this is only for internal types (because internal types must return a value, not a variable, and will not be used as an lvalue). For user-defined types, it is very important that the return value is a constant.

const and class

  • 1. const object const Point px(10,10);//Define const object, the compiler must ensure that this object cannot be changed during its life cycle.
  • 2. Const member variables can only be initialized using the initialization member list in the constructor. Trying to initialize const member variables in the constructor body will cause a compilation error. The initialization member list is in the form: 2.X::X (int ir): r(ir) {} //Assume r is a const member variable of class X
  • 3.const member function. For calls to all non-const member functions of constant objects, the compiler will prohibit and report an error during compilation in order to ensure the const characteristics of the object. So if you want your member function to operate on a const object, declare the function as a const member function. If f( ) is a member function in a class, its declaration is in the form: int f( ) const; //const is placed at the end of the function. The compiler will check this function. Any attempt to change member variables and call non-const member functions in this function will be considered illegal. Note: Neither the constructor nor the destructor of a class can be a const function.
