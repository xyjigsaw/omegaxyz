SQL Server performs cascade delete and update

TechnologyDatabasesDatabasesSQL
Problem with deleting tuples dependent on foreign codes in SQL statements: When creating a foreign key table, you need to define the foreign key to allow "cascade deletion." Otherwise, when deleting the parent table data, an error that the foreign key cannot be deleted will be reported.Cascade delete:When we do not add cascade delete to the key, when deleting the data in the primary key table (data referenced by the foreign key table), an error will be reported and cannot be deleted. The associated foreign key data must be deleted first before the data in the primary key table can be deleted. However, if cascade delete is set when creating a new foreign key, then when we delete the data in the primary key table, the database will automatically delete the associated foreign key table data for us.Cascading updates:We modify the fields in the primary key table that are associated with the foreign key table (usually the primary key of the primary key table, mssql seems to have to be the primary key). If we do not set up cascade update, then we will be prompted that we cannot update at this time, because there is still data in the foreign key table that is being associated with this data, but if we set up cascade update, then the data in the foreign key table will automatically update for us. Add cascading SQL code:
ALTER TABLE [name]
ADD CONSTRAINT
[name] FOREIGN KEY([key name]) REFERENCES [table name]([key name])
ON DELETE CASCADE
