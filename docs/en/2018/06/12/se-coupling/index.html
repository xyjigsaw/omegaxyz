Detailed explanation of software engineering coupling

TechnologySoftware EngineeringSoftware Engineering
Coupling is also called inter-block connection. It refers to a measure of the closeness of interconnection between modules in a software system structure. The closer the connection between modules, the stronger the coupling, and the worse the independence of the module. The level of coupling between modules depends on the complexity of the interface between modules, the calling method and the information transferred.
  • Content CouplingContent coupling occurs between two modules if the following situations occur. (If one of the two modules directly references the content of the other module)
    1. One module directly accesses the internal data of another module;
    2. One module does not go to another module through the normal entrance;
    3. The two modules have some overlap in program code (can only appear in assembly language);
    4. A module has multiple entries.
Example: Module P modifies a statement of module Q, and module P branches to a local label of module Q. Disadvantages: Both modules need to be reused when reusing.
  • Common CouplingIf a group of modules all access the same common data environment, the coupling between them is called common coupling. The common data environment can be a global data structure, a shared communication area, a public coverage area of ​​memory, etc.
Disadvantages; ① Conflicts with structured programming, the code is unreadable ② Modifications to global variables must be consistent ③ Difficult to reuse ④ Unsafe, serious data exposure
  • External CouplingWhen a group of modules all access the same global simple variable instead of the same global data structure, and the information about the global variable is not passed through the parameter table, it is called external coupling.
  • Control CouplingIf one module obviously controls the selection of the function of another module by transmitting control information such as switches, flags, names, etc., it is control coupling.
Disadvantages: The two modules are not independent.Note: Control coupling is related to logical cohesion.
  • Stamp CouplingIf a group of modules passes record information through parameter tables, it is imprinted coupling. It is a substructure of a certain data structure, not a simple variable (the data structure is passed as a parameter).
Note: Imprint coupling occurs when a data structure is passed as a parameter, but the called module uses only part of the component.
  • Data CouplingIf one module accesses another module and exchanges input and output information with each other through data parameters (not control parameters, public data structures or external variables), then this coupling is called data coupling. (All parameters are similar data items)
Note: Data coupling occurs when a data structure is passed as a parameter, but the entire data element is used by the called module. Differences from imprint coupling.
  • Nondirect CouplingIf there is no direct relationship between two modules, the connection between them is completely realized through the control and calling of the main module, which is indirect coupling. This coupled module has the strongest independence.
