C++ polymorphic virtual function dynamic binding

Programming LanguagesC & C++TechnologyC++

definition

Dynamic binding is the act of linking a procedure call with corresponding code. A binding that refers to the code associated with a given procedure call that is known only at runtime. It is a specific form of polymorphic implementation.

principle

In C++, dynamic binding occurs when a virtual function is called through a reference or pointer to a base class. The fact that a reference (or pointer) can point to either a base class object or a derived class object is the key to dynamic binding. A virtual function called with a reference (or pointer) determines at run time that the called function is defined by the actual type pointed to by the reference (or pointer). Dynamic binding in C++ is implemented through virtual functions. Virtual functions are implemented through a virtual function table. This table records the address of the virtual function to solve the problems of inheritance and overwriting and ensure that the correct function can be called according to the actual type of the object during dynamic binding. According to the C++ standard specification, the compiler must ensure that the pointer to the virtual function table exists at the front of the object instance (this is to ensure that the offset of the virtual function is correctly obtained). This means that we get this virtual function table through the address of the object instance, and then we can traverse the function pointers and call the corresponding function.

shortcoming

1. Dynamic binding needs to be looked up in the virtual function table when the function is called, so the performance is slightly lower than that of static function calling. 2. Accessing the derived class's own virtual function through a base class type pointer will cause an error. The relationship between virtual functions, dynamic binding, and runtime polymorphism: Virtual functions are the basis of dynamic binding; dynamic binding is the basis of runtime polymorphism.

Dynamically bind two conditions

(1) Only virtual functions can be dynamically bound, non-virtual functions cannot be dynamically bound. (2) Function calls must be made through a reference or pointer of the base class type. Corresponding to static binding Static binding refers to an expression type that is determined by analyzing the program text without considering the execution-time semantics of the expression. Static binding relies only on the form of the program text containing the expression, which does not change while the program is running. Simply put, it is context-free and its type can be determined at compile time. Dynamic binding and static binding Static binding: compile-time binding, called through objects Dynamic binding: runtime binding, implemented through address

Code example

#include <iostream>
using namespace std;
class Base
{
public:
    virtual void f1()
    {
        cout<<"Base"<<endl;
    }
};
class Drived1:public Base
{
public:
    void f1()
    {
        cout<<"Drived1"<<endl;
    }
};
class Drived2:public Base
{
public:
    void f1()
    {
        cout<<"Drived2"<<endl;
    }
};
void Test(Base* pB)
{
    pB->f1 ();
}
int main()
{
    Base b;
    Drived1 d1;
    Drived2 d2;
    Test(&b);
    Test(&d1);
    Test(&d2);
    return 0;
}
Output result: Base Driven1 Driven12

Problems with multiple inheritance

Declare a vehicle base class with member functions such as Run and Stop, from which the bicycle class and the motorcar class are derived. The motorcycle class is derived from bicycle and motorcar, and they all have member functions such as Run and Stop. Observe the role of virtual functions. Among them, set the Run and Stop functions of the base class (vehicle) as virtual functions Code:
#include<iostream>
using namespace std;

class vehicle{
public:
    virtual void Run()const{cout<<"Vehicle Run"<<endl;}
    virtual void Stop()const{cout<<"Vehicle Stop"<<endl;}
};

class bicycle:public vehicle{
private:
    int Height;
public:
    void Run()const{cout<<"Bicycle Run"<<endl;}
    void Stop()const{cout<<"Bicycle Stop"<<endl;}
};

class motorcar:public vehicle{
private:
    int SeatNum;
public:
    void Run()const{cout<<"Motorcar Run"<<endl;}
    void Stop()const{cout<<"Motorcar Stop"<<endl;}
};


class motorcycle:public bicycle,public motorcar{
private:
public:
    void Run()const{cout<<"Motorcycle Run"<<endl;}
    void Stop()const{cout<<"Motorcycle Stop"<<endl;}
};

void fun(vehicle *ptr)
{
    ptr->Run();
    ptr->Stop();
}

void fun(motorcycle *ptr)
{
    ptr->Run();
    ptr->Stop();
}

int main()
{
    vehicle v;
    bicycle b;
    motorcar m;
    motorcycle x;
    fun(&v);
    fun(&b);
    fun(&m);
    fun(&x);
}
The classes vehicle, motorcar, and bicycle in the program class all belong to the same class group and are derived through public derivation, so they meet the assignment compatibility rules. At the same time, the function members Run and Stop of the base class vehicle are declared as virtual functions. The object pointer is used in the program to access the function members, completing dynamic binding.Why does motorcycle set up a separate function (using a pointer)?At this time, if you still use the first fun function above for the vehicle, ambiguity (multiple inheritance) will occur. The solution is to use a virtual base class (note not a virtual function) or overloading, without using dynamic binding.
