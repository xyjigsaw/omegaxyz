Detailed explanation of software engineering cohesion

TechnologySoftware EngineeringSoftware Engineering

Introduction

Cohesion marks how closely the elements within a module are integrated with each other. It is a natural extension of the concepts of information hiding and localization. Cohesion measures the connections within a module from a functional perspective. A good cohesive module should do exactly one thing. It describes the functional connections within the module. Cohesion and coupling are closely related. A module with high coupling with other modules means low cohesion, while a module with high cohesion means low coupling with other modules. When designing software, you should strive to achieve high cohesion and low coupling.

serendipitous cohesion

Definition: A module that performs multiple completely unrelated operations has accidental cohesion. Disadvantages ① Poor maintainability ② Modules cannot be reused Correction: Split the module into smaller modules, each performing an operation. Example: Call within a function to read a file and print the current time.

logical cohesion

Definition: This kind of module combines several related functions together. Each time it is called, the parameters passed to the module determine which function the module should complete. Example: void fun(int a) When a==1, it outputs "OK" and when a==other values, it outputs "ERROR". (In other cases, multiple parameters may block other actual parameters)

temporal cohesion

Definition: A module formed by combining actions that need to be executed simultaneously is called a temporal cohesion module. Disadvantages: Difficult to reuse

process cohesion

Definition: The combination of components or operations allows the subsequent component or operation to be called immediately after the previous component or operation is called, even if no data is transferred between the two. Simply put, if the processing elements within a module are related and must be executed in a specific order, it is called process cohesion. Disadvantages: Difficult to reuse Solution: Split the module into separate operations.

communicative cohesion

Definition: A module performs a series of operations related to the sequence of steps to be followed by the product, and if all operations are performed on the same data, the module has communicative cohesion. Example: Calculate the speed and send it to the printer and output the result. Disadvantages: Difficult to reuse Solution: Split the module into separate operations.

functional cohesion

All components of all elements in the module exist to complete the same function and jointly complete a single function. The module can no longer be divided. That is, the module only includes all the components necessary to complete a certain function. These components are closely related and indispensable. Pros: Its functionality is clear. To determine whether a module has functional cohesion, it can usually be seen from the module name. If the module name has only one verb and a specific goal (singular noun), it is generally functional cohesion, such as "calculate water bills", "calculate output value" and other modules. Functional cohesion generally occurs at lower levels of software architecture diagrams.

informational cohesion

Definition: If a module performs many operations, each has its own entry point, the code for each operation is relatively independent, and all operations are performed on the same data structure. Note: ① The various operations of logical cohesion are intertwined with each other, while the operation codes of the informational cohesion module are completely independent. ②Informational cohesion modules are mainly used to implement an abstract data type.

Summary

1. Functional cohesion: implement a function. 2. Information cohesion: Complete multiple functions, and each function operates on the same data structure. 3. Communication cohesion: Each functional part within the module uses the same input data or generates the same output data. 4. Process cohesion: A certain part of the flow chart is divided into modules, that is, process cohesion. For example, the loop part, the judgment part, and the calculation part are divided into three modules. 5. Temporal cohesion: Also known as classic cohesion, most of them are multi-functional modules. The execution of each function of the module is related to time, such as initialization module and termination module. 6. Logical cohesion: Combine several related functions together, and determine which function to execute based on the decision parameters sent to the module. 7. Accidental cohesion: There is no connection between the parts in the module, or even if there is a connection, it is very loose. It is the module with the lowest cohesion program.
