2. padding

In the previous part, a 6*6 image was convolved through a 3*3 filter, and a 4*4 image was obtained. Assuming that the output image size is n*n and the filter size is f*f, the output image size is (n−f+1)*(n−f+1) The disadvantage of this convolution operation is that the size of the convolution image will continue to shrink. In addition, the element in the upper left corner of the image is only used by one output, so the pixels at the edge of the image are used less in the output, which means that you lose a lot of information on the edge of the image. In order to solve these two problems, the padding operation is introduced, that is, before the image convolution operation, the image is padded with 0 along the edge of the image. For a 3*3 filter, when our padding width is 1, we can ensure that the output image is as large as the input image. Two modes of padding: ①Valid：no paddingInput image n*n, filter f*f, the output image size is: (n−f+1)*(n−f+1)②Same: The output image is the same size as the input image
