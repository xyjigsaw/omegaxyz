SPOOLing technology

Technology操作系统操作系统原理

Introduction to SPOOLing

SPOOLing (ie, external device online parallel operations), the abbreviation of Simultaneous Peripheral Operations On-Line, is a technology about how slow character devices exchange information with the computer host, often called "spooling technology". SPOOLing technology is actually a simultaneous online operation technology of peripheral devices, also known as queue dump technology. It adds a queued dump link of "input well" and "output well" between input and output. Virtuality is one of the four major features of OS. If one physical CPU can be virtualized into multiple logical CPUs through multiprogramming technology, allowing multiple users to share one host, then one physical I/O device can be virtualized into multiple logical I/O devices through SPOOling technology, which also allows multiple users to share one physical I/O device.

composition

1. Input wells and output wells:The storage areas for input wells and output wells are indiskDeveloped on. The data in the input and output wells are generally organized and managed in the form of files, and these files are called well files. A file only stores the input or output data of a certain process, and the data input or output files of all processes are linked into an input and output queue.2. Input buffer and output buffer:The input buffer and output buffer storage areas are inMemoryDeveloped in. Mainly used to alleviate the speed mismatch between CPU and disk. The input buffer is used to temporarily store the data transmitted by the input device, and then transferred to the input well; the same is true for the output buffer.3. Input process and output process:The input process is also called the pre-input process, which is used to simulate the peripheral control machine during offline input, transfer the data required by the user from the input device to the input buffer, and then store it in the input well. When the CPU needs it, it reads data directly from the input well into memory. On the contrary, the same is true for the output.4. Well management procedures:Used to control the exchange of information between jobs and disk wells.

Features

1. Improved I/O speed:I/O operations performed on data have evolved from I/O operations performed on low-speed I/O devices to accessing data in disk buffers, just like offline input and output, which improves I/O speed and alleviates the speed mismatch between the CPU and low-speed I/Os devices.2. Transform exclusive equipment into shared equipment: Because in the spool printer system, no device is actually allocated to any process, but only a free disk block is allocated to the process in the disk buffer and an I/O request table is established.3. Implemented virtual device functions:At a macro level, for each process, they think that they have an exclusive device, even if multiple processes are actually using an exclusive device at the same time. It can also be said that the spool system realizes the function of transforming an exclusive device into several corresponding logical devices.

How does SPOOLing technology virtualize one printer into multiple printers?

Answer: Transforming a dedicated printer into a printer that can be shared by multiple users is a typical example of applying SPOOLing technology. The specific method is: for the user's print output, the system does not actually allocate the printer to the user process. Instead, it first applies for a free disk area in the output well and sends the data to be printed into it; then it applies for and fills in the request print form for the user, and hangs the form on the request print queue. If the printer is idle, the output program takes the table from the head of the requesting print queue, transfers the data to be printed from the output well to the memory buffer, and then prints until the print queue is empty.
