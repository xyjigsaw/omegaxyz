Introduction to DOM

The Document Object Model (DOM) is a standard programming interface recommended by the W3C organization for processing extensible markup languages. It is a platform- and language-independent application programming interface (API) that can dynamically access programs and scripts and update their content, structure, and style of www documents (currently, HTML and XML documents are partially defined through descriptions). The document can be further processed and the results of the processing can be added to the current page. DOM is a tree-based API document that requires the entire document to be represented in memory during processing.The W3C DOM standard is divided into 3 different parts The HTML DOM defines the objects and properties of all HTML elements, as well as the methods to access them.

Models and extensions

Document Object Model DOM

DOM, the Document Object Model, is a standard interface specification formulated by the W3C. It is a standard API for processing HTML and XML files. DOM provides an access model to the entire document, treating the document as a tree structure. Each node of the tree represents an HTML tag or a text item within the tag. The DOM tree structure accurately describes the interrelationships between tags in HTML documents. The process of converting an HTML or XML document into a DOM tree is called parsing. After the HTML document is parsed, it is converted into a DOM tree, so the processing of the HTML document can be achieved by operating the DOM tree. The DOM model not only describes the structure of the document, but also defines the behavior of the node object. Using the methods and properties of the object, you can easily access, modify, add and delete the nodes and content of the DOM tree. 。

DOM tree extension

According to the definition of W3C, the attributes of DOM tree nodes include tag name (nodeName), node type (value: TagTxt), node content (data), parent node object collection (parent Node), child node object collection (firstChild, lastChild), sibling node object collection (previous Sibling, nextSibling), etc. These attributes of DOM tree nodes give the basic content and structural information of the page, but cannot reflect the relevance of tags, attributes and content to the topic, and therefore lack the semantics required for topic extraction. The general idea of ​​expanding the DOM tree is to consider the categories of HTML page tags and the impact of tag attribute values ​​on page theme information, incorporate this impact into the calculation of page content elements, semantically expand DOM tree nodes, and introduce node influence factors to characterize the importance of the node in the tree.

DOM tree node semantic extension

In order to increase the semantic information of the correlation between DOM tree nodes and page theme information, calculate the importance of node content, and add attributes such as HTML tag category (Category), number of non-link words (WordNum), number of hyperlinks (LinkNum), attribute set (Attibution), and influence factor (Influence) to the node to expand its semantics. HTML tags can be divided into 5 categories according to their functions:
  • Tags describing the title and page summary information: such as <title>, <meta>, etc.
  • Tags for planning web page layout: such as 〈table〉, 〈tr〉, 〈td〉, 〈p〉, 〈div〉, etc., whose function is to describe the layout structure of web page content.
  • Tags describing display characteristics: such as 〈b〉, 〈I〉, 〈strong〉, 〈h1〉-〈h6〉, etc., whose function is to emphasize key content and attract people’s attention.
  • Tags related to hyperlinks represent content correlation information between web pages.
  • Other tags, such as setting the image tag <img>, will be ignored during text extraction.
According to the semantic function of HTML tags in describing web page features, DOM tree nodes are divided into six categories: title category (TITLE), text category (CONTENT), visual category (VISION), block category (BLOCK), hyperlink category (LINK) and other categories (OTHER). Different categories of nodes have different importance for Web information extraction.
  • Title class (TITLE): refers to the exclusive category of title tags in HTML documents.
  • Content category (CONTENT): refers to the tag category that contains the body content of the web page, such as the <td> tag that contains text.
  • Visual category (VISION): refers to the tag category that describes the display characteristics of the page, such as <b>, <strong>, etc.
  • BLOCK: refers to the tag category used to divide web page content into blocks, such as <table>, <tr>, etc.
  • Hyperlink category (LINK): refers to the tag category containing hyperlinks, such as <a>.
  • Other categories (OTHER): refers to label types that do not belong to the above five categories.

interface

  • Node interface: It is the base type of nodes in documents. Defines basic methods for accessing and changing the document structure.
  • Document interface: It represents the entire document. Various nodes (elements, comments, processing instructions, etc.) in the document can be created. The created nodes have an OwnerDoculnent attribute representing the Document object that created them.
  • DocumentFragment interface: It represents a subtree of the document tree, which is quite a small document.
  • Attr interface: It represents the attributes of element nodes. What's interesting is that it is not considered a child node of the element node and does not form part of the DOM tree. It is also not a direct child node of the DocumentFragment node.
  • CharacterData interface: It maintains the DOMsitrgn string and provides an interface for read and write operations. But it does not directly correspond to a certain type of node in the document.
  • Text interface: It inherits from CharacterData. A continuous piece of text representing an element or attribute. It has a derived interface CDATAsection, the purpose is: the content of the CDATASeciton node will not be converted in any way; when using the nomraliez method in Node, adjacent Text nodes will be merged into one node, but using CDATASeciton can avoid merging.
  • Comment interface: It also inherits from CharacterData. Represents the text content in the annotation.
  • NodeList interface: used to manage an ordered set of nodes.
  • Entity interface: It represents an entity; EntityReference represents a reference to an entity.
  • NamedNodeMap interface: used to manage unordered node sets.
  • DOMImplementation interface: It provides an interface independent of instances of the DOM model. CreateDocument can create a Document object; haseFature can determine whether the DOM implementation supports a certain module.
  • Notation interface: It represents the symbol definition in the document.
  • ProcessingInstruction interface: It represents processing instructions.
  • DOMException interface: exception handling. Errors caused by logical errors in the program, data loss, or instability of the DOM implementation itself. During program processing, an error value is returned by the method.

BOM introduction

BOM (Browser Object Model) refers to the browser object model, which is a model used to describe the hierarchical relationship between objects. The browser object model provides an object structure that is independent of content and can interact with the browser window. The BOM is composed of multiple objects. The Window object representing the browser window is the top-level object of the BOM, and other objects are sub-objects of this object.

What is BOM?

  • BOM is the abbreviation of Browser Object Model, referred to as browser object model.
  • The BOM provides objects that interact with the browser window independently of the content.
  • Since BOM is mainly used to manage communication between windows, its core object is window
  • BOM consists of a series of related objects, and each object provides many methods and properties.
  • BOM lacks standards. The standardization organization for JavaScript syntax is ECMA, and the standardization organization for DOM is W3C (WHATWG, WebHypertextApplicationTechnologyWorkingGroup - the Web Hypertext Application Technology Working Group is currently working hard to promote the standardization of BOM)
  • The BOM was originally part of the Netscape browser standard

Main functions

  • 1. The ability to pop up a new browser window;
  • 2. The ability to move, close and change the size of the browser window;
  • 3. Navigation object that can provide detailed information of the WEB browser;
  • 4. A local object that can provide detailed information about the page loaded by the browser;
  • 5. A screen object that provides detailed information about the user's screen resolution;
  • 6. Support Cookies;
  • 7. Internet Explorer extends the BOM to include ActiveX object classes, and ActiveX objects can be implemented through JavaScript.
