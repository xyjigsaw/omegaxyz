Ant colony algorithm (ACO) traveling salesman problem (TSP) path planning MATLAB implementation

高级语言技术域Matlab机器学习进化算法MatlabACO算法

The origin of ant colony algorithm

Ant colony optimization (ant colony optimization) was first proposed by Marco Dorigo and others in 1991. In the process of researching new algorithms, they discovered that when ant colonies are looking for food, they can quickly find the target by exchanging foraging information by secreting a biological hormone called pheromone. Based on this, they proposed an ant colony algorithm based on the principle of positive information feedback. The basic idea of ​​the ant colony algorithm comes from the shortest path principle of ants foraging in nature. According to the observations of insect scientists, it is found that although ants in nature have underdeveloped vision, they can find the shortest path from the food source to the nest without any prompts, and adaptively search for new best paths after the surrounding environment changes. When ants are looking for food sources, they can release a hormone called pheromone along the path they walk, so that other ants within a certain range can detect it. When more and more ants pass through some paths, there will be more and more pheromones, and the probability that the ants will choose this path will be higher. As a result, the pheromones on this path will increase, and the probability of ants taking this path will increase, and life will continue. This selection process is known as the autocatalytic behavior of ants. For a single ant, it does not want to find the shortest path, but only chooses based on probability; for the entire ant colony system, they have achieved the objective effect of finding the optimal path. This is swarm intelligence.

What can the ant colony algorithm do?

Ant colony algorithm is a bionic algorithm designed based on simulating the shortest path behavior of ants looking for food. Therefore, generally speaking, ant colony algorithm is used to solve the shortest path problem, and it has really achieved better results in the Traveling Salesman Problem (TSP, a problem of finding the shortest path). At present, it has been gradually applied to other fields, including graph coloring problems, vehicle scheduling problems, integrated circuit design, communication networks, data cluster analysis, etc.Function optimization problem MATLAB implementation: https://omegaxyz.com/2018/01/26/aco/ Robot path planning: https://omegaxyz.com/2018/01/27/aco_routes/ More ACO algorithms:https://omegaxyz.com/tag/aco/

TSP problem

The Traveling Salesman Problem, also known as the Traveling Salesman Problem (TSP), is one of the famous problems in mathematics. Suppose there is a traveling businessman who wants to visit n cities. He must choose the path he wants to take. The limit of the path is that each city can only be visited once, and he must eventually return to the original city of departure. The goal of path selection is to require the path distance to be the minimum value among all paths.

The code to be implemented in this article

①Problem modelingThe capitals of 31 provinces, municipalities and autonomous regions are drawn on the Cartesian coordinate system and expressed by coordinates. The distance between two cities is expressed by the two-dimensional distance formula.②Initialization parametersm is the population number, n is the number of nodes (here refers to the number of cities)③Construct solution spaceRandomly place each individual at a different point and perform iterative updates.④Update pheromonesCalculate the shortest path in this round and update the pheromone.⑤Determine whether to terminate

Advantages of ACOs

① Using a positive feedback mechanism, it is not easy to fall into local optimality. ② Use pheromones to achieve interaction between individuals, which is conducive to information sharing. ③ Parallel programming is possible, and multiple individuals can perform parallel calculations, effectively reducing time.

MATLAB code

%% 清空环境变量
clear all
clc

%% 导入数据
load citys_data.mat

%% 计算城市间相互距离
n = size(citys,1);
D = zeros(n,n);
for i = 1:n
    for j = 1:n
        if i ~= j
            D(i,j) = sqrt(sum((citys(i,:) - citys(j,:)).^2));
        else
            D(i,j) = 1e-4;      
        end
    end    
end

%% 初始化参数
m = 50;                              % 蚂蚁数量
alpha = 1;                           % 信息素重要程度因子
beta = 5;                            % 启发函数重要程度因子
rho = 0.1;                           % 信息素挥发因子
Q = 1;                               % 常系数
Eta = 1./D;                          % 启发函数
Tau = ones(n,n);                     % 信息素矩阵
Table = zeros(m,n);                  % 路径记录表
iter = 1;                            % 迭代次数初值
iter_max = 500;                      % 最大迭代次数 
Route_best = zeros(iter_max,n);      % 各代最佳路径       
Length_best = zeros(iter_max,1);     % 各代最佳路径的长度  
Length_ave = zeros(iter_max,1);      % 各代路径的平均长度  

%% 迭代寻找最佳路径
while iter <= iter_max
    % 随机产生各个蚂蚁的起点城市
      start = zeros(m,1);
      for i = 1:m
          temp = randperm(n);
          start(i) = temp(1);
      end
      Table(:,1) = start; 
      % 构建解空间
      citys_index = 1:n;
      % 逐个蚂蚁路径选择
      for i = 1:m
          % 逐个城市路径选择
         for j = 2:n
             tabu = Table(i,1:(j - 1));           % 已访问的城市集合(禁忌表)
             allow_index = ~ismember(citys_index,tabu);
             allow = citys_index(allow_index);  % 待访问的城市集合
             P = allow;
             % 计算城市间转移概率
             for k = 1:length(allow)
                 P(k) = Tau(tabu(end),allow(k))^alpha * Eta(tabu(end),allow(k))^beta;
             end
             P = P/sum(P);
             % 轮盘赌法选择下一个访问城市
             Pc = cumsum(P);     
            target_index = find(Pc >= rand); 
            target = allow(target_index(1));
            Table(i,j) = target;
         end
      end
      % 计算各个蚂蚁的路径距离
      Length = zeros(m,1);
      for i = 1:m
          Route = Table(i,:);
          for j = 1:(n - 1)
              Length(i) = Length(i) + D(Route(j),Route(j + 1));
          end
          Length(i) = Length(i) + D(Route(n),Route(1));
      end
      % 计算最短路径距离及平均距离
      if iter == 1
          [min_Length,min_index] = min(Length);
          Length_best(iter) = min_Length;  
          Length_ave(iter) = mean(Length);
          Route_best(iter,:) = Table(min_index,:);
      else
          [min_Length,min_index] = min(Length);
          Length_best(iter) = min(Length_best(iter - 1),min_Length);
          Length_ave(iter) = mean(Length);
          if Length_best(iter) == min_Length
              Route_best(iter,:) = Table(min_index,:);
          else
              Route_best(iter,:) = Route_best((iter-1),:);
          end
      end
      % 更新信息素
      Delta_Tau = zeros(n,n);
      % 逐个蚂蚁计算
      for i = 1:m
          % 逐个城市计算
          for j = 1:(n - 1)
              Delta_Tau(Table(i,j),Table(i,j+1)) = Delta_Tau(Table(i,j),Table(i,j+1)) + Q/Length(i);
          end
          Delta_Tau(Table(i,n),Table(i,1)) = Delta_Tau(Table(i,n),Table(i,1)) + Q/Length(i);
      end
      Tau = (1-rho) * Tau + Delta_Tau;
    % 迭代次数加1，清空路径记录表
    iter = iter + 1;
    Table = zeros(m,n);
end

%% 结果显示
[Shortest_Length,index] = min(Length_best);
Shortest_Route = Route_best(index,:);
disp(['最短距离:' num2str(Shortest_Length)]);
disp(['最短路径:' num2str([Shortest_Route Shortest_Route(1)])]);

%% 绘图
figure(1)
plot([citys(Shortest_Route,1);citys(Shortest_Route(1),1)],...
     [citys(Shortest_Route,2);citys(Shortest_Route(1),2)],'o-');
grid on
for i = 1:size(citys,1)
    text(citys(i,1),citys(i,2),['   ' num2str(i)]);
end
text(citys(Shortest_Route(1),1),citys(Shortest_Route(1),2),'       起点');
text(citys(Shortest_Route(end),1),citys(Shortest_Route(end),2),'       终点');
xlabel('城市位置横坐标')
ylabel('城市位置纵坐标')
title(['蚁群算法优化路径(最短距离:' num2str(Shortest_Length) ')'])
figure(2)
plot(1:iter_max,Length_best,'b',1:iter_max,Length_ave,'r:')
legend('最短距离','平均距离')
xlabel('迭代次数')
ylabel('距离')
title('各代最短距离与平均距离对比')
