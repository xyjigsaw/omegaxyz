Feature discretization and selection algorithm based on PSO

TechnologyMachine LearningEvolutionary AlgorithmsPSO算法特征选择
Previously, we introduced the importance of Feature Selection (FS) and discretized data, gave an overview of the importance of PSO in FS and some commonly used methods, introduced the background of FS and discretization, and introduced the entropy-based cutting point and the minimum description length principle (MDLP) used in this article. Today we will learn some methods of discretizing feature selection using PSO. Today we will introduce EPSO and PPSO. Both EPSO and PPSO follow the basic steps shown in Figure 1. After initialization, particles are iteratively evaluated and updated until a stopping condition is met. To evaluate particles, the training data is first discretized and features are selected based on evolutionary cut points. The converted data is then put into the learning algorithm to calculate the fitness. Based on this adaptability, pbest and gbest are updated and used to update the particle's position.
The discretization and FS steps work the same in both methods. To achieve discretization, if an eigenvalue is smaller than a certain cutoff point, it is converted to 0; otherwise, it is 1. If all values ​​of a feature are converted to the same discrete value, then it is considered an irrelevant feature because it cannot distinguish instances of different classes. FS is accomplished by eliminating these useless features. On the basis of the improved classification performance of the entire discretized data, the remaining discrete features were evaluated.

EPSO

The main idea of ​​EPSO is to use BBPSO to directly evolve a tangent point that can be any value within the corresponding eigenvalue range [MinF···MaxF]. The position of each particle represents a candidate solution, which is an n-dimensional real vector corresponding to the dimension of the problem. Figure 2 gives an example of a particle position and its corresponding candidate solution. In this example, the first dimension of the particle, representing the cut point of the first characteristic (F1), needs to have a value in the range [8.5, 25.7]. If the update point of a feature F is outside this range, it will be set to the nearest boundary.
(1) Particle initialization: Due to the huge search space of multi-variable discretization on high-dimensional data. This means that for those features that were not selected among the initial candidates, their cut points will be set to the maximum value of the corresponding feature. For other selected features, their cut points are initialized using the best entropy-based cut point that satisfies the MDLP. In principle, they can be initialized based on any value within the range of the corresponding characteristic. However, a completely random initial cut point may result in slower convergence. Furthermore, the information gain of a feature’s optimal cut point is an indicator of its relevance. Therefore, features with larger information gain have a greater probability of being selected during the initialization process.
(2) Particle evaluation: Based on the cut points generated by the particles, the training data is converted into a new training set of discrete values ​​and a smaller number of features, thanks to the elimination of features whose cut points are equal to the minimum or maximum values. For example, in Figure 2, the cut point of F3 is equal to its maximum value and the cut point of F5 is equal to its minimum value, both features will be discarded.
The discretization and FS solution of each particle are then evaluated based on the classification accuracy of the transformed training set. By evaluating the entire discrete data, the proposed method can evaluate the segmentation points of all selected features while taking feature interactions into account. PPSO

In EPSO, BBPSO is free to generate a shear point within the corresponding range. This can lead to a huge search space, especially in multivariate methods with multidimensional data. Therefore, in order to narrow the search space to highly potential regions, in PPSO, we use BBPSO to select a cut point from the potential breakpoints of each feature. Potential cut points are entropy-based cut points, and their information gain satisfies the MDLP criterion mentioned earlier. Each feature may have a different number of possible cut points, which are calculated and stored in a table of possible cut points. Figure 3 gives an example of this table and particle positions and corresponding candidates. Each particle position is an integer vector representing the selected clipping point index. Therefore, the size of the vector is equal to the number of original features, while the evolved value needs to be between 1 and the number of potential cut points for the corresponding feature. For example, in Figure 3, the first feature (F1) has two possible cut points, indexes 1 and 2. Therefore, the first dimension of the particle needs to fall in the range [1,2]. If it is 2, then the cut point 6.8 is chosen to discretize F1.
During the update process, if the updated value of a dimension is outside the range of the cut point index, it is set to 0, indicating that the corresponding feature does not have a good cut point and should therefore be ignored.
(1) Particle initialization: Each particle position is initialized to a random feature subset, including some selected features whose tangent point index is different from 0. Selected features will have their cut point index set to the index of the optimal MDLP cut point. Similar to EPSO, features with higher information gain will have a higher chance of selection.
In order to make PPSO general to all problems, PPSO uses a restricted size for all datasets. Then during the evolution, when the BBPSO seems to be stuck in a local optimum, if the current gbest fitness has at least a 10% advantage over the last gbest fitness, the BBPSO will be reset to a larger size (scaling mechanism). The purpose of this mechanism is to start searching for small feature subsets while opening up opportunities for larger and better feature subsets.
(2) Particle evaluation: Based on the selected cut point index of each feature, the cut point value is retrieved from the potential cut point table. It is then used to discretize the corresponding features. However, if the evolutionary cut point index of a feature is 0, then the feature is considered unselected. Therefore, in Figure 3, F2 and F4 are not selected in this example.
In EPSO, classification accuracy is used as a measure of the fitness index of each particle. This can make it difficult to distinguish situations where the boundaries between classes are quite large, allowing many different models to achieve the same accuracy. Furthermore, while wrapper methods are capable of producing highly accurate solutions, filtering methods are generally faster and more general. Combining the strengths of these two methods in a merit function is expected to produce better solutions. In addition, combining these two methods can also better distinguish subtle differences between feature subsets, providing a smoother fitness environment to facilitate the search process. However, simply combining these measures may be computationally impractical. Therefore, we need to find a smart way to combine them without requiring more running time. Among commonly used filtering methods, distance is a multivariate measurement method that can evaluate the discriminative ability of feature sets and is used as the basic measurement method of KNN. Therefore, combining this method with the KNN wrapper method does not increase the calculation time, since the distance measurement is only calculated once but used twice.
