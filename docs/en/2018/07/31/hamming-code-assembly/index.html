Introduction to Hamming Code

The performance of Hamming code is to detect one bit error, which is developed based on the idea of ​​parity check code. It adds r check bits to K bits of data to form a new codeword of K+r bits, and distributes each binary bit of the data to several different parity check groups. In this way, when one of the bits changes, it will cause parity errors in several related groups. Through different combinations of these error groups, it is possible to indicate which bit has an error, so that error correction can be performed. To use Hamming code for verification, you first need to determine the number of digits in the check code. In this experiment, Hamming code only requires one bit of error detection and correction. At this time, r check bits can represent 2 to the power of r different states. In order to achieve one-bit error detection and correction, one state needs to be used to indicate that the information is correct, and k+r states are used to indicate that each of the K-bit data bits and r-bit check bits is incorrect, with a total of k+r+1 states. Therefore, it is required:

2 to the power of r>=k+r+1

In this experiment, assume that the data bits are D, with a total of 4 bits, marked as D4...D1; the check bit is P, with a total of 3 bits, marked as P3...P1; the Hamming code obtained by encoding is H, with a total of 7 bits, marked as H7...H1, and the bit number 1 represents the lowest bit. The encoding rules of Hamming code are: (1) Each check bit Pi is assigned to the i-1 power of bit number 2 in the Hamming code H. Data bits are placed on the vacant bits in H, and the data bits are arranged and allocated bit by bit in order from low to high. (2) Each bit code Hi of Hamming code (including data bits and check bits) is checked by multiple check bits. The relationship is that the bit number of each bit being checked is equal to the sum of the bit numbers of the check bits that verify the bit. The purpose of this arrangement is to hope that the result of the check can correctly reflect the bit number of the error bit.

Implementation details

① Take out each bit bit by bit: Move the bit to be taken out to the lowest bit through the shift instruction, and take it out and save it to other registers through the AND Ri,01H instruction. ② On the currently taken out bit, determine which check bit the current bit is calculated and XOR operation with the check bit register. For example: the fourth bit of data bit D4 is currently fetched, and P1=D1⊕D2⊕D4, P2=D1⊕D3⊕D4, P3=D2⊕D3⊕D4, the fourth bit needs to be calculated in P1, P2, and P3. At this time, the D4 bit is XORed with the three registers that hold P1, P2, and P3 to realize the bit-fetching operation. ③ Change the value of the three check digits into an eight-digit binary number and output the result.
