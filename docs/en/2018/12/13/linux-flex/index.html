Compilation principles using flex on Unix/Lunix-like systems

Technology编译原理编译原理
Flex, formerly lex, is a lexical analyzer generation tool based on the UNIX environment that was completed in 1975 by Mike Lesk and Eric Schmidt, who was an intern at AT&T at the time. This lex is very famous, but its efficiency is too low and it has bugs, which makes it very uncomfortable to use. Later, Vern Paxson of Berkeley Lab rewrote lex in C and named it flex (Fast Lexical Analyzer Generator).The following is how to use it, the flow chart is shown above ①First create a .l file and write in it 
%{
int chars = 0;
int words = 0;
int lines = 0;
%}

%%

[a-zA-Z]+ {words++; chars += strlen(yytext);}
\n  { chars++; lines++; }
.  { chars++; }

%%

int main(int argc, char **argv){
   yylex();
   printf("lines: %8d words: %8d chars: %8d\n", lines, words, chars);
}
② Next open the terminalEnter the following command: flex test.l //Generate one and call itFile of "lex.yy.c" ③Enter the following command to generate a.out file to read input and output/* this action would generate a file named a.out which would be used for reading inputs and print outputs */ cc lex.yy.c -ll //The parameter -ll is for running on Mac. If it is not easy to use (running on Linux), try the parameter -lfl ④Execute a.outEnter ./a.out on the command lineEnter some characters and press ctrl+D to exit. Finally, the number of lines, words, and chars will appear.References:https://blog.csdn.net/u013225150/article/details/51241256 http://www.itdaan.com/keywords/Flex+Bison+Using+flex+on+OSX+%E5%9C%A8mac%E4%B8%8A%E4%BD%BF%E7%94%A8Flex.html
