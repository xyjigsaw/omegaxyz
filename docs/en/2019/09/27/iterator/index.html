Iterator pattern C++ implementation

Programming LanguagesC & C++TechnologySoftware EngineeringC++Software Engineering设计模式

Introduction

  • Provides a way to sequentially access the individual elements of an aggregate object without exposing the object's internal representation.

motivation

  • An aggregate object such as a List should provide a way for others to access its elements without exposing its internal structure.
  • Depending on your needs, you may want to traverse this list in different ways.
  • The iterator pattern can solve these problems. The key idea of ​​this pattern is to separate access and traversal of the list from the list object and put it into an iterator object. The iterator class defines an interface for accessing the elements of the list.
  • For example, a list class (List) may require a list iterator class. The relationship between them is as follows:
  • Note that iterators and lists are coupled together, and the client object must know that the traversal is a list and not some other aggregate structure.
  • It would be nice to have a way to change the aggregate class without changing the client code. This goal can be achieved by generalizing the iterator concept to polymorphic iteration.
 

applicability

  • Access the interior of an aggregate object without exposing its internal representation.
  • Supports multiple traversals of aggregate objects.
  • Provides a unified interface for traversing different aggregate structures. 

structure

participants

  • Iterator
    • Iterators define the interface for accessing and traversing elements
  • ConcreteIterator
    • Concrete iterator implements the iterator interface
    • Keep track of the current position while traversing the aggregate
  • Aggregate
    • Aggregation defines an interface that creates corresponding iterator objects
  • ConcreteAggregate
    • The concrete aggregate implements the interface that creates the corresponding iterator. This operation returns an appropriate instance of ConcreteIterator.
 

cooperation

  • ConcreteIterator keeps track of the current object in the aggregate and can calculate successor objects to be traversed.

Effect

  • It supports traversing a collection in different ways
  • Iterators simplify the interface of aggregates
  • There can be multiple traversals on the same aggregate

Example

Implement a linked list class (define your own data structure, do not use the data structure that comes with the language). Each node in the linked list stores an int variable and a double variable. Use the iterator pattern to implement an external iterator, which can traverse the linked list from beginning to end or from end to beginning and print the data in each node.

code

#include <iostream>
using namespace std;

class Node{
private:
    int val1;
    double val2;
    Node *next;
    Node *pre;
public:
    Node(){
        next = NULL;
        pre = NULL;
    };
    Node(int x1, double x2){
        next = NULL;
        pre = NULL;
        val1 = x1;
        val2 = x2;
    }
    void SetNext(Node *n){
        next = n;
    };
    Node *GetNext(){
        return next;
    }
    void SetPre(Node *n){
        pre = n;
    }
    Node *GetPre(){
        return pre;
    }
    int getVal1(){
        return val1;
    }
    double getVal2(){
        return val2;
    }
};

template<class Node>
class Iterator{
public:
    Iterator() {};
    virtual ~Iterator() {};
    virtual Node* first() = 0;
    virtual Node* last() = 0;
    virtual Node* next() = 0;
    virtual Node* pre() = 0;
    virtual Node* nowPos() = 0;
    virtual int len() = 0;
    virtual void SetDirect(bool s) = 0;
};

template<class Node>
class Aggregate{
public:
    Aggregate() {};
    virtual ~Aggregate() {};
    virtual Iterator<Node>* createIterator() = 0;
    virtual Node* GetFirst() = 0;
    virtual Node* GetLast() = 0;
    virtual int GetSum() = 0;
    virtual void Insert(int a, double b) = 0;
    virtual void Init() = 0;
}; 

template<class Node>
class ConcreteIterator : public Iterator<Node>{
public:
    ConcreteIterator(Aggregate<Node> *a) :aggr(a){
        cur = aggr->GetFirst()->GetNext();
    };
    ~ConcreteIterator() {};
    Node* first(){
        Node * p = new Node();
        p = aggr->GetFirst()->GetNext();
        return p;
    }

    Node* last(){
        Node * p = new Node();
        p = aggr->GetLast();
        return p;
    }

    Node* pre(){
        Node * p = cur->GetPre();
        cur = cur->GetPre();
        return p;
    }

    Node* nowPos(){
        return cur;
    }

    Node* next(){
        Node * p =  cur->GetNext();
        cur = cur->GetNext();
        return p;
    }

    void SetDirect(bool s){
        if(s){
            cur = aggr->GetFirst()->GetNext();
        }else{
            cur = aggr->GetLast();
        }
    }

    int len(){
        return aggr->GetSum();
    }

private:
    Aggregate<Node> *aggr;
    Node* cur;
};

template<class Node>
class LinkList : public Aggregate<Node>{
private:
    Node* first;
    Node* last;
    int sum;
public:
    Iterator<Node>* createIterator(){
        return new ConcreteIterator<Node>(this);
    }
    LinkList(){
        first = NULL;
        last = NULL;
        sum = 0;
    }
    Node* GetFirst(){
        return first;
    }
    Node* GetLast(){
        return last;
    }
    int GetSum(){
        return sum;
    }
    void Init(){
        first = new Node();
    }
    void Insert(int a, double b){
        Node *p = first;
        Node *s = new Node(a, b);
        for (int i = 0; i < sum; i++){
            p = p->GetNext();
        }
        s->SetNext(NULL);
        s->SetPre(p);
        p->SetNext(s);
        last = s;
        sum++;
    }
    ~LinkList(){}
};

int main(){
    Aggregate<Node> * aggr = new LinkList<Node>();
    aggr->Init();
    aggr->Insert(1, 2.1);
    aggr->Insert(2, 4.2);
    aggr->Insert(3, 6.3);
    aggr->Insert(4, 8.4);
    aggr->Insert(8, 16.5);
    
    

    Iterator<Node> * it = aggr->createIterator();
    
    cout<<"正序"<<endl;
    it->SetDirect(true);
    while(it->nowPos()!= NULL){
        cout<<it->nowPos()->getVal1()<<"  "<<it->nowPos()->getVal2()<<endl;
        it->next();
    }
    cout<<"倒序"<<endl;
    it->SetDirect(false);
    do{
        cout<<it->nowPos()->getVal1()<<"  "<<it->nowPos()->getVal2()<<endl;
        it->pre();
    }while(it->nowPos()!= it->first()->GetPre());
    return 0;
}

Summary

①Since the iterator of this experiment is a linked list and needs to be traversed from beginning to end or from end to beginning, this linked list is essentially adoubly linked list. ② Iterators need to be used for traversal instead of updating the structure of the linked list itself, so the iterators need to encapsulate the previous, next, current node, head node, and tail node positions. ③Since it is a linked list, you need to give nodes when creating the linked list and insertingdynamic allocationspace, otherwise it will not operate properly.
