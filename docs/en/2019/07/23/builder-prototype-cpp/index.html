generator pattern

The generator pattern can separate the internal representation of a product from the product generation process, so that a building process can generate product objects with different internal representations.Builder pattern structure: Builder role:Provide an abstract interface to standardize the construction of each component of the product object. In general, this interface is independent of the application's business logic. The one who directly creates product objects in the pattern is the ConcreteBuilder role. The concrete builder class must implement the methods required by this interface: one is the construction method, and the other is the result return method.Concrete Builder role:This role is filled by classes that are closely related to the application and create product instances when called by the application. The main tasks completed by this role include: implementing the interface provided by the Builder role and completing the process of creating product instances step by step. After the building process is complete, provide examples of the product.Director role:Classes that assume this role invoke the concrete builder role to create product objects. The director does not have specific knowledge of the product class. It is the specific builder object that really has specific knowledge of the product class. Product role: Products are complex objects under construction.

Prototype pattern

In the prototype mode, the created object realizes the creation process by requesting the prototype object to copy the prototype object itself. Of course, the created object needs to know the type of the prototype object. This means that the created object only needs to know the type of the prototype object to obtain more prototype instance objects. There is no need to care at all about how these prototype objects are created.Shallow copy: Use a known instance to assign values вҖӢвҖӢto the member variables of the newly created instance one by one. This method is called shallow copy. Deep copy: When the copy constructor of a class not only copies the values вҖӢвҖӢof all non-reference member variables of the object, it also creates new instances for the member variables of the reference type and initializes them to the formal parameter instance values.

Example

Use the generator pattern to simulate the production of IBM computers. The main structure of IBM computers is expressed as follows:class IBM{ В  string monitor=вҖқIBMmonitorвҖқ; В  string keyboard=вҖқIBMkeyboardвҖқ; В  string mouse=вҖқIBMmouseвҖқ; В  Motherboard* MB; В  void display(); } inMBIt is a motherboard class, its main structure is as follows: class Motherboard{ string CPU; string RAM; }That is, the motherboard contains the CPU and RAM. display is a function that prints each component, mainly used to check whether it is produced correctly. The construction sequence is to produce the motherboard first, then the monitor, keyboard and mouse. After using the generator mode to produce the first IBM computer, the prototype mode was used to make two more copies of the computer.
