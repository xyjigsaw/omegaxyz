1. Introduction

1.1 Positioning and goals

Today, with highly developed computer technology, the use of information technology to effectively manage large amounts of complex information has become a common and practical means. On the one hand, this greatly reduces the cost of bookkeeping and manpower. On the other hand, the powerful computing power of modern computers and the widespread deployment of networks have greatly simplified the processing and flow of large amounts of information. The student online examination system is an important component in evaluating students' abilities. It greatly improves teachers' work efficiency. It not only reduces the requirements for paper examination papers, but also reflects the requirements of a conservation-oriented society. This system involves students' online program ability assessment examination, academic performance interrogation, and comprehensive processing of many related information. In order to facilitate teachers to further understand students' performance, it is urgent to develop an online student examination system. The student online examination system integrates part of the management work of test questions, computer marking, and score inquiry into a unified platform. All management personnel work together and cooperate with each other to understand students' programming level in a timely manner. At the same time, it can also facilitate teachers to provide hierarchical guidance according to the different situations of individual students.

1.2 Objects

The intended readers of this Software Requirements Specification are:
  • Program teaching platform development manager
  • Technical manager
  • All members of the project team
  • Test team members
  • SQA personnel
  • Other persons authorized by the development company to have access to this document

1.3 Software requirements analysis theory

Software requirements analysis is what is obtained by studying user needs, fully understanding the complete functions of the user's software requirements, confirming the user's software functional requirements, and establishing a confirmable and verifiable basic basis. Software requirements analysis is the beginning of a project and the most important key point in project implementation. According to the analysis results of relevant institutions, the designed software products have problems such as incompleteness and inaccuracy.80％The above are caused by requirements analysis errors, and fundamental functional problems are particularly prominent due to requirements analysis errors. Therefore, software requirements analysis is a critical step for a successful project.

1.4 Software requirements analysis goals

Provide a comprehensive description of the functions of the implemented software to help users judge the correctness, consistency and completeness of the implemented functions, and encourage users to think carefully and comprehensively about the software requirements before starting the software design. Understand and describe all information required for software implementation, providing a baseline for software design, validation, and verification. Provide a basis for software managers to calculate software costs and prepare software development plans. The specific content of the requirements analysis can be summarized into six aspects: functional requirements of the software, interface between the software and hardware or other external systems, non-functional requirements of the software, reverse requirements of the software, restrictions on software design and implementation, and reading supporting information. Software requirements analysis should try to provide all the information about the functional requirements of software implementation, so that software designers and software testers no longer need to contact the demand side. This requires that the software requirements analysis content should be correct, complete, consistent and verifiable. In addition, in order to ensure the quality of software design and facilitate the adjustment and verification of software functions, software requirements must be expressed in an unambiguous, traceable and modifiable manner.

2. Overview of requirements

2.1 Project background

The software to be developed is called "Computer Programming Ability Online Assessment System". The proposer of this project is the School of Computer Science and Technology of Anhui University, and the developer is the person in charge of the 16th-level software engineering class of Anhui University. The main users are undergraduate graduate students of Anhui University. The software is independent of other systems, a complete system of its own, and easy to apply.

2.2 Requirements Overview

The following is a requirements analysis for the design of an online assessment system for computer programming abilities. First of all, because the exam is oriented to certain specific objects, examinees should undergo identity verification when entering the system. After entering the examination system, examinees should be able to select the examination ability level according to their own needs, so the system should also have the function of selecting examination difficulty (divided into top level, Level A, and Level B). In order to standardize the online exam, the questions and amount of questions in the test paper should be the same for each candidate, but the test questions are not the same. Online exams are based on a network environment, and test papers should be dynamically generated after randomly selecting test questions from the server's database. In addition, the system should also control the examination time and require examinees to hand in papers when the time is up. After the examinee selects the answer and submits it, the computer should automatically judge the paper and display the score after it is obtained. After the exam is completed, you can return to the login interface or check the results and rankings.

This system is an online assessment system based on network technology. The administrator maintains the question bank through the network, adding test questions, modifying test questions, deleting test questions and other operations; candidates complete operations such as exams and score inquiries through the system; the system automatically composes papers and completes operations such as reviewing test papers and counting scores.

3.3 Data flow analysis

After logging into the system, students select a set of test questions from the test question bank, and then start answering the questions. After completing the answers, they submit them to the system. The system will complete the review of the test papers and calculate the scores. Students can log in to query. After logging in to the system, the administrator performs maintenance and updates on the system.

4. Software, hardware and external system interface requirements

4.1 User interface

The user interface is the part of the program that users can see and interact with. It is very important to design a good user interface. This design will provide users with a beautiful, generous, intuitive, and simple-to-operate user interface.

4.2 Hardware requirements

Mobile terminal hardware configuration should follow the following principles: have high reliability, availability and security.   • Web Browser:0+, Chrome, Opera, Safari, Firefox and any browser that supports the HTML5 standard.
  • Standard resolution:1024*768、1920*1080、2K

5. Reliability and availability requirements

5.1 Performance requirements

  • processing power
Since it is an online assessment system, its processing capacity mainly considers the maximum number of concurrent users that the system can carry. According to the actual planning, the maximum number of concurrent users that the system can carry is at least the total number of students in the school * φ, where φ is a constant from 0 to 1, depending on the server capacity.
  • response time
In order to provide online assessment services quickly, the system should be able to respond to online assessment requests quickly. The response time for the user to finally get the result is not only related to the system response speed, but also related to the network conditions. Therefore, higher requirements are required for the Web server.

5.2 Security requirements

The transmitted data is encrypted using a high-strength encryption algorithm (DES), so that even if the data is leaked or intercepted, the relevant data content cannot be identified, ensuring data security. For the data exchanged between the client and the server, the secure socket sublayer (SSL, SSL encrypted transmission is mainly for WEB data transmission, designed based on the security considerations of the transmission of important information) is used for information exchange, and important information is exchanged between the client's mobile terminal and the server.
