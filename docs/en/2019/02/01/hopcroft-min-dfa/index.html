DFAMinimization principle

The so-called automaton simplification problem is to construct another deterministic finite automaton DFA M' for any deterministic finite automaton DFA M, such that L (M) = L (M'), and the number of states of M' is not more than the number of states of M, and it can be said with certainty that an M' with the smallest number of states can be found. Let's first introduce some related basic concepts. Let Siis a state of the automaton M, starting from SiThe set of all symbol strings that can be derived is denoted L (Si）。  There are two states Siand Sj, if there is L(Si）＝L（Sj), then it is called Siand SjIt is an equivalent state. In the automaton shown in the figure above, L(B)=L(C)={1}, all state B and state C are equivalent states. For another example, the set of symbol strings derived from the final state must contain the empty string ε, but the set of symbol strings derived from the non-terminal state cannot contain the empty string ε, so the final state and the non-terminal state are not equivalent. For the concept of equivalence, we can also define it from another perspective. Given a DFA M, if it starts from a certain state P and takes the string w as the input, the DFA M will end in the final state, and if it starts from another state Q and takes the string w as the input, the DFA M will end in the non-terminal state. The string w is said to distinguish the state P from the state Q. Two states that are indistinguishable are called equivalent states. Let Siis a state of the automaton M, if it is impossible to reach the state S from the starting statei, then it is called Siinto useless state. Let SiIs a state of the automaton M. If it turns to itself for any input symbol a and cannot reach the terminal state, it is called SiFor the state of death. The key to simplifying DFA is to divide its state set into some mutually disjoint subsets, so that the states between any two disjoint subsets are distinguishable, and any two states in the same subset are equivalent. In this way, one state can be used as a representative and other equivalent states can be deleted, and then irrelevant states can be deleted, thereby obtaining a DFA with the smallest number of states. The following is a detailed introduction to the simplification algorithm of DFA:
  • First, divide the state of DFA M into the terminal state set K1and the non-terminal state set K2。
K＝K1∪K2From the above definition, K1and K2are not equivalent.
  • Each state set is further divided according to the following method each time until no new divisions are generated.
Assume that the i-th division has divided the state set into k groups, that is: K＝K1(i)∪K2(i)∪…∪Kk(i)For the state set Kj(i)Each state in (j=1,2,…,k) is checked one by one, and there are two states Kj’、 Kj’’∈Kj(i), and for the input symbol a, there is: F(Kj'，a）＝Km F（Kj''，a）＝KnIf Kmand Knbelong to the same state set, then Kj’ and Kj'' into the same set, otherwise Kj’ and Kj'' is divided into two sets.
  • Repeat step (2) until each set can no longer be divided. At this time, the states in each state set are equivalent.
  • Merging equivalent states means taking any one state from the set of equivalent states as a representative and deleting all other equivalent states.
  • If there are irrelevant states, delete them.
According to the above method, the finite automaton is simplified, and the simplified automaton is the automaton with the least state of the original automaton.

HopcroftAlgorithm principle

Algorithmic abstraction:
  • 1: Q/θ ← {F, Q − F}
  • 2: while (∃U, V ∈ Q/θ, a ∈ Σ) s.t. Equation 1 holds do
  • 3: Q/θ ← (Q/θ − {U}) ∪ {U ∩ δ^-1(V, a), U − U ∩ δ^-1(V, a)}
  • 4: end while
Algorithm refinement:
  • 1:W ← {F, Q − F} # Some versions are just W ← {F }
  • 2: P ← {F, Q − F}
  • 3: while W is not empty do
  • 4:     select and remove S from W
  • 5:     for all a ∈ Σ do
  • 6:         la ← δ^-1(S, a)
  • 7:         for all R in P such that R ∩ la ≠ ∅ and R ∉ la do
  • 8:             partition R into R1 and R2: R1 ← R ∩ la and R2 ← R − R1
  • 9:             replace R in P with R1 and R2
  • 10:           if R ∈ W then
  • 11:               replace R in W with R1 and R2
  • 12:           else
  • 13:                 if |R1| ≤ |R2| then
  • 14:                     add R1 to W
  • 15:                 else
  • 16:                     add R2 to W
  • 17:                 end if
  • 18:             end if
  • 19:         end for
  • 20:    end for
  • 21: end while

from graphviz import Digraph
import random
from copy import deepcopy


class Part:
    def __init__(self, src, edge, dst):
        self.src = src
        self.edge = edge
        self.dst = dst


def move(DFA, src, edge):
    for i in range(len(DFA)):
        if DFA[i].src[0] == src and DFA[i].edge == edge:
            return DFA[i].dst[0]
    return ''


def get_source_set(target_set, char, DFA):
    global allstatus
    allstatusSet = set(allstatus)
    source_set = set()
    for state in allstatusSet:
        try:
            if move(DFA, state, char) in target_set:
                source_set.update(state)
        except KeyError:
            pass
    return source_set


def hopcroft_algorithm(DFA):
    global sigma
    global endSet
    global allstatus
    cins = set(sigma)
    termination_states = set(endSet)
    total_states = set(allstatus)
    not_termination_states = total_states - termination_states

    P = [termination_states, not_termination_states]
    W = [termination_states, not_termination_states]

    while W:

        A = random.choice(W)
        W.remove(A)

        for char in cins:
            X = get_source_set(A, char,DFA)
            P_temp = []

            for Y in P:
                S = X & Y
                S1 = Y - X

                if len(S) and len(S1):
                    P_temp.append(S)
                    P_temp.append(S1)

                    if Y in W:
                        W.remove(Y)
                        W.append(S)
                        W.append(S1)
                    else:
                        if len(S) <= len(S1):
                            W.append(S)
                        else:
                            W.append(S1)
                else:
                    P_temp.append(Y)
            P = deepcopy(P_temp)
    return P


def indexMinList(minList, a):
    for i in range(len(minList)):
        if a in minList[i]:
            return i
    return ''

Difficulties and solutions

  • In the process of DFA minimization, it is limited to remove useless states and then use the subset partitioning algorithm to process. According to the data, I found a fast DFA minimization algorithm called Hopcroft algorithm. Through this algorithm, DFA of alphabets of different sizes can be processed.
  • After performing the subset splitting algorithm, equivalent nodes should be merged and renamed using graphviz output. At this step, when merging and renaming nodes, it is necessary to pay attention to the in-degree and out-degree of all duplicate nodes. Only in this way can the edges of the new minimized DFA be avoided.
