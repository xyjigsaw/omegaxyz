Introduction

For LR grammar, we can automatically construct the corresponding LR analysis table. In order to construct an LR analysis table, we need to define an important concept - the standard sentence pattern of grammar "living prefix". A prefix that does not contain any symbols after such a handle is called a live prefix. At any time during the LR analysis process, the grammar symbols in the stack (from the bottom of the stack) X1X2…XmIt should form a live prefix, and after allocating the remaining parts of the input string, it should become a canonical sentence pattern (if the entire input string indeed forms a sentence). Therefore, as long as the scanned part of the input string remains reducible to a live prefix, it means that the scanned part is error-free. For a grammar G, we can construct a finite automaton that can identify all live prefixes of G, and then convert this automaton into an LR analysis table. Perform LR analysis according to the LR analysis table, which can ensure that during the analysis process, if the analyzed sentence is correct, the grammar symbols in the stack (from the bottom of the stack to the top) always constitute live prefixes. If each state (item set) in the living prefix recognition automaton of the extended grammar of a grammar G does not exist in the following situations: (1) contains both shift items and reduction items; (2) contains multiple reduction items, then G is said to be an LR(0) grammar. The set of states of the automaton is the normative family of LR(0) itemsets of the grammar. There are three ways to construct a grammar-identifying prefix DFA: (1) Find the regular expression of live prefix according to the formal definition, then construct NFA from this regular expression and then determine it as DFA; (2) Find all the items of the grammar, construct an NFA that identifies live prefixes according to certain rules, and then determine it into a DFA; (3) Use the closure function (CLOSURE) and the steering function (GO(I,X)) to construct the itemset specification family of LR(0) of the grammar G’, and then use the transition function to establish the connection relationship between states to obtain a DFA that identifies live prefixes.   The prefix of a symbol string refers to any header of the symbol string, including the empty string ε. For example, for the symbol string abc, its prefixes are ε, a, ab, abc. If the input string has no errors, the live prefix of a canonical sentence pattern is a prefix of the sentence pattern, but it does not contain any symbols after the handle. It is called a live prefix because a standard sentence pattern can be formed by concatenating the uninputted symbol string after the prefix. The relationship between live prefixes and handles is as follows: (1) The live prefix contains all the symbols of the handle, indicating that the right part β of the production A→β has appeared on the top of the stack. (2) The live prefix contains only part of the symbol of the handle, indicating A→β1β2The right substring β of1has appeared on the top of the stack, expecting to see β from the input string2Launch symbol. (3) The live prefix does not contain any symbol of the handle. At this time, the symbol string derived from the right part of A→β is expected. Each production formed by adding a dot anywhere on the right side (candidate) of each production in grammar G is called an LR(0) item. For example, the production A® xyz has the following items: A®.xyz, A®x.yz, A®xy.z, A®xyz. The productions marked with dots can be used to determine how much of the right-hand symbol of each production that characterizes the grammar under analysis has been recognized (appears on the top of the stack). (1) A→β. The right part β of the production A→β has appeared on the top of the stack. (3) A→.β marks any symbol without a handle on the top of the stack. At this time, the symbol string derived from the right part of A→β is expected. (4) For the LR(0) item of A→ε, there is only A→. . Let the grammar G = (VT，VN, S, P) is a context-free grammar. If there is a canonical derivation SAw12w (where A12P), then item A12 is said to be valid for live prefix = 1, that is, LR(0) valid item. In an intuitive sense, an LR(0) item indicates how much of the production we see at a certain step in the analysis process has been recognized. The dot in the LR(0) item can be seen as the dividing line between the top of the analysis stack and the input string. The left side of the dot is the part that has entered the analysis stack, and the right side is the symbol string that is currently input or continues to be scanned. Different LR(0) items reflect different situations at the top of the analysis stack. We divide LR(0) projects into four categories according to their different functions: (1) Reduction items: Expression form: A→a. This type of LR(0) item indicates that handle a happens to be contained in the stack, that is, part of the content at the top of the current stack constitutes the desired handle, and should be reduced as A→a. (2) Accepted projects: Expression: →a. Among them is the only starting symbol of grammar. This type of LR(0) item is actually a special reduction item, which means that the content in the analysis stack happens to be a, and if →a is used for reduction, the entire analysis will be successful. (3) Move in items: Expression form: A→a.(bVT) This type of LR(0) item indicates that the analysis stack is a live prefix that does not completely contain a handle. In order to form a live prefix that just has a handle, b needs to be moved into the analysis stack. (4) Items to be booked: Expression form: A→α.Bβ (BVN) This type of LR(0) item indicates that the analysis stack is a live prefix that does not completely contain a handle. In order to form a live prefix that just has a handle, the corresponding content in the current input string should be reduced to B first.   After giving the definition and classification of LR(0) items, we start from these LR(0) items to construct a finite automaton that can recognize all prefixes of the grammar. The steps are: first construct a non-deterministic finite automaton that can recognize all live prefixes of the grammar, then determinize and minimize it, and finally obtain the required deterministic finite automaton. A method of constructing a non-deterministic finite automaton that identifies all live prefixes of grammar G from the LR(0) items of grammar G: (1) It is stipulated that the first LR(0) item (i.e. →.A) of the production containing the grammar start symbol (let →A) be the only initial state of NFA. (2) Let all LR(0) items correspond to a state of NFA and the corresponding state of the LR(0) item as a reduction item is the final state. (3) If state i and state j come from the production of the same grammar G and the dots of the two states LR(0) items only differ by one position, that is: If i is X→X1X2·…Xi-1·Xi…Xn, j is X→X1X2…Xi·Xi+1…Xn, then lead a mark X from state iiarc to state j. (4) If state i is a pending item (assuming X→α·Aβ), then draw an ε arc from state i to all states A→·r.   In order to make the "accepted" state easy to identify, we usually extend the grammar G. Assume that grammar G is a grammar starting with S. We construct a grammar that contains the entire G, but it introduces a non-terminal symbol that does not appear in G, and adds a new production →S, starting with →S. Then, we call it the extended grammar of G. In this way, there will be a state containing only item →S, which is the only "accepting" state. If I is an itemset of grammar G', the closure CLOSURE(I) of I is defined and constructed as follows:
  • The items of I are all in CLOSURE(I).
  • If A→a.Bb belongs to CLOSURE(I), then every form is B→.The item of g also belongs to CLOSURE(I).
  • Repeat (2) until CLOSURE(I) no longer expands.
Define the conversion function as follows: GO(I,X)=CLOSURE(J) Among them: I is the state containing a certain item set, X is a grammar symbol, J={ A→aX .b | A→a.X b∈I}. The leftmost item whose dot is not on the right side of the production is called the core. The only exception is S′→.S. Therefore, J obtained by using the GOTO (I, X) state transition function is the core of the state closure item set after the turn. Use the closure function (CLOSURE) and the conversion function (GO(I,X)) to construct the itemset specification family of LR(0) of grammar G’. The steps are as follows:
  • Set the item S′→.S as the core of the initial state set, and then find the closure CLOSURE ({S′→.S}) on the core to obtain the closure item set of the initial state.
  • Apply the conversion function GO (I, X) = CLOSURE (J) to the initial state set or other constructed itemset to find the closure itemset of the new state J.
  • Repeat (2) until no new item sets appear.
Calculate the LR(0) itemset specification family C={I0，I1, ... In } algorithm pseudo code is as follows: Procedure itemsets(G’); Begin C := { CLOSURE ({S’®.S})} Repeat For every item set I and every grammar symbol X in C Do if GO(I,X) is not empty and does not belong to C Then put GO(I,X) into C Until C no longer increases End; An item set may contain multiple items. If move and reduce items exist at the same time, it is called a move-reduce conflict. If When reduction and reduction items exist at the same time, it is called a reduce-reduce conflict. Let’s look at a specific example:   We hope to build an LR parser based on a DFA that recognizes live prefixes of a grammar, so we need to study the different roles of the items in each item set (state) of this DFA. Let us say that project A→β1.β2Live prefix αβ1is valid provided that there is a normative derivation. In general, the same item may be valid for several live prefixes (this is the case when an item appears in several different collections). If the reduction item A→β1. is valid for live prefixes, then it tells us that the symbol string should be reduced to A, that is, the live prefix becomes αA. If you move into item A→β1.β2If the live prefix is ​​valid, it tells us that the handle has not yet been formed, so the next action should be to move in. However, there may be situations where several items are valid for the same live prefix. And they tell us different and conflicting things to do. This conflict may be resolved by looking ahead a few more input symbols. For each live prefix, we can construct its valid itemset. In fact, a valid itemset with live prefix γ is exactly the itemset (state) that is reached after reading γ starting from the initial state of the DFA mentioned above. In other words, at any time, the live prefix X in the analysis stack1X2…XmThe effective item set is exactly the top state S of the stackmthe set represented. This is a basic theorem of LR analysis theory. In fact, the item set (state) at the top of the stack embodies all useful information in the stack - history. We have defined the LR(0) grammar before. Let's take a look at how the LR(0) analysis table is constructed. For LR(0) grammar, we can directly construct the LR analysis table from its item set specification family C and the state transition function GO of the living prefix identification automaton. The following is the algorithm for constructing an LR(0) analysis table. Assume C={I0, I1,…,In}, let each item set IkThe subscript k is a state of the analyzer. Therefore, the LR(0) analysis table of G' contains states 0, 1,..., n. Let that contain the item S'→. S of IkThe subscript k is the initial state. ACTION subtables and GOTO subtables can be constructed as follows: (1) If item A→α.aβ belongs to IkAnd GO (Ik, a)= Ij, a is the terminal symbol, then set ACTION[k, a] to "move state j and symbol a into the stack", abbreviated as "sj"; (2) If item A→α. Belong to Ik, then, for any terminal a, set ACTION[k, a] to "use production A → α to reduce", abbreviated as "rj"; Among them, assume that A→α is the j-th production of grammar G'; (3) If item S'→S. Belong to Ik, then set ACTION[k, #] to "accept", abbreviated as "acc"; (4) If GO (Ik, A)= Ij, A is a non-terminal symbol, then set GOTO[k, A]=j; (5) Any blank cells in the analysis table that cannot be filled with information from 1 to 4 above will be marked with an "error mark". The analysis table containing ACTION and GOTO parts constructed according to the above algorithm, if each entry does not contain multiple definitions, is called an LR(0) analysis table of grammar G. A grammar G with an LR(0) table is called an LR(0) grammar, and an LR(0) grammar is unambiguous.

  • Data structure:This experiment is written in python language, and all data structures are represented by lists.
Including grammar, item set itemSet, DFA status, terminal symbols and terminal symbols, and two-dimensional array LRO analysis table LR0TABLE
  • closure function closure,The input of the closure function is a project. The item set of a project is found by reading the global variable itemset without using recursion. It determines whether the closure changes each time. If it does not change, it means the end.
  • Gotofunction,Enter the current project and converted letters, and obtain the converted state by moving the point back one position. If GOTO fails, -1 is returned.

grammar = []

def closure(item):
    global itemSet
    dot = []
    dot.append(item)
    olddot = []
    while len(dot) != len(olddot):
        olddot = copy.deepcopy(dot)
        temp = []
        for i in range(len(dot)):
            for j in range(len(itemSet)):
                if dot[i].index('·')+1 < len(dot[i]) and dot[i][dot[i].index('·')+1] == itemSet[j][0] and itemSet[j][itemSet[j].index('>')+1] == '·':
                    temp.append(itemSet[j])
        for k in range(len(temp)):
            if temp[k] not in dot:
                dot.append(temp[k])
    return dot


def goto(item, a):
    global itemSet
    for i in range(len(item)):
        if item[i] == '·' and i!= len(item)-1:
            if item[i+1] == a:
                item2 = item[:i]+item[i+1]+'·'+item[i+2:]
                if item2 in itemSet:
                    return item2
    return -1


def findItem(item):
    global DFA
    for i in range(len(DFA)):
        if item in DFA[i]:
            return i
    return -1

Problems and difficulties encountered

  • In this experiment, we need to write the closure function and the goto function. There are two methods: one is to use recursion, and the other is to use the while statement. Each time an item is written into the collection to obtain closure until no changes occur.
  • The second difficulty is to draw the analysis table based on DFA. In this experiment, DFA nodes and edges are stored separately, and the index function is used to find the corresponding edges when drawing the table.
  • The third difficulty is to determine whether the sentence pattern is LR (0) grammar. Since it is a simulation, in the experiment, not only the analysis table needs to be known, but also the original grammar, terminal symbols and non-terminal symbols. In the experiment, the method of separating ACTION judgment and GOTO judgment was adopted.
