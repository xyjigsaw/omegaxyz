Application of hash table (hash table)

Programming LanguagesC & C++Data StructuresTechnologyC++Data StructuresPAT
Hash table (also called hash table), is a data structure that is directly accessed based on the key value. That is, it accesses records by mapping the key value to a location in the table,to speed up the search. This mapping function is calledhash function, the array that stores records is called a hash table. Given a table M, there is a function f(key). For any given keyword value key, if the address of the record containing the keyword in the table can be obtained after substituting the function, the table M is called a hash (Hash) table, and the function f(key) is a hash (Hash) function.

conflict resolution

"Handling conflicts" means: finding the next hash address for the conflicting keyword. There are usually two types of methods to deal with conflicts: Open Addressing and Chaining. The former stores all nodes in the hash table T[0..m-1]; the latter usually links nodes that are synonymous with each other into a singly linked list, and places the head pointer of this linked list in the hash table T[0..m-1].

1 Open address method

This methodBasic ideaYes: When an address conflict occurs, continue to detect other storage units in the hash table according to a certain method until an empty location is found. This process can be described by the following formula: H i ( key ) = ( H ( key ) + d i ) mod m ( i = 1,2,… , k ( k ≤ m – 1)) Among them: H (key) is the direct hash address of the keyword key, m is the length of the hash table, and di is the address increment for each re-detection. When using this method, first calculate the direct hash address H (key) of the element. If the storage unit is already occupied by other elements, continue to look at the storage unit with the address H (key) + d 2. Repeat this until a storage unit is found to be empty, and then store the data element with the key key in that unit. The increment d can be taken in different ways and has different names depending on how it is taken: (1) d i = 1, 2, 3, ... Linear detection and then hashing; (2) d i = 1^2, - 1^2, 2^2, - 2^2, k^2, -k^2... Secondary detection and then hashing; (3) d i = pseudo-random sequence, pseudo-random rehash;Example 1Suppose the hash function H (key) = key mod 7, and the address space of the hash table is 0 ~ 6. For the key sequence (32, 13, 49, 55, 22, 38, 21), the hash table is constructed by linear detection and then hashing and secondary detection and then hashing.①Linear Probing The basic idea of this method is:Consider the hash table T[0..m-1] as a circular vector. If the initial probe address is d (i.e. h(key)=d), then the longest probe sequence is: d, d+l, d+2,…,m-1,0,1,…,d-1 That is: when probing, start from address d, first probe T[d], then probe T[d+1],..., until T[m-1], and then loop to T[0], T[1],..., until T[d-1] is probed.The exploration process terminates in three situations:(1) If the currently probed unit is empty, it means that the search failed (if it is inserted, the key will be written into it); (2) If the currently probed unit contains key, the search is successful, but the insertion fails; (3) If no empty unit or key is found when T[d-1] is detected, both search and insertion will mean failure (the table is full at this time).aggregation or accumulationWhen using the linear probing method to resolve conflicts, when there are nodes at positions i, i+1,..., i+k in the table, a node with a hash address of i, i+1,..., i+k+1 will be inserted at position i+k+1. This phenomenon of nodes with different hash addresses competing for the same subsequent hash address is called aggregation or clustering. This will cause nodes that are not synonyms to be in the same exploration sequence, thereby increasing the length of the exploration sequence, that is, increasing the search time. If the hash function is not good or the filling factor is too large, the accumulation phenomenon will be aggravated.② Quadratic ProbingThe exploration sequence of the secondary exploration method is: hi=(h(key)+i*i)%m 0≤i≤m-1 //That is di=i2That is, the detection sequence is d=h(key), d+12，d+22,…,wait. The disadvantage of this method is that it is not easy to explore the entire hash space.③Double HashingThis method is one of the best methods of open addressing. Its probing sequence is: hi=(h(key)+i*h1(key))%m 0≤i≤m-1 //That is di=i*h1(key) That is, the exploration sequence is: d=h(key), (d+h1(key))%m, (d+2h1(key))%m,…, etc. This method uses two hash functions h(key) and h1(key), so it is also called the double hash function detection method. Note: There are many ways to define h1(key), but no matter which method is used to define it, the value of h1(key) must be relatively prime with m, so that the conflicting synonym addresses can be evenly distributed in the entire table, otherwise it may cause cyclic calculation of synonym addresses. [Example] If m is a prime number, then h1(key) takes any number between 1 and m-1 and is relatively prime with m. Therefore, we can simply define it as: h1(key)=key%(m-2)+1 [Example] For example 9.1, we can take h(key)=key%13, and h1(key)=key%11+1. [Example] If m is a power of 2, then h1 (key) can be any odd number between 1 and m-1.

2. Zipper method

①The zipper method to resolve conflictsThe zipper method solves conflicts by linking all nodes whose keywords are synonyms in the same singly linked list. If the length of the selected hash table is m, the hash table can be defined as a pointer array T[0..m-1] consisting of m head pointers. All nodes with hash address i are inserted into a singly linked list with T[i] as the head pointer. The initial value of each component in T should be a null pointer. In the zipper method, the filling factor α can be greater than 1, but generally α ≤ 1. ② Advantages of zipper methodCompared with the open addressing method, the zipper method has the following advantages: (1) The zipper method is simple to deal with conflicts and has no accumulation phenomenon, that is, non-synonymous words will never conflict, so the average search length is shorter; (2) Since the node space on each linked list in the zipper method is dynamically applied for, it is more suitable for situations where the table length cannot be determined before creating the table; (3) In order to reduce conflicts, the open addressing method requires the filling factor α to be small, so when the node size is large, a lot of space will be wasted. In the zipper method, α ≥ 1 is acceptable, and when the nodes are large, the added pointer domain in the zipper method can be ignored, thus saving space; (4) In a hash table constructed using the zipper method, the operation of deleting nodes is easy to implement. Just simply delete the corresponding node on the linked list. For the hash table constructed by the open address method, deleting a node cannot simply empty the space of the deleted node, otherwise the search path of the synonym node that is filled in the hash table after it will be truncated. This is because in various open address methods, empty address units (ie open addresses) are conditions for search failure. Therefore, when you perform a deletion operation on a hash table that uses the open address method to handle conflicts, you can only mark the deleted node for deletion, but you cannot actually delete the node.③Disadvantages of the zipper methodThe disadvantage of the zipper method is that pointers require additional space, so when the node size is small, the open addressing method is more space-saving. If the saved pointer space is used to expand the size of the hash table, the filling factor can be made smaller, which in turn reduces conflicts in the open addressing method, thus increasing the average search speed.

Application (PAT A 1078)

1078 Hashing (25 points)
The task of this problem is simple: insert a sequence of distinct positive integers into a hash table, and output the positions of the input numbers. The hash function is defined to be H(key)=key%TSize where TSize is the maximum size of the hash table. Quadratic probing (with positive increments only) is used to solve the collisions. Note that the table size is better to be prime. If the maximum size given by the user is not prime, you must re-define the table size to be the smallest prime number which is larger than the size given by the user.

Input Specification:

Each input file contains one test case. For each case, the first line contains two positive numbers: MSize (≤10​4​​) and N(≤MSize) which are the user-defined table size and the number of input numbers, respectively. Then N distinct positive integers are given in the next line. All the numbers in a line are separated by a space.

Output Specification:

For each test case, print the corresponding positions (index starts from 0) of the input numbers in one line. All the numbers in a line are separated by a space, and there must be no extra space at the end of the line. In case it is impossible to insert the number, print "-" instead.

Sample Input:

4 4
10 6 4 15

Sample Output:

0 1 4 -
C++ code: 
#include<iostream>
#include<cstdio>
#include<vector>
using namespace std;
 
int m,n;
vector<int> arr,flag;
bool isprime(int n){
	if(n<=1)
		return false;
	for(int i=2;i*i<=n;i++)
		if(n % i == 0)
			return false;
	return true;
}
 
void insert(int num){
	for(int j=0;j<n;j++){
		int key = (num+j*j)%m;
		if(flag[key] == 0){
			flag[key] = 1;
			printf("%d",key);
			return ;
		}
	}
	printf("-");
}
 
int main(){
	cin>>m>>n;
	while(!isprime(m))m++;
	
	arr.resize(n,0);
	for(int i=0;i<n;i++)
		cin>>arr[i];
	
	flag.resize(m,0);
	for(int i=0;i<n;i++){
		insert(arr[i]);
		if(i != n-1)
			printf(" ");
	}
	return 0;
}
