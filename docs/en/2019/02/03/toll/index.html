Eliminate left recursion in a grammar

Programming LanguagesC & C++Technology编译原理C++编译原理

Introduction

1. Elimination of direct left recursion

It is relatively easy to eliminate direct left recursion in productions. For example, suppose the rule for nonterminal P is

P→Pα / β

Among them, β is a symbol string that does not start with P. Then, we can rewrite the rule of P into the following indirect left-recursive form:

P→βP’

 P’→αP’ / ε

These two rules are equivalent to the original rules, that is, the symbol strings deduced from P in the two forms are the same. Let there be a simple expression grammar G[E]:

E→E+T/ T

T→T*F/ F

F→（E）/ I

After eliminating direct left recursion, we get the following grammar:

E→TE’

E’ →+TE’/ ε

T→FT’

T’ →*FT’/ ε

F→（E）/ I

Considering the more general case, suppose that the rule for nonterminal P isP→Pα1 / Pα2 /…/ Pαn / β1 / β2 /…/βmAmong them, αi(I=1, 2,...,n) are not ε, and each βj(j=1, 2,...,m) do not start with P. Rewriting the above rules into the following form can eliminate the direct left recursion of P:P→β1 P’ / β2 P’ /…/βm P’ P’ →α1P’ / α2 P’ /…/ αn P’ /ε

2. Elimination of indirect left recursion

The way to eliminate indirect left recursion is to rewrite the indirect left recursive grammar into a direct left recursive grammar, and then rewrite the grammar using a method to eliminate direct left recursion. If a grammar does not contain a cycle, that is, a derivation of the form PP, nor a production with ε as the right-hand side, then the following algorithm can be used to eliminate all left recursion of the grammar. Eliminate left recursive algorithm:
  • Arrange all non-terminal symbols of grammar G in any order, for example, A1，A2，…，An。
for (i＝1;i<=n;i++) for (j=1; j<=i-1; j++) { Let the shape be like Ai→AjThe production of γ is rewritten as Ai→δ1γ /δ2γ /…/δkγ Among them Aj→δ1 /δ2 /…/δkis about AjAll rules; Eliminate AiDirect left recursion in rules; }
  • Simplify the grammar obtained by (2), that is, remove redundant rules.
This algorithm can be used to rewrite the above grammar to eliminate left recursion. First, let the order of nonterminal symbols be R, Q, S. For R, there is no direct left recursion. Substituting R into the relevant rules in Q, the rules of Q become Q→Sab/ ab/ b. The substituted Q does not contain direct left recursion. Substituting it into S, the rule of S becomes S→Sabc/abc/bc/c. At this time, S has direct left recursion. After eliminating the direct left recursion of S, the entire grammar is obtained:

S→abcS’/ bcS'/ cS'

S’ →abcS'/ ε

Q→Sab/ ab/ b

R→Sa/ a

It can be seen that starting from the starting symbol S of the grammar, Q and R can never be reached, so the rules about Q and R are redundant. They are deleted and simplified. Finally, the grammar G[S] is obtained:

S→abcS'/ bcS’/ cS'

S' →abcS'/ ε

Of course, if the non-terminal symbols of the grammar are sorted differently, the final grammar may be different in form, but they are all equivalent. For example, if the order of the above non-terminal symbols is selected as S, Q, R, then the final grammar G[R] obtained is:

R→bcaR'/ caR'/ aR’

R' →bcaR'/ ε

It is easy to prove that the above two grammars are equivalent. Indicates whether there is left recursion and the type of left recursion. For direct left recursion, it can be changed to direct right recursion; for indirect left recursion (also called grammatical left recursion), the equivalent grammar after eliminating left recursion with different arrangements of non-terminal symbols should be given according to the algorithm. (There should be n! kinds)

c++ code

Written in C++, there are three modules in total. The first module converts introductory left recursion into direct left recursion, the second module eliminates direct left recursion, and the last module is the main function module.

string DFS(string start,string *principle,int count,int temp){
    string x = "";
    temp++;
    for(int i=0;i<start.length();i++){
        if(start[i]>='A'&&start[i]<='Z'&&temp<count){
            x+=start[i];
            for(int j = 0;j<count-1;j++){
                if(principle[j][0]==start[i]){
                    x+=DFS(principle[j],principle,count,temp);
                }
            }
        }
    }
    return x;
}


void removeDirect(int i, int count2){
    //消除直接左递归，即将形如A->Ab|c 的转化为 A->cA'和A'->bA'|~
    string p1="",p2="";
    size_t flag1=3,flag2=0;
    char ch=principle[i][0];
    bool last=false; //判断是否结束
    while(flag1!=string::npos) //寻找文法中的|符号，如果h找不到则退出
    {
        flag2=principle[i].find_first_of("|",flag1+1); //在文法中找到第flag+1个|
        if(flag2==string::npos)flag2=principle[i].length(); //只有一个|
        if(principle[i][flag1]==ch)
        {
            last=true;
            p1+=principle[i].substr(flag1+1,flag2-flag1-1)+ch+"\'|";//加上‘
        }
        else
        {
            p2+=principle[i].substr(flag1,flag2-flag1)+ch+"\'|";//加上’
        }
        flag1=principle[i].find_first_not_of("|",flag2+1);
    }
    p2[p2.length()-1]='\0';
    if(last)  //结束时加上空~
    {
        temp[count2]=ch+("\'->"+p1+"~");
        count2++;
        principle[i].replace(3,principle[i].length()-3,p2);
    }
}


void removeIndirect(int i,int j){
    //间接左递归变直接左递归
    int start=2;
    char aj=principle[j][0];
    //修改产生式
    bool rgt=false;
    int count1=0;
    string tt[MAX_SIZE];
    size_t s=0,e=0;
    do
    {
        start++;
        if(principle[i][start]==aj)//如果满足Ai->Aj*
        {
            size_t es=principle[i].find_first_of("|",start+1);
            if(es==string::npos)
                es=principle[i].length();
            string te=principle[i].substr(start+1,es-start-1);
            if(!rgt)
            {
                s=principle[j].find_first_not_of("|",3);
                while(s!=string::npos)
                {
                    e=principle[j].find_first_of("|",s+1);
                    if(e==string::npos)
                        e=principle[j].length();
                    tt[count1]=principle[j].substr(s,e-s);
                    count1++;
                    s=principle[j].find_first_not_of("|",e+1);
                }
                rgt=true;
            }
            int k=0;
            string ttl="\0";
            for(;k<count1-1;k++)
                ttl+=tt[k]+te+"|";
            ttl+=tt[k]+te;
            principle[i].replace(start,es-start,ttl);
        }
        start = principle[i].find_first_of("|",start+1);
    }while(start!=string::npos);
}

Difficulties encountered and solutions

Since grammars come in various forms, various situations must be taken into consideration when eliminating recursion. Generally speaking, the unified grammar format must be solved first, so grammars with the same left part of non-terminal symbols need to be merged with the | symbol. Next, we need to solve the problem of indirect left recursion, so we convert the indirect left recursion into direct left recursion. Finally direct left recursion will be eliminated. There are two quantities to be judged during the elimination process,One is the position of |, the other is the position of the non-terminal symbol, since there are multiple |s in the merged grammar string, and a new converted grammar will be generated, it needs to be processed with a while statement until the form of all grammars no longer changes. The second problem is that after eliminating the left-recursive grammar, some non-terminal symbols and their productions are useless, so they need to be removed. Use DFS to detect non-terminal symbols starting from the start symbol S, which can eventually solve this problem.
