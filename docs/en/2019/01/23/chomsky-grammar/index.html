Introduction to grammar

1．0Type grammar (phrase grammar)If for some grammar G, each rule in P has the following form:u:: = vAmong them, u∈V＋, v∈V*, then the grammar G is called a type 0 grammar or phrase grammar, abbreviated as PSG. The corresponding language of type 0 grammar or phrase structure grammar is called type 0 language or phrase structure language L0. Since this kind of grammar has no other restrictions, Type 0 grammar is also called an unrestricted grammar, and its corresponding language is called an unrestricted language. Any type 0 language is recursively enumerable, so type 0 language is also called a recursively enumerable set. This language can be recognized by a Turing machine.2．1Type Grammar (Context-sensitive Grammar)If for some grammar G, each rule in P has the following form:xUy:: = xuyAmong them, U∈VN; u∈V＋; The rule of type 1 grammar is that the U on the left and the u on the right have the same upper x and lower y. When using this rule for derivation, if you want to replace U with u, you must do it when there is x in front and y in the back, which shows the context-related characteristics. The language determined by type 1 grammar is type 1 language L1, and type 1 language can be recognized by linear bounded automaton.3．2type grammar (context-free grammar)If for some grammar G, each rule in P has the following form:U :: = uAmong them, U∈VN; u∈V+, then the grammar G is called a type 2 grammar or a context-free grammar, abbreviated as CFG. According to this rule, for context-free grammar, when using this rule for derivation, there is no need to consider the context in which the non-terminal symbol U is located. U can always be replaced with u, or u can be reduced to U, which shows the characteristics of context-free grammar. The language determined by Type 2 grammar is Type 2 language L2, and Type 2 language can be recognized by a non-deterministic pushdown automaton. The grammar that generally defines a programming language is context-free. This is the case with C language. Therefore, context-free grammars and corresponding languages ​​have attracted greater interest and attention.4．3Type grammar (regular grammar, linear grammar)If for some grammar G, each rule in P has the following form:U :: = T  or  U :: = WTAmong them, T∈VT; U, W∈VN, then the grammar G is called a left-linear grammar. If for some grammar G, each rule in P has the following form:U :: = T  or  U :: = TWAmong them, T∈VT; U, W∈VN, then the grammar G is called a right linear grammar. Left linear grammar and right linear grammar are commonly called type 3 grammar or regular grammar, sometimes also called finite state grammar, abbreviated as RG. By definition, when applying the rules to a regular grammar, a single nonterminal symbol can only be replaced by a single terminal symbol, or by a single nonterminal symbol plus a single terminal symbol, or by a single terminal symbol plus a single nonterminal symbol. The language determined by type 3 grammar is type 3 language L3, and type 3 language can be identified by a certain finite state automaton. In common programming languages, most lexical-related grammars belong to Type 3 grammars. It can be seen that the above four types of grammars, from type 0 to type 3, have increasingly stronger production restrictions. The latter types are all subsets of the previous type, and the function of describing language is getting weaker and weaker. The relationship between the four types of grammars and the languages they represent can be expressed as: Type 0 Type 1 Type 2 Type 3; that is, L0 L1 L2 L3

Grammar data structure:Considering that the grammar is a four-tuple, including Vn as a non-terminal symbol, Vt as a terminal symbol, P as a rule of the grammar, S as an identifier or start symbol, and flag as the type of grammar, so the following class in C++ is used to define the grammar, and a set collection is used to save certain attributes of each grammar (not repeated).
#include<cstdio>
#include<iostream>
#include<string>
#include<vector>
#include<set>
#define MAX_LENGTH 10
using namespace std;


struct Principle{
    string left;
    string right;
};

struct Grammar{
    set<char> Vn;
    set<char> Vt;
    vector<Principle> P;
    char S;
    int flag = -1;
};

void input(vector<Principle> &principleSet)
{
    char tempLeft[MAX_LENGTH];
    char tempRight[MAX_LENGTH];
    while(scanf("%[^-]->%s",tempLeft,tempRight)!=EOF){
        getchar();
        Principle temp;
        temp.left = tempLeft;
        temp.right = tempRight;
        principleSet.push_back(temp);
    }
}

void analyseGram(Grammar &G)
{
    G.S = G.P.front().left[0];//获得起始符
    for(int i=0;i<G.P.size();++i){
        Principle &temp = G.P[i];
        for(int j = 0;j<temp.left.size();++j){
            char v = temp.left[j];
            if(!isupper(v)){
                G.Vt.insert(v);
            }else{
                G.Vn.insert(v);
            }
        }
        for(int j = 0;j<temp.right.size();++j){
            char v = temp.right[j];
            if(!isupper(v)){
                G.Vt.insert(v);
            }else{
                G.Vn.insert(v);
            }
        }
    }
}

bool existVNT(string s)
{
    for(int i=0;i<s.size();i++){
        if(isupper(s[i])){
            return true;
        }
    }
    return false;
}

void judgeGram(Grammar &G)
{

    for(int i=0;i<G.P.size();++i){
        if(existVNT(G.P[i].left)){
            G.flag = 0;
            break;
        }
    }
    int temp = 0;
    for(int i=0;i<G.P.size()&&G.flag==0;++i){
        if(G.P[i].left.size()<=G.P[i].right.size()){
            temp++;
        }
    }
    if(temp==G.P.size()){
        G.flag=1;
    }
    temp = 0;
    for(int i=0;i<G.P.size()&&G.flag==1;++i){
        if(1==G.P[i].left.size()&&isupper(G.P[i].left[0])){
            temp++;
        }
    }
    if(temp==G.P.size()){
        G.flag=2;
    }
    temp = 0;
    for(int i=0;i<G.P.size()&&G.flag==2;++i){
        if(1==G.P[i].left.size()&&isupper(G.P[i].left[0])&&
           (1==G.P[i].right.size()&&!isupper(G.P[i].right[0])||
            2==G.P[i].right.size()&&!isupper(G.P[i].right[0])&&
            isupper(G.P[i].right[G.P[i].right.size()-1]))){temp++;}
    }
    if(temp==G.P.size()){
        G.flag=3;
    }
}

void printResult(Grammar &G)
{
    cout<<"G=(Vn,Vt,P,S)是"<<G.flag<<"型文法"<<endl<<endl;
    cout<<"非终结符Vn有：";
    for(set<char>::iterator i = G.Vn.begin();i!=G.Vn.end();++i){
        cout<<*i<<" ";
    }
    cout<<endl;
    cout<<"终结符Vt有：";
    for(set<char>::iterator i = G.Vt.begin();i!=G.Vt.end();++i){
        cout<<*i<<" ";
    }
    cout<<endl;
    cout<<"开始符为："<<G.S<<endl;
    cout<<"规则P如输入所示"<<endl;
    cout<<endl;
}

int main()
{
    Grammar G;
    G.Vn.clear();
    G.Vt.clear();
    input(G.P);
    G.flag = 3;
    analyseGram(G);
    judgeGram(G);
    printResult(G);
}

Difficulties and Solutions

  • Creation of data structures
In order to facilitate code reuse in future experiments, a good data structure type needs to be established. Therefore, I wrote this experiment in C++ and used containers in C++, such as set and vector.
  • How to determine grammatical type
This part is the focus of this experiment. How to effectively judge the type of grammar is a difficult problem. After analysis, I decided to judge the type of grammar from top to bottom, from low to high. First determine whether it is a low-level grammar, and then determine whether it is a high-level grammar. Many branch statements appear during the judgment process, so certain modules can be proposed, such as the non-terminal judgment module, which can be integrated into functionsbool existVNT(string s)。
  • If the format of the input in the initial experiment is wrong, we can have two solutions to this problem. One is to immediately judge whether it is a legal rule when inputting, or to output an error message when judging the grammar type.
