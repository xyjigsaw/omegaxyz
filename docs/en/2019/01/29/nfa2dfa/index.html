Compilation principle NFA to DFA-Python and C++ implementation

Programming LanguagesC & C++TechnologyPython编译原理C++CPython编译原理
A certain finite automaton, if the successor state of each state transition is unique, is called a deterministic finite automaton (DFA); if the successor states after the transition are not all unique, it is called an indeterminate finite automaton (NFA). The starting working state of a finite automaton is called the initial state, and the ending working state is called the terminal working state or receiving state. If each node of the state transition diagram in the previous section is regarded as a certain state, the initial node is the initial state, the termination node is the termination state, and each edge represents a conversion relationship, then the working state of a finite automaton can be described by a state transition diagram, and the previous diagram can be regarded as a finite automaton. The finite automaton is in the initial state 0. When the symbol a is read, the automaton transitions from state 0 to the subsequent state 1. After another symbol b is read, the automaton transitions from state 1 to the subsequent state 2. When the automaton reads a symbol string, the automaton starts from the initial state, goes through a series of state transitions, and finally reaches the terminal state, then the symbol string is said to be received or recognized by the automaton, otherwise it cannot be received by the automaton.A deterministic finite automaton (DFA) M can be defined as a five-tuple, M = (K, ∑, F, S, Z), where:
  • K is a finite non-empty set, and each element in the set is called a state;
  • ∑ is a finite alphabet, and each element in ∑ is called an input symbol;
  • F is a single-valued conversion function from K×∑→K, that is, F (R, a) = Q, (R, Q∈K) means that the current state is R. If the character a is input, it will go to state Q. State Q is called the successor state of state R;
  • S∈K is the only initial state;
  • ZK, is a final state set.
It can be seen from the definition that a finite automaton has only one initial state, but it can have multiple final states. Each state has at most one successor state for any input symbol in the alphabet.  For DFA M, if there is a path from an initial state node to a certain final state node, then it is said that the string formed by connecting the markers of all arcs on this path can be accepted by DFA M. If the initial state node of M is also the final state node, it is said that ε can be accepted (or recognized) by M, and the set of all strings (words) that DFA M can accept is recorded as L (M).An uncertain finite automaton (NFA) M can be defined as a five-tuple, M = (K, ∑, F, S, Z), where:
  • k is a finite non-empty set, and each element in the set is called a state;
  • ∑ is a finite alphabet, and each element in ∑ is called an input symbol;
  • F is a transformation function from a subset of K×∑→K;
  • SK, is a non-empty initial state set;
  • ZK, is a final state set.
It can be seen from the definition that the main difference between an uncertain finite automaton NFA and a deterministic finite automaton DFA is: (1) The initial state S of NFA is a state set, that is, multiple initial states are allowed; (2) NFA allows states to have the same symbol on an output edge, that is, there can be multiple successor states for the same input symbol. That is, F in DFA is a single-valued function, while F in NFA is a multi-valued function. Therefore, the deterministic finite automaton DFA can be regarded as a special case of the uncertain finite automata NFA. Like DFA, NFA can also be represented by matrices and state transition diagrams. For NFA M, if there is a path from an initial state node to a certain final state node, then it is said that the string formed by connecting the marks of all arcs on this path (except ε) can be accepted by M. The set of all strings (words) that NFA M can accept is denoted as L(M). Since DFA is a special case of NFA, any symbol string that can be accepted by DFA must be accepted by NFA. Let M1and M2is a finite automaton on the same letter set Σ, if L(M1）＝L（M2), then it is called a finite automaton M1and M2equivalence. From the above definition, we can see that if two automata can accept the same language, they are said to be equivalent. DFA is a special case of NFA, so for every NFA M1There is always a DFA M2, making L(M1）＝L（M2). That is, a language that can be accepted by an indeterminate finite automaton can always find an equivalent deterministic finite automaton that can accept the language.NFADeterminize as DFAThe same string α can be generated by multiple paths. In practical applications, as an automaton that describes the control process, it is usually a deterministic finite automaton DFA. Therefore, it is necessary to convert the uncertain finite automaton into an equivalent deterministic finite automaton. This process is called the determinization of the uncertain finite automaton, that is, the deterministic transformation of NFA into DFA. The following introduces a deterministic algorithm for NFA, which is called the subset method:
  • If all initial states of NFA are S1，S2，…，Sn, then let the initial state of DFA be:
S＝[S1，S2，…，Sn], The square brackets are used to indicate a state composed of several states.
  • Suppose there is a state in the state set K of DFA [Si，Si+1，…，Sj], if for a certain symbol a∈∑, there is F ({ Si，Si+1，…，Sj }，a）={ Si’，Si+1’，…，Sk’ }
Then let F ({ Si，Si+1，…，Sj }，a）={ Si’，Si+1’，…，Sk’} is a conversion function of DFA. If [ Si’，Si+1’，…，Sk‘] is not in K, then add it to K as a new state.
  • Repeat step 2 until no new states are added to K.
  • All the states obtained above constitute the state set K of DFA, the transformation function constitutes F of DFA, and the alphabet of DFA is still the alphabet Σ of NFA.
  • Any state in DFA that contains the final state of NFA is the final state of DFA.
For the above-mentioned NFA deterministic algorithm-subset method, another more operable description method can also be used, and we give its detailed description below. First, two relevant definitions are given. Assuming that I is a subset of the NFA M state set K (i.e. I∈K), then define ε-closure (I) as:
  • If Q∈I, then Q∈ε-closure(I);
  • If Q∈I, then any state Q’ that can be reached by passing through any ε arc starting from Q, then Q’∈ε-closure (I).
The state set ε-closure (I) is called the ε closure of state I. Assume NFA M = (K, ∑, F, S, Z), if I∈K, a∈∑, then define Ia=ε-closure(J), where J is the set of all states starting from ε-closure(I) and arriving through an arc a. The essence of NFA determinism is to use a subset of the original state set as a state on DFA, and convert the conversion between original states into a conversion between subsets, thereby determinizing the uncertain finite automaton. After determinization, the number of states may increase, and some equivalent states may appear, in which case simplification is required.

    for(int i=0;i<num;i++){

void outputNFA(vector<Part> NFA){
    for(int i=0;i<NFA.size();i++){
        cout<<NFA[i].src<<" "<<NFA[i].edge<<" "<<NFA[i].dst<<endl;
    }
}

bool check(string str, char x){
    for(int i=0;i<str.length();i++){
        if(str[i]==x)
            return true;
    }
    return false;
}

string closure(string setp, vector<Part> NFA){
    for(int i=0;i<setp.length();i++){
        for(int j=0;j<NFA.size();j++){
            if(setp[i]==NFA[j].src[0]&&NFA[j].edge[0]=='&'){
                if(!check(setp,NFA[j].dst[0])){
                    setp+=NFA[j].dst[0];
                }
            }
        }
    }
    return setp;
}

string move(string t, char a, vector<Part> NFA){
    string temp="";
    for(int i=0;i<t.length();i++){
        for(int j=0;j<NFA.size();j++){
            if(t[i]==NFA[j].src[0]&&NFA[j].edge[0]==a){
                if(!check(temp,NFA[j].dst[0])){
                    temp+=NFA[j].dst[0];
                }
            }
        }
    }
    return temp;
}

bool checkINrawDFA(vector<string> rawDFA, string u){
    for(int i=0;i<rawDFA.size();i++){
        if(rawDFA[i]==u)
            return true;
    }
    return false;
}

int checkFlag(vector<bool> t){
    for(int i=0;i<t.size();i++){
        if(t[i]==false)return i;
    }
    return -1;
}

from graphviz import Digraph

class Part:
    def __init__(self, src, edge, dst):
        self.src = src
        self.edge = edge
        self.dst = dst


def check(str, x):
    for i in range(len(str)):
        if str[i]==x:
            return True
    return False


def closure(setp, NFA):
    t = len(setp)
    i = 0
    while i < t:
        for j in range(len(NFA)):
            if setp[i] == NFA[j].src[0] and NFA[j].edge[0]=='&':
                if not check(setp, NFA[j].dst[0]):
                    setp += NFA[j].dst[0]
                    t += 1
        i += 1
    return setp


def move(t, a, NFA):
    temp = ""
    for i in range(len(t)):
        for j in range(len(NFA)):
            if t[i] == NFA[j].src[0] and NFA[j].edge[0]==a:
                if not check(temp, NFA[j].dst[0]):
                    temp += NFA[j].dst[0]
    return temp


def checkINrawDFA(rawDFA, u):
    for i in range(len(rawDFA)):
        if rawDFA[i] == u:
            return True
    return False


def checkFlag(t):
    for i in range(len(t)):
        if t[i]==False:
            return i
    return -1

def checkINdex(rawDFA, u):
    for i in range(len(rawDFA)):
        if rawDFA[i] == u:
            return i
    return -1

difficulty

The basic algorithm is the subset method, and the core functions are the move and closure functions. Problems encountered:
  • How to construct the data structure of the subset so that it can be easily and quickly converted from NFA to DFA. The solution is to use strings as subsets, such as "abc" which represents the set of states a, b and c.
  • The numbering problem of the generated DFA needs to be dynamically numbered according to the results of Move and closure.
  • For the graphical interface display, I used networkx in python at first, but finally found that networkx could not draw self-loops, so I finally switched to graphviz for graphical interface display.
