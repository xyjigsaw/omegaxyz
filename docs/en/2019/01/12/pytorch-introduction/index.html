Getting started with PyTorch

PyTorch is a Python package built on top of the Torch library designed to accelerate deep learning applications. PyTorch provides a NumPy-like abstraction to represent tensors (or multidimensional arrays) that can take advantage of GPUs to accelerate training. PyTorch can be regarded as numpy with GPU support, and also as a powerful deep neural network with automatic derivation function. In addition to Facebook, it has also been adopted by organizations such as Twitter, CMU and Salesforce.

Basic knowledge

The difference between scalars, vectors, and tensors is very clear in the figure below. The key data structure of PyTorch is tensors, which are multi-dimensional arrays. Its function is similar to NumPy's ndarray object. As follows, we can use torch.Tensor() to create a tensor.
# Generate a 2-D pytorch tensor (i.e., a matrix)
pytorch_tensor = torch.Tensor(10, 20)
print("type: ", type(pytorch_tensor), " and size: ", pytorch_tensor.shape )
If you need a NumPy-compatible representation, or you want to create a PyTorch tensor from an existing NumPy object.
# Convert the pytorch tensor to a numpy array:
numpy_tensor = pytorch_tensor.numpy()
print("type: ", type(numpy_tensor), " and size: ", numpy_tensor.shape) 
# Convert the numpy array to Pytorch Tensor:
print("type: ", type(torch.Tensor(numpy_tensor)), " and size: ", torch.Tensor(numpy_tensor).shape)

PyTorch and TensorFlow

For now, TensorFlow is clearly preferred over PyTorch for various reasons. TensorFlow is large, experienced and best suited for practical applications. is the obvious choice for most machine learning and deep learning experts because of the vast number of features it offers and most importantly, its proven application in the market. It has better community support and multi-language API is available. It has a good documentation base and makes it easy to produce thanks to the code from preparation to use. Therefore, it is more suitable for people who want to start deep learning, or organizations looking to develop deep learning models. Although PyTorch is relatively new and has a small community, it is fast and efficient. In short, it gives you all the advantages of Python's usefulness and ease of use. Due to its efficiency and speed, it is a great choice for small research-based projects. As mentioned earlier, companies like Facebook, Twitter, and others are using PyTorch to train deep learning models. However, its use has not yet become mainstream. PyTorch's potential is obvious, but it's not ready to take on the TensorFlow beast. However, given its growth, the day is not far away when PyTorch will be further optimized and offer more features until comparisons with TensorFlow can be made.

pytorch entry code

import torch

dtype = torch.FloatTensor
# dtype = torch.cuda.FloatTensor # Uncomment this to run GPU

# N is batch size; D_in is input dimenssion
# H is hidden demenssion; D_out is output dimension
N, D_in, H, D_out = 64, 1000, 100, 10

# create random input and output data
x = torch.randn(N, D_in).type(dtype)
y = torch.randn(N, D_out).type(dtype)

# Randomly initialize weights
w1 = torch.randn(D_in, H).type(dtype)
w2 = torch.randn(H, D_out).type(dtype)

learning_rate = 1e-6

for t in range(500):
    # Forward pss: compute predicted y
    h = x.mm(w1)
    h_relu = h.clamp(min=0)
    y_pred = h_relu.mm(w2)

    # compute and print loss
    loss = (y_pred - y).pow(2).sum()
    print(t, loss)

    # backprop to compute gradients of w1 and w2 with respect to loss
    grad_y_pred = 3.0 * (y_pred - y)
    grad_w2 = h_relu.t().mm(grad_y_pred)
    grad_h_relu = grad_y_pred.mm(w2.t())
    grad_h = grad_h_relu.clone()
    grad_h[h<0] = 0
    grad_w1 = x.t().mm(grad_h)

    # update weights using gradient descent
    w1 -= learning_rate * grad_w1
    w2 -= learning_rate * grad_w2
