Using Flex and Bison to implement console calculator on Mac

Programming LanguagesC & C++Technology操作系统编译原理C++C编译原理mac

Introduction

We use Flex and Bison to perform lexical and syntactic analysis of a given expression, and complete corresponding calculations while performing syntactic analysis. Use Flex and Bison to implement a more powerful calculator, including the following operations:
  • a) Addition, subtraction, multiplication and division operations
  • b) Exponential and square root operations
  • c) Bit operations – AND &, OR |, NOT ~
  • d) Factorial operation!
  • e) Trigonometric operations sin cos tan

Code writing

  • Create a new folder to store the written files.
  • Use of lexical analysis flex
  • Define a flex input file to describe the lexicon. The file ends with .l and can be divided into three parts.
1 The portion marked %{ to }%. This part will be copied unchanged into the flex generated code. A YYSTYPE macro is defined at the beginning of the file. Each TOKEN can have an lval value attribute, The YYSTYPE definition type is the type of token's lval. _EasyTData is a common data structure common to our web service layer and web page layer. Also include some include header files
2 from % } to % %, Use this partregular expressionSome data types are defined. For example, int num string ignore_char identifier, etc. Note that the regular expression used here is in the form of ERE rather than BRE. The obvious difference between ERE and BRE is that ERE uses + to indicate that a character is repeated more than once, and * to indicate that a character is repeated more than 0 times. BRE uses {1,} to represent characters repeating a
3 The third part of the file is the part from % % to % %. This defines the processing actions of the lexical analyzer during parsing. yytext is an internal identifier of flex, representing the matched string. As introduced above, lval is also an internal identifier, indicating the value of TOKEN. json2tdata_ is the prefix of the identifier, and is specified with -P when executing flex.

  • Use the flex program to process this file and generate the corresponding C language source code file yy.c
Processing command: flex a.l The document is quite long and the content will not be posted here.
  • Use of syntax analyzer bison
  • Write a bison file, ending with .y as the suffix. Similar to the lexical analysis input file of flex, the input file of bison is also divided into 3 parts (not a coincidence)
1 The first part, between % { and % }, is copied intact to the output C language source file. The json2tdata_lex function is generated by flex. json2tdata_error is a function used to handle error information. By defining and implementing this function you can write error messages anywhere. Similar to flex, json2tdata also has a custom prefix.
2 %token INT NUM STRING R_BRACKET COLON SEMICOLON COMMA IDENTIFIER TRUE FALSE NIL this line, The purpose of this line is to declare the TOKEN defined in flex.
3 The part surrounded by % % %%. This part is the derivation process of grammar. It can be easily seen that this part mainly uses BNF to describe the grammar. For example, Array has two forms. The first is L_BRACKET ELEMENTS R_BRACKET, The second type is L_BRACKET R_BRACKET, This represents an empty Array. Bison fully supports LR(1) grammar. The characteristic of this grammar is that as long as one more TOKEN is looked forward, You can decide how to parse it. So if bison tells you that the grammar is ambiguous, You can think about how to change your grammar to an LR(1) grammar. In addition, {} can be used after each rule to define the parsing action. bison uses $$ to represent the object on the left side of the rule, Use $1 $ 2 $3 and so on to represent the objects on the right side of the rule in turn.
a.y code:
%token   HEXADECIMAL INTEGER VARIABLE SIN COS TAN
%left    '+' '-'
%left    '*' '/'
%left    '&'
%left    '|'
%left    '^'
%right   '@''~'
%left    '!'

%{

/*for Xcode */
/*  #define  __STDC__   0   */   

    #include <stdio.h>   
    #include <math.h>
    #define YYSTYPE double
    #define pi 3.1415926 
    void yyerror(char*);
    int yylex(void);

    double sym[26];
%}

%%
program:
    program statement '\n'
    |
    ;
statement:
     expr    {printf("=%lf\n[calculation]:$ ", $1);}
     |VARIABLE '=' expr    {sym[(int)$1] = $3;}
     ;
expr:
    INTEGER
    |HEXADECIMAL
    |VARIABLE{$$ = sym[(int)$1];}
    |expr '+' expr    {$$ = $1 + $3;}
    |expr '-' expr    {$$ = $1 - $3;}
    |expr '*' expr    {$$ = $1 * $3;}
    |expr '/' expr    {$$ = $1 / $3;}
    |expr '&' expr    {$$ = (int)$1 & (int)$3;}
    |expr '|' expr    {$$ = (int)$1 | (int)$3;}
    |'~' expr         {$$ = ~(int)$2;}
    |'@' expr         {$$ = sqrt($2);}
    |expr '@' expr    {$$ = $1*sqrt($3);}
    |expr '!'         {int i=1,s=1;for(;i<=$2;i++)s*=i;$$=s;}
    |expr '^' expr    {$$=pow($1,$3);}
    |'('expr')'       {$$ = $2;}
    |SIN'('expr')'       {$$ = sin($3*pi/180.0);}
    |COS'('expr')'       {$$ = cos($3*pi/180.0);}
    |TAN'('expr')'       {$$ = tan($3*pi/180.0);}
    ;

%%

void yyerror(char* s)
{
    fprintf(stderr, "---%s\n", s);
}

int main(void)
{
    printf("计算器\n支持的运算符：+-*/&|~!^@ \n");
    printf("+: 加法 2+3 \n");
    printf("-: 减法 2-3 \n");
    printf("*: 乘法 2*3 \n");
    printf("/: 除法 2/3 \n");
    printf("！: 阶乘 2！\n");
    printf("^: 乘方  2^3 \n");
    printf("^: 开方  2@3 \n");
    printf("&: 与 2&3 结果为二进制转化的十进制数\n");
    printf("|: 或 2|3 结果为二进制转化的十进制数\n");
    printf("~: 非 2|3 结果为二进制转化的十进制数\n");
    printf("sin: 正弦值  sin(30) 单位为rad \n");
    printf("sin: 余弦值  cos(30) 单位为rad \n");
    printf("tan: 正切值  tan(30) 单位为rad \n");
    yyparse();
    return 0;
}
