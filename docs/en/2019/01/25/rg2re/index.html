Type 3 grammar (regular grammar, linear grammar)

 If for some grammar G, each rule in P has the following form:U :: = T  or  U :: = WTwhere T∈VT；U,W∈VN, then the grammar G is called a left-linear grammar. If for some grammar G, each rule in P has the following form:U :: = T  or  U :: = TWwhere T∈VT；U, W∈VN, then the grammar G is called a right linear grammar. Left linear grammar and right linear grammar are commonly called type 3 grammar or regular grammar, sometimes also called finite state grammar, abbreviated as RG. By definition, when applying the rules to a regular grammar, a single nonterminal symbol can only be replaced by a single terminal symbol, or by a single nonterminal symbol plus a single terminal symbol, or by a single terminal symbol plus a single nonterminal symbol. The language identified by type 3 grammar is type 3 language L3, Type 3 languages ​​can be identified by deterministic finite state automata. The words of a programming language can be generated by a regular grammar. For example, the definition of an identifier can be described by a regular grammar as follows: <identifier>::=<letter>/<identifier><letter>/<identifier><number> Apparently, this grammar describes the set of alphanumeric strings starting with a letter. Now let's introduce another representation suitable for describing words - regular expressions. Regular expressions are also called regular expressions, and the set described by each regular expression is called a regular set. The reason why regular expressions are used to describe is mainly based on the following reasons:
  • The lexical rules are simple and do not require strict representation like context-free grammar. It is easier to use regular expression notation to understand a defined set of symbols than to understand a language defined by a set of rewriting rules;
  • It is easier to construct efficient recognition programs from regular expressions than context-free grammars;
  • An identification program can be automatically constructed from a certain regular expression, which can identify strings in a set of strings represented by the regular expression, thus reducing the workload of lexical analysis to be introduced later.
  • It can be used to process various other information flows. For example, it has been applied to some pattern recognition problems, bibliographic retrieval systems, and text editing programs.

Regular expressions and regular sets

There is an alphabet Σ. The regular expression on Σ and the regular set it represents are recursively defined as follows:
  • ε and Φ are both regular expressions on Σ, and the regular sets they represent are {ε} and Φ respectively, where ε is the empty string and Φ is the empty set;
  • Any a∈∑ is a regular expression, and the regular set it represents is {a};
  • if e1and e2are any regular expressions on Σ, and the regular sets represented respectively are L(e1) and L(e2),but:
    • e1/e2is also a regular expression, and the regular set represented is L(e1 / e2）＝L（e1）∪L（e2）。
    • e1 e2is also a regular expression, and the regular set represented is L(e1 e2）＝L（e1）L（e2）。
    • （e1）*is also a regular expression, and the regular set represented is L((e1）*）＝L（e1）*。
In the definition, (1) and (2) define atomic regular expressions, while (3) shows that regular expressions on the alphabet Σ can be composed of atomic regular expressions or simpler regular expressions through union, connection and closure operations to form general regular expressions.

Properties of regular expressions

If two regular expressions e1and e2If the regular sets they represent are the same, that is, their values ​​are equal, they are said to be equivalent. Marked as e1＝e2。

The relationship between regular expressions and regular grammar

The value of a regular expression is a regular set, which is another representation of a regular language. It is not difficult to see that, except for the symbol Φ, the meaning of a regular expression is similar to the meaning of the right part of a non-terminal symbol rule of a regular grammar. For example, for <number> ::= 0/1/2/…/9, the set of strings generated by nonterminal numbers is the same as the set of strings defined by the regular expression 0/1/2/…/9. The regular set Φ corresponds to a language that does not contain any sentences. The purpose of its introduction is mainly for theoretical completeness.

Grammar data structure:Considering that the grammar is a four-tuple, including Vn as a non-terminal symbol, Vt as a terminal symbol, P as a rule of the grammar, S as an identifier or start symbol, and flag as the type of grammar, so the following class in C++ is used to define the grammar, and a set collection is used to save certain attributes of each grammar (not repeated).

	public：
    		set<char> Vn;

#include<cstdio>
#include<iostream>
#include<string>
#include<vector>
#include<set>
#define MAX_LENGTH 10
using namespace std;

struct Principle{
    string left;
    string right;
};

struct Grammar{
    set<char> Vn;
    set<char> Vt;
    vector<Principle> P;
    set<Principle> R;
    char S;
    int flag = -1;
};

void input(vector<Principle> &principleSet,int time)
{
    char tempLeft[MAX_LENGTH];
    char tempRight[MAX_LENGTH];
    int temp2=0;
    while(scanf("%[^-]->%s",tempLeft,tempRight)!=EOF){
        getchar();
        Principle temp;
        temp.left = tempLeft;
        temp.right = tempRight;
        principleSet.push_back(temp);
        temp2++;
        if(temp2==time)break;
    }
}

void analyseGram(Grammar &G)
{
    G.S = G.P.front().left[0];
    for(int i=0;i<G.P.size();++i){
        Principle &temp = G.P[i];
        for(int j = 0;j<temp.left.size();++j){
            char v = temp.left[j];
            if(!isupper(v)){
                G.Vt.insert(v);
            }else{
                G.Vn.insert(v);
            }
        }
        for(int j = 0;j<temp.right.size();++j){
            char v = temp.right[j];
            if(!isupper(v)){
                G.Vt.insert(v);
            }else{
                G.Vn.insert(v);
            }
        }
    }
}

bool existVNT(string s)
{
    for(int i=0;i<s.size();i++){
        if(isupper(s[i])){
            return true;
        }
    }
    return false;
}

void judgeGram(Grammar &G)
{

    for(int i=0;i<G.P.size();++i){
        if(existVNT(G.P[i].left)){
            G.flag = 0;
            break;
        }
    }
    int temp = 0;
    for(int i=0;i<G.P.size()&&G.flag==0;++i){
        if(G.P[i].left.size()<=G.P[i].right.size()){
            temp++;
        }
    }
    if(temp==G.P.size()){
        G.flag=1;
    }
    temp = 0;
    for(int i=0;i<G.P.size()&&G.flag==1;++i){
        if(1==G.P[i].left.size()&&isupper(G.P[i].left[0])){
            temp++;
        }
    }
    if(temp==G.P.size()){
        G.flag=2;
    }
    temp = 0;
    for(int i=0;i<G.P.size()&&G.flag==2;++i){
        if(1==G.P[i].left.size()&&isupper(G.P[i].left[0])&&
           (1==G.P[i].right.size()&&!isupper(G.P[i].right[0])||2==G.P[i].right.size()&&!isupper(G.P[i].right[0])&&
            isupper(G.P[i].right[G.P[i].right.size()-1]))){temp++;}
    }
    if(temp==G.P.size()){
        G.flag=3;
    }
}

Difficulties encountered and solutions

① Usually if there are only two rules with the same non-terminal symbol S->a, S->b, then it can be directly simplified to S->a|b. If there are multiple target results as S->a|b|c, then special processing is required, and a stack or recursive call can be used to generate multiple OR expressions. ②The identification of closure*, that is, rule 2, is also the difficulty of this experiment. It is necessary to identify both the left and right sides of the grammar rules at the same time. ③ Sometimes the results obtained after passing rules 1, 2 and 3 are still incorrect, which requires the use of debugging tools in the integrated development environment. Improving debugging skills is also very important.④Finally, after reading the relevant literature, I found that converting regular grammar into regular expressions can be realized using the idea of ​​solving simultaneous equations, so the code can be written using MATLAB's method of solving symbolic functions.
