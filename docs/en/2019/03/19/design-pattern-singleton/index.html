Design pattern - Singleton pattern

Programming LanguagesC & C++TechnologySoftware EngineeringC++Software Engineering设计模式
Singleton mode, also known as singleton mode. definition:Make sure there is only one instance of a class, instantiate it yourself and provide this instance to the entire system. It is a commonly used design pattern and it belongs tocreator mode. The singleton pattern allows only one instance, hiding the object creation entry by privatizing the constructor, and instead provides a public interface for obtaining the singleton of the class. It has a wide range of application scenarios, such as creating management classes (only one management entity is required), or being applied to scenarios where objects need to be unique. Be careful not to abuse the singleton pattern, otherwise it will make code maintenance difficult.

detail

  1. A class is only allowed to create unique objects
  2. It is forbidden to create objects outside the class: private constructor: private or protected
  3. The only object maintained internally in the class: static member variables
  4. Provides methods for accessing singleton objects: static member functions that return the only instance constructed within the class

How to create

  1. Hungry Chinese style: No matter whether the singleton object is used or not, it is created as soon as the program starts.
  2. Lazy style: Singleton objects are created when used and destroyed when not in use.

C++ code

#include <iostream>
using namespace std;
 
class Singleton{
    public:
        static Singleton *GetInstance(){
            if (m_Instance == NULL ){
                m_Instance = new Singleton ();
                cout<<"实例生成！"<<endl;
            }else cout<<"实例已有！"<<endl;
            return m_Instance;
        }
        static void DestoryInstance(){
            if (m_Instance != NULL ){
                delete m_Instance;
                m_Instance = NULL ;
                cout<<"实例消除成功！"<<endl;
            }
        }
     
    private:
        Singleton(){
        }
        static Singleton *m_Instance;
};
Singleton *Singleton::m_Instance = NULL;
 
int main()
{
    Singleton *singletonObj1 = Singleton::GetInstance();
    Singleton *singletonObj2 = Singleton::GetInstance();
    Singleton::DestoryInstance();
    return 0;
}
