Minimum spanning tree Kruskal algorithm template question 2 C++ implementation

Programming LanguagesC & C++Data StructuresTechnologyC++Data StructuresC贪心算法poj
Basic idea:(1) Construct a subgraph containing only n vertices and an empty edge set. If each vertex in the graph is regarded as the root node of a tree, then it is a forest containing n trees. (2) Select an edge with the smallest weight from the edge set E of the network. If the two vertices of this edge belong to different trees, add it to the subgraph. In other words, the two trees where the two vertices are located are combined into one tree; on the contrary, if the two vertices of the edge have fallen on the same tree, it is not advisable. Instead, you should take the edge with the smallest weight and try again (3) and so on, until there is only one tree in the forest, that is, the subgraph contains n-1 edges.Vernacular:(1) Remove all edges in the picture. (2) Add edges to the graph in order from small to large weights to ensure that no loops are formed during the addition process. (3) Repeat the previous step until all vertices are connected. At this time, a minimum spanning tree is generated. This is a greedy strategy.difficulty:Determine whether the addition of an edge <u, v> will form a cycle in the selected edge set. Solution: Use union search to find the root nodes of the tree where the two vertices u and v are located. If the root nodes are the same, it means that u and v are in the same tree, and then u and v will form a ring when connected; if the root nodes are different, then u and v are not in the same tree, and they will not form a ring when connected, but the two trees will be merged.The time complexity of Kruskal's algorithm is O(eloge) (e is the number of edges in the network),Therefore, compared with Prim's algorithm, it is suitable for finding the minimum spanning tree of a network with sparse edges. Kruskal's algorithm finds the minimum spanning tree of the network from another approach. Assuming a connected network N = (V, {E}), let the initial state of the minimum spanning tree be a non-connected graph T = (V, {∮}) with only n vertices and no edges. Each vertex in the graph forms a connected component. Select the edge with the smallest cost in E. If the vertex attached to the edge falls on a different connected component in T, add this edge to T. Otherwise, discard this edge and select the next edge with the smallest cost. And so on, until all vertices in T are on the same connected component.

Bailian Tangled in Cables

Total time limit:
1000ms
Memory limit:
65536kB
describe
You are the owner of SmallCableCo and have purchased the franchise rights for a small town. Unfortunately, you lack enough funds to start your business properly and are relying on parts you have found in an old warehouse you bought. Among your finds is a single spool of cable and a lot of connectors. You want to figure out whether you have enough cable to connect every house in town. You have a map of town with the distances for all the paths you may use to run your cable between the houses. You want to calculate the shortest length of cable you must have to connect all of the houses together.
enter
Only one town will be given in an input.
  • The first line gives the length of cable on the spool as a real number.
  • The second line contains the number of houses, N
  • The next N lines give the name of each house's owner. Each name consists of up to 20 characters {a–z,A–Z,0–9} and contains no whitespace or punctuation.
  • Next line: M, number of paths between houses
  • next M lines in the form
< house name A > < house name B > < distance > Where the two house names match two different names in the list above and the distance is a positive real number. There will not be two paths between the same pair of houses.
output
The output will consist of a single line. If there is not enough cable to connect all of the houses in the town, output Not enough cable If there is enough cable, then output Need < X > miles of cable Print X to the nearest tenth of a mile (0.1).
Sample input
100.0
4
Jones
Smiths
Howards
Wangs
5
Jones Smiths 2.0
Jones Howards 4.2
Jones Wangs 6.7
Howards Wangs 4.0
Smiths Wangs 10.0
Sample output
Need 10.2 miles of cable
C++ implementation（AC） 
#include<iostream>
#include<vector>
#include<unordered_map>
#include<algorithm>
#include<cstdio>
using namespace std;

double cableLen = 0;
int n, m;
int father[501];

struct node{
    string src;
    string dst;
    double val;
};

int findFather(int x){
    while(x != father[x])x = father[x];
    return x;
}

bool unionSet(int x, int y){
    int fx = findFather(x);
    int fy = findFather(y);
    if(fx != fy){
        father[fy] = fx;
        return true;
    }
    return false;
}

bool cmp(node a, node b){return a.val < b.val;}

int main(){
    double anslen = 0;
    int k = 0, i = 0;
    scanf("%lf%d", &cableLen, &n);
    unordered_map<string, int> name;
    for(int i = 0; i < n; i++){
        string str;
        cin>>str;
        father[i] = i;
        name[str] = i;
    }
    scanf("%d", &m);
    vector<node> pool(m);
    for(int i = 0; i < m; i++){
        cin>>pool[i].src>>pool[i].dst>>pool[i].val;
    }
    sort(pool.begin(), pool.end(), cmp);
    while(k < n - 1){
        if(unionSet(name[pool[i].src], name[pool[i].dst])){
            anslen += pool[i].val;
            k++;
        }
        i++;
    }
    if(anslen <= cableLen)
        printf("Need %.1f miles of cable\n", anslen);
    else printf("%s\n", "Not enough cable");
    return 0;
}
