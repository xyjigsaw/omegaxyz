WordPress database error establishing a database connection error

TechnologyWebWebwordpress
As a webmaster who cannot afford a large server and can only use Alibaba Cloud student machines, I often encounter the error establishing a database connection. This is an error when establishing a database connection. It's very frustrating to get this error because all of a sudden we haven't moved anything at all.

Why does this error occur?

  1. The database login credentials are incorrect or changed, the database server is unresponsive, and the database is damaged.
  2. Database connection errors may also occur when website traffic is too heavy.
  3. The server is affected by multiple services. You can access other services to see if they are normal and determine whether it is caused by the database server.
  4. Whether the user's permissions are sufficient.
  5. Whether remote connection rejection is set up.

How to solve

Restart the server! ! ! This method is the most effective!So besides restarting, do we have any other methods? The first thing you should do is make sure you are getting the same error on both the frontend of your site and the backend of your site (wp-admin). If the error message is the same on both pages "Error establishing database connection". We can follow the steps below. Visit http://domain name/wp-admin/maint/repair.php You can see:Note that you can use the FTP tool to download wp-config.php and then upload it againAdd the following code in the wp-config.php file:
define('WP_ALLOW_REPAIR', true);
Note that the location of the added code is best in the middle.After saving, we visit the following url again: http://domain name/wp-admin/maint/repair.php You can see:The main function of accessing this URL is to repair and optimize the damaged database.After the modification is completed, we need to delete the code added above from the wp-config.php file. (If not deleted, it will pose a threat to the system) The method is also to use FTP.If you change the root password or database user password, you also need to change this file wp-config.php. Make sure the username and password for the configured database are correct.
define('DB_NAME', '数据库名');
define('DB_USER', '数据库用户名');
define('DB_PASSWORD', '数据库密码');
define('DB_HOST', '主机');
Note: Before any operation on the database, please make a backup to avoid accidents.
