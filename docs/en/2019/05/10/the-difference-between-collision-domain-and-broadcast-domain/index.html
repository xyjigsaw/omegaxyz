The difference between collision domain and broadcast domain

技术域计算机网络网络原理
Before talking about collision domains (collision domains) and broadcast domains, we must first introduce three network interconnection devices-hubs, switches and routers.

hub

The hub is a device that works at the physical layer. When it receives the data, it copies the data and sends the data to all interfaces once. Therefore, all interfaces of the hub are a collision domain and a broadcast domain.

switch

A switch is different from a hub. A switch works as a data link layer device. It can identify data frames and MAC addresses. The way it works is very different from a hub. Switches rely on MAC address tables to forward data. Data that is not in the MAC address table is broadcast. So we say that each interface of the switch is a collision domain, and all interfaces of the switch belong to a broadcast domain.

router

Routers are devices that work at the network layer. Routers rely on routing tables to forward data. For broadcast traffic the router will process but not forward the data. So we say that each interface of the router belongs to the same collision domain and broadcast domain. Routers can be used to isolate broadcasts.

Equipment summary

  • All ports of the hub (HUB) are in the same broadcast domain and collision domain, so the HUB cannot separate the collision domain and broadcast domain.
  • All ports of the switch are in the same broadcast domain, and each port is a collision domain. Therefore, the switch can separate the collision domain, but not the broadcast domain. Virtual LAN (Vlan) technology can isolate broadcast domains.
  • Each port of the Router belongs to a different broadcast domain and collision domain.
It can also be understood this way:
  • HUB is a first-tier device so it cannot separate conflict domains;
  • Switches and bridges are layer 2 devices so they can separate conflict domains;
  • Routers are layer 3 devices, so they can split both collision domains and broadcast domains.

conflict domain

The collection of all workstations connected to the same wire, or the collection of all nodes on the same physical network segment or the collection of nodes on the Ethernet competing for the same bandwidth.This domain represents the area in which conflicts occur and propagate, and this area can be considered a shared segment. In the OSI model, the collision domain is regarded as the first layer concept. The devices connected to the same collision domain include Hub, Repeater or other devices that simply copy signals. In other words, all nodes connected by Hub or Repeater can be considered to be in the same conflict domain, and it will not divide conflict domains. The second-layer devices (bridges, switches) and the third-layer devices (routers) can all divide conflict domains, and of course can also connect different conflict domains. Simply put, the Repeater can be regarded as a cable, and the network bridge can be regarded as a bundle of cables.

broadcast domain

A collection of nodes that receive the same broadcast message.For example, if any node in the set transmits a broadcast frame, all other nodes that can receive the frame are considered part of the broadcast frame. Since many devices are prone to broadcasting, if not maintained, a large amount of bandwidth will be consumed and the efficiency of the network will be reduced. Since the broadcast domain is considered a second-layer concept in OSI, nodes connected to first- and second-layer devices such as hubs and switches are considered to be in the same broadcast domain. Routers and Layer 3 switches can divide broadcast domains, that is, they can connect different broadcast domains.
