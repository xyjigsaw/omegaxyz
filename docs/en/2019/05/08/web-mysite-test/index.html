Introduction to Web Testing

Web-based system testing is different from traditional software testing. It not only needs to check and verify whether it runs according to the design requirements, but also tests whether the display of the system on different users' browsers is appropriate. Web testing requires the following aspects:
  • 1. Functional testing
  • 2. Performance testing (including load/stress testing)
  • 3. User interface testing
  • 4.Compatibility testing
  • 5. Security testing
  • 6. Interface testing

link test

Links are a main feature of Web application systems. They are the main means of switching between pages and guiding users to pages with unknown addresses. Link testing can be divided into three aspects. First, test whether all links actually link to the linked page as indicated; secondly, test whether the linked page exists; finally, ensure that there are no orphan pages on the web application system. The so-called orphan page means that there is no link pointing to the page, and it can only be accessed by knowing the correct URL address. Link testing can be automated and there are many tools available for this. Link testing must be completed during the integration testing phase, that is, link testing is performed after all pages of the entire Web application system are developed.

Xenu

The following uses Xenu for connection testing: Xenu is a green software used to detect the validity of website links. It is simple to use, supports up to 100 threads (Parallel threads), and the detection speed is very fast. When testing 100,630 6-layer connections on a website, it only took 1 hour and 40 minutes to use the default 100 threads. Of course, it consumes more network resources. Common status
  • ok, mail host ok: indicates that the link is normal;
  • timeout, no connection, no such host: indicates that the access has timed out or cannot be accessed (it does not mean that it is an empty link);
  • not found: If it is not found, it means that it is not found, that is, an empty link.
  • no info to return: No object is returned, that is, an empty page.
  • no object data: There is no object data, which is common in access errors such as 400 errors when accessing the server.
  • keep same verb: The URL is consistent, such as <a href="URL link A">URL link A</a>. After the software recognizes this status, it is recommended to check it in the source code. If there is only a form similar to the example just now, there is no need to worry. If there are other forms of links, you need to check whether they meet the SEO optimization standards before making modifications.
